2. Advanced Filtering in Compass

In Compass, click on the "Filter" bar and type these queries to find specific documents:

📋 Comparison Operators
// Price greater than $100 { "price": { "$gt": 100 } } // Price between $20 and $100 { "price": { "$gte": 20, "$lte": 100 } } // Stock less than 50 { "stock": { "$lt": 50 } } // Rating not equal to 4.5 { "rating": { "$ne": 4.5 } }
📋 Logical Operators
// Electronics with price > $100 OR rating > 4.8 { "$or": [ { "category": "Electronics", "price": { "$gt": 100 } }, { "rating": { "$gt": 4.8 } } ] } // Stationery AND stock > 200 { "$and": [ { "category": "Stationery" }, { "stock": { "$gt": 200 } } ] } // NOT Electronics { "category": { "$ne": "Electronics" } }
📋 Array & Element Operators
// Products with "gaming" tag { "tags": "gaming" } // Products that have BOTH "office" AND "accessory" tags { "tags": { "$all": ["office", "accessory"] } } // Products that have at least 2 tags { "tags": { "$size": 2 } } // Products where rating exists { "rating": { "$exists": true } } // Products where price is a number { "price": { "$type": "number" } }
📋 Regular Expressions (Text Search)
// Products starting with "Gaming" { "name": { "$regex": "^Gaming", "$options": "i" } } // Products containing "Pro" anywhere { "name": { "$regex": "Pro", "$options": "i" } } // Products ending with "Mouse" { "name": { "$regex": "Mouse$", "$options": "i" } }
3. Sorting & Field Selection (Projection)

In Compass, use the "Sort" and "Project" fields to organize your data:

📊 Sorting
// Sort by price descending (highest first) { "price": -1 } // Sort by rating descending, then name ascending { "rating": -1, "name": 1 } // Sort by createdAt (oldest first) { "createdAt": 1 }
👁️ Projection (Select specific fields)
// Show only name, price, and rating (hide _id) { "name": 1, "price": 1, "rating": 1, "_id": 0 } // Show everything except stock { "stock": 0 }
Pro Tip: Use projections to reduce data transfer and make queries faster!
4. Aggregation Pipeline – The Power of MongoDB

The Aggregation Pipeline is MongoDB's most powerful feature – it lets you process, transform, and analyze data through a series of stages. In Compass, click the "Aggregations" tab to build pipelines visually!

🔧 Common Aggregation Stages
$match
Filter documents (like WHERE in SQL)
$group
Group documents by a field
$sort
Sort documents
$project
Select/transform fields
$limit
Limit number of documents
$skip
Skip documents (pagination)
$unwind
Deconstruct arrays
$lookup
Join collections (like SQL JOIN)
📝 Example 1: Group by Category (Total Value)
Stage 1: $group – Group by category and calculate totals
[ { $group: { _id: "$category", // Group by category totalProducts: { $sum: 1 }, // Count products averagePrice: { $avg: "$price" }, // Average price totalStock: { $sum: "$stock" }, // Total stock count maxPrice: { $max: "$price" }, // Highest price minPrice: { $min: "$price" } // Lowest price } }, { $sort: { totalProducts: -1 } } // Sort by most products first ]
📝 Example 2: Products with Rating > 4.5, Sorted by Price
[ { $match: { rating: { $gt: 4.5 } } }, // Filter high-rated products { $sort: { price: -1 } }, // Sort by price descending { $limit: 5 } // Get top 5 ]
📝 Example 3: Add Discount Field & Round Prices
[ { $project: { name: 1, originalPrice: "$price", discountedPrice: { $multiply: ["$price", 0.9] }, // 10% discount roundedPrice: { $round: ["$price", 0] }, // Round to integer category: 1, inStock: { $gt: ["$stock", 0] } // Boolean: true if stock > 0 } } ]
📝 Example 4: Analyze Tags (Unwind Array)
[ { $unwind: "$tags" }, // Create one document per tag { $group: { _id: "$tags", // Group by each tag count: { $sum: 1 }, // Count products with this tag averagePrice: { $avg: "$price" } // Average price for tag } }, { $sort: { count: -1 } }, // Most popular tags first { $limit: 5 } // Top 5 tags ]
5. Advanced Aggregation Examples
📈 Revenue Analysis (Price × Stock)
[ { $project: { name: 1, category: 1, potentialRevenue: { $multiply: ["$price", "$stock"] } } }, { $group: { _id: "$category", totalRevenue: { $sum: "$potentialRevenue" } } }, { $sort: { totalRevenue: -1 } } ]
📅 Monthly Product Creation Count
[ { $group: { _id: { year: { $year: "$createdAt" }, month: { $month: "$createdAt" } }, count: { $sum: 1 }, products: { $push: "$name" } } }, { $sort: { "_id.year": 1, "_id.month": 1 } } ]
⭐ Rating Distribution (Bucketing)
[ { $bucket: { groupBy: "$rating", boundaries: [0, 3, 4, 4.5, 5], default: "Other", output: { count: { $sum: 1 }, products: { $push: "$name" } } } } ]
6. Indexes – Making Queries Fast

Indexes dramatically speed up queries. In Compass, click the "Indexes" tab to manage them.

🔍 Creating an Index
// Single field index { "category": 1 } // Compound index (multiple fields) { "category": 1, "price": -1 } // Text index (for text search) { "name": "text", "tags": "text" } // Unique index (prevents duplicates) { "email": 1 } with unique: true
📊 When to Create Indexes
✅ Create indexes for:

• Fields used in WHERE (filter) clauses
• Fields used for sorting
• Fields used in JOINs ($lookup)
• Unique fields (email, username)

⚠️ Avoid indexes for:

• Small collections (< 1000 docs)
• Fields with low cardinality (e.g., boolean)
• Fields that change frequently

Performance Tip: Use Compass's "Explain Plan" feature to see if your query is using an index!
