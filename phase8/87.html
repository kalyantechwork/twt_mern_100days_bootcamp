🛡️ Protected Routes in React
Route Guards · Auth Context · Redirect to Login · Role-Based Access Control

1️⃣ What are Protected Routes?

Protected Routes are pages or routes that only authenticated users can access. Examples: Dashboard, Profile, Admin Panel, Settings.

User clicks Dashboard → Check if logged in? → Yes → Show Dashboard → No → Redirect to Login
💡 Analogy: Like a VIP section at a concert — only users with a valid ticket (authentication token) can enter.

🔐 Security

Prevent unauthorized access to private pages

👤 User Experience

Redirect to login instead of showing error

📊 Data Protection

Sensitive data only visible to authenticated users

🏢 Professional

Every real app has protected routes

3️⃣ Basic Protected Route Component
// components/ProtectedRoute.jsx
import { Navigate } from 'react-router-dom';
import { useAuth } from '../context/AuthContext';

function ProtectedRoute({ children, allowedRoles = [] }) {
const { user, loading } = useAuth();

if (loading) return <div>Loading...</div>;

if (!user) {
return <Navigate to="/login" replace />;
}

if (allowedRoles.length > 0 && !allowedRoles.includes(user.role)) {
return <Navigate to="/unauthorized" replace />;
}

return children;
}

export default ProtectedRoute;

<Route path="/register" element={<Register />} />

{/* Protected Routes */}
<Route path="/dashboard" element={
<ProtectedRoute>
<Dashboard />
</ProtectedRoute>
} />

<Route path="/profile" element={
<ProtectedRoute>
<Profile />
</ProtectedRoute>
} />

{/* Role-Based Protected Route */}
<Route path="/admin" element={
<ProtectedRoute allowedRoles={['admin']}>
<AdminPanel />
</ProtectedRoute>
} />
</Routes>
</AuthProvider>
</BrowserRouter>
);
}
5️⃣ Auth Context — Global Authentication State
// context/AuthContext.jsx
import { createContext, useContext, useState, useEffect } from 'react';

const AuthContext = createContext();

export function AuthProvider({ children }) {
const [user, setUser] = useState(null);
const [loading, setLoading] = useState(true);

useEffect(() => {
// Check localStorage for existing token on app load
const token = localStorage.getItem('token');
if (token) {
// Verify token and load user data
fetchUser(token).then(userData => {
setUser(userData);
setLoading(false);
}).catch(() => {
localStorage.removeItem('token');
setLoading(false);
});
} else {
setLoading(false);
}
}, []);

const login = async (email, password) => {
const response = await authAPI.login({ email, password });
localStorage.setItem('token', response.token);
setUser(response.user);
};

const logout = () => {
localStorage.removeItem('token');
setUser(null);
};

return (
<AuthContext.Provider value={{ user, login, logout, loading }}>
{children}
</AuthContext.Provider>
);
}

export const useAuth = () => useContext(AuthContext);
6️⃣ Role-Based Access Control (Admin vs User)
// AdminRoute.jsx — only admin can access
function AdminRoute({ children }) {
const { user } = useAuth();
if (!user) return <Navigate to="/login" />;
if (user.role !== 'admin') return <Navigate to="/unauthorized" />;
return children;
}

// Usage in component — conditional rendering based on role
function Dashboard() {
const { user } = useAuth();
return (
<div>
<h1>Welcome {user.name}</h1>
{user.role === 'admin' && (
<button>Admin Settings</button>
)}
</div>
);
}

8️⃣ Redirect After Login — Preserve Intended Destination
// Login.jsx
import { useNavigate, useLocation } from 'react-router-dom';

function Login() {
const navigate = useNavigate();
const location = useLocation();
const from = location.state?.from?.pathname || '/';

const handleLogin = async () => {
await login(email, password);

}

// In ProtectedRoute
if (!user) {
return <Navigate to="/login" state={{ from: location }} replace />;

⚠️ Common Protected Route Mistakes
  • ❌ Only protecting routes on frontend (backend must also protect APIs!)
  • ❌ Storing user role in localStorage without validation from backend
  • ❌ Not handling loading state → flash of protected content before redirect
  • ❌ Forgetting to add replace in Navigate → user can go back to protected page
💼 Common Interview Questions

❓ How do you protect routes in React?

Create a ProtectedRoute component that checks authentication and redirects to login if not authenticated.

❓ How to implement role-based access?

Check user role in ProtectedRoute and redirect to unauthorized page if role not allowed.

❓ How to preserve intended destination after login?

Use location.state to store the original URL and navigate back after login.
