1️⃣ Full Stack Architecture — React + Node.js

Today we connect our React frontend with Node.js/Express backend. This is the core of MERN stack — frontend and backend working together!

React Frontend (Port 5173) → API Request (fetch/Axios) → Node.js Backend (Port 5000) → Response → Frontend Updates UI
💡 Analogy: React is the restaurant customer (requests food). Node.js/Express is the kitchen (prepares response). The API is the waiter delivering between them.

3️⃣ Step 1 — Create Backend API (Express)
// backend/server.js
const express = require('express');
const cors = require('cors');

const app = express();
const PORT = 5000;

// Middleware
app.use(cors()); // Allow frontend to call backend
app.use(express.json()); // Parse JSON request bodies

// Sample data
const users = [
{ id: 1, name: "Kalyan", email: "kalyan@example.com" },
{ id: 2, name: "Priya", email: "priya@example.com" }
];

// API Routes
app.get('/api/users', (req, res) => {
res.json(users);
});

app.post('/api/users', (req, res) => {
const newUser = { id: Date.now(), ...req.body };
users.push(newUser);
res.status(201).json(newUser);
});

app.listen(PORT, () => {
console.log(`Server running on port ${PORT}`);
});
4️⃣ CORS — Why It's Needed

🌐 CORS = Cross-Origin Resource Sharing

Security feature that prevents websites from calling APIs on different domains without permission.

🛠️ React (localhost:5173) → Backend (localhost:5000)

Different ports = different origins. CORS must be enabled!

// Without CORS — Browser blocks request ❌
// With CORS — Browser allows request ✅

npm install cors
app.use(cors()); // Allow all origins (for development)
5️⃣ Step 2 — Frontend API Call with Axios
// frontend/src/services/api.js
import axios from 'axios';

const API_URL = 'http://localhost:5000/api';

export const fetchUsers = async () => {
const response = await axios.get(`${API_URL}/users`);
return response.data;
};

export const addUser = async (user) => {
const response = await axios.post(`${API_URL}/users`, user);
return response.data;
};
📦 Axios vs Fetch: Axios automatically transforms JSON, has better error handling, and shorter syntax. Install with `npm install axios`.
6️⃣ React Component — Display Users
// frontend/src/components/UserList.jsx
import { useState, useEffect } from 'react';
import { fetchUsers, addUser } from '../services/api';

function UserList() {
const [users, setUsers] = useState([]);
const [loading, setLoading] = useState(true);
const [newName, setNewName] = useState('');

useEffect(() => {
loadUsers();
}, []);

const loadUsers = async () => {
const data = await fetchUsers();
setUsers(data);
setLoading(false);
};

const handleAddUser = async () => {
const newUser = await addUser({ name: newName, email: `${newName}@example.com` });
setUsers([...users, newUser]);
setNewName('');
};

if (loading) return <div>Loading...</div>;
return (
<div>
<h2>Users ({users.length})</h2>
<ul>{users.map(user => <li key={user.id}>{user.name} - {user.email}</li>)}</ul>
<input value={newName} onChange={(e) => setNewName(e.target.value)} placeholder="Name" />
<button onClick={handleAddUser}>Add User</button>
</div>
);
}
7️⃣ Vite Proxy — Avoid CORS in Development
// frontend/vite.config.js
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';

export default defineConfig({
plugins: [react()],
server: {
proxy: {
'/api': {
target: 'http://localhost:5000',
changeOrigin: true,
}
}
}
});
🚀 Proxy Benefit: Frontend calls `/api/users` instead of `http://localhost:5000/api/users` — no CORS issues!

  • ❌ Forgetting CORS → browser blocks requests (check console)
  • ❌ Backend not running → frontend can't connect (check terminal)
  • ❌ Wrong API URL → 404 errors
  • ❌ Not handling async/await properly → missing await
💼 Common Interview Questions

❓ What is CORS?

Security feature that restricts cross-origin requests. Enable with `cors()` middleware.

❓ What is the Vite proxy?

Dev feature that forwards API requests to backend, avoiding CORS issues.
