2️⃣ What is Multer?

Multer is a Node.js middleware for handling multipart/form-data, primarily used for uploading files. It processes files and makes them available in req.file or req.files.

Client (multipart/form-data) → Multer Middleware → Parse Files → req.file (single) or req.files (multiple) → Your Route Handler
📦 Installation: npm install multer
3️⃣ Basic Setup — Store on Local Disk
const multer = require('multer');
const path = require('path');

// Configure storage
const storage = multer.diskStorage({
destination: (req, file, cb) => {
cb(null, 'uploads/'); // Save to 'uploads' folder
},
filename: (req, file, cb) => {
const uniqueSuffix = Date.now() + '-' + Math.round(Math.random() * 1E9);
cb(null, file.fieldname + '-' + uniqueSuffix + path.extname(file.originalname));
}
});

// File filter (optional — only allow images)
const fileFilter = (req, file, cb) => {
const allowedTypes = ['image/jpeg', 'image/png', 'image/jpg', 'image/gif'];
if (allowedTypes.includes(file.mimetype)) {
cb(null, true);
} else {
cb(new Error('Only images are allowed'), false);
}
};

const upload = multer({ storage, fileFilter, limits: { fileSize: 5 * 1024 * 1024 } }); // 5MB limit
4️⃣ Single File Upload — Express Route
// backend/routes/upload.js
const express = require('express');
const multer = require('multer');
const router = express.Router();

// Setup multer (as above)
const upload = multer({ storage, fileFilter });

// Single file upload (field name = 'avatar')
router.post('/upload-avatar', upload.single('avatar'), (req, res) => {
try {
if (!req.file) {
return res.status(400).json({ error: 'No file uploaded' });
}
// File URL to save in database
const fileUrl = `/uploads/${req.file.filename}`;
res.json({ message: 'File uploaded successfully', fileUrl, file: req.file });
} catch (error) {
res.status(500).json({ error: error.message });
}
});

// Multiple files upload (field name = 'images')
router.post('/upload-multiple', upload.array('images', 5), (req, res) => {
const fileUrls = req.files.map(file => `/uploads/${file.filename}`);
res.json({ message: 'Files uploaded', files: fileUrls });
});

setUploading(false);
}
};

return (/* JSX with input type="file" */);
}

8️⃣ Cloud Storage — Upload to Cloudinary (Production)
npm install cloudinary multer-storage-cloudinary

const cloudinary = require('cloudinary').v2;
const { CloudinaryStorage } = require('multer-storage-cloudinary');

cloudinary.config({
cloud_name: process.env.CLOUD_NAME,
api_key: process.env.CLOUD_API_KEY,
api_secret: process.env.CLOUD_API_SECRET
});

const storage = new CloudinaryStorage({
cloudinary: cloudinary,
params: {
folder: 'user-avatars',
allowed_formats: ['jpg', 'png', 'jpeg']
}
});

const upload = multer({ storage });
// Then use the same upload.single() — files go to Cloudinary!
☁️ Why Cloudinary? No need to manage server storage, automatic CDN, image optimization, resizing, and transformations built-in.

⚠️ Common Multer Mistakes
  • ❌ Forgetting enctype="multipart/form-data" in HTML form → files not sent
  • ❌ Not creating uploads folder before first upload → ENOENT error
  • ❌ Not handling file size limits → server crashes on large files
  • ❌ Storing files directly in database (store path, not file content)
💼 Common Interview Questions

❓ What is Multer?

Node.js middleware for handling multipart/form-data (file uploads).

❓ How to validate file type?

Use fileFilter option in multer config to check mimetype.

❓ Where should files be stored in production?

Cloud storage (Cloudinary, AWS S3) — not on server disk.
