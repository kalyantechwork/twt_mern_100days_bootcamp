1️⃣ JWT Structure — What's Inside?

JWT (JSON Web Token) is an open standard for securely transmitting information between parties. It consists of three parts separated by dots.

xxxxx.yyyyy.zzzzz Header.Payload.Signature

📋 Header

{ "alg": "HS256", "typ": "JWT" }
Algorithm & token type

📦 Payload

{ "id": 123, "email": "user@example.com", "exp": 1234567890 }
Claims (user data + expiry)

🔐 Signature

HMACSHA256(base64UrlEncode(header) + "." + base64UrlEncode(payload), secret)
Verifies token wasn't tampered

2️⃣ Generating JWT (jsonwebtoken library)
npm install jsonwebtoken

const jwt = require('jsonwebtoken');
const JWT_SECRET = process.env.JWT_SECRET || 'your-secret-key-min-32-chars';
const JWT_EXPIRES_IN = '7d'; // 7 days

// Generate token
const generateToken = (user) => {
return jwt.sign(
{ id: user._id, email: user.email, role: user.role }, // payload
JWT_SECRET, // secret
{ expiresIn: JWT_EXPIRES_IN } // options
);
};

// Example usage in login route
const token = generateToken(user);
res.json({ user, token });
3️⃣ Verifying JWT — Authentication Middleware
const verifyToken = (req, res, next) => {
try {
// Get token from Authorization header
const authHeader = req.headers.authorization;
if (!authHeader || !authHeader.startsWith('Bearer ')) {
return res.status(401).json({ error: 'No token provided' });
}

const token = authHeader.split(' ')[1];
const decoded = jwt.verify(token, JWT_SECRET);
req.user = decoded; // Attach user info to request
next();
} catch (error) {
if (error.name === 'TokenExpiredError') {
return res.status(401).json({ error: 'Token expired' });
}
return res.status(403).json({ error: 'Invalid token' });
}
};

// Use in protected routes
router.get('/profile', verifyToken, (req, res) => {
res.json({ user: req.user });
});

5️⃣ Refresh Tokens — Seamless User Experience
// Refresh token model (store in database)
const refreshTokenSchema = new mongoose.Schema({
token: { type: String, required: true },
userId: { type: mongoose.Schema.Types.ObjectId, ref: 'User' },
expiresAt: { type: Date, required: true }
});

// Generate refresh token (longer expiry)
const generateRefreshToken = (userId) => {
const refreshToken = jwt.sign({ id: userId }, REFRESH_SECRET, { expiresIn: '30d' });
// Save to database
await RefreshToken.create({ token: refreshToken, userId, expiresAt: new Date(Date.now() + 30*24*60*60*1000) });
return refreshToken;
};

// Refresh endpoint
router.post('/refresh-token', async (req, res) => {
const { refreshToken } = req.body;
const storedToken = await RefreshToken.findOne({ token: refreshToken });
if (!storedToken || storedToken.expiresAt < new Date()) {
return res.status(401).json({ error: 'Invalid refresh token' });
}
const decoded = jwt.verify(refreshToken, REFRESH_SECRET);
const newAccessToken = generateToken({ _id: decoded.id });
res.json({ accessToken: newAccessToken });
});
🔄 Refresh Token Flow: Access token (short-lived, 15min) → expires → use refresh token → get new access token → user stays logged in.
6️⃣ JWT Security Best Practices
  • ✅ Use strong JWT secret (min 32 characters, random string)
  • ✅ Store JWT_SECRET in environment variables (.env), never in code
  • ✅ Set short expiry for access tokens (15 minutes to 1 hour)
  • ✅ Use refresh tokens for long-lived sessions
  • ✅ Store tokens in httpOnly cookies (more secure) instead of localStorage
  • ✅ Always use HTTPS in production
  • ✅ Don't store sensitive data in JWT payload (it's base64 encoded, not encrypted)
7️⃣ Complete JWT Authentication Flow
Register/Login → Server validates credentials → Generates Access Token + Refresh Token → Client stores tokens ↓ Client sends Access Token in Authorization header for protected requests ↓ Server verifies token → valid → serves data ↓ Access Token expires → Client sends Refresh Token to /refresh endpoint → Server issues new Access Token
8️⃣ Frontend JWT Handling — Axios Interceptors
// axiosConfig.js
import axios from 'axios';

const api = axios.create({
baseURL: process.env.VITE_API_URL
});

// Request interceptor — add token
api.interceptors.request.use((config) => {
const token = localStorage.getItem('accessToken');
if (token) config.headers.Authorization = `Bearer ${token}`;
return config;
});

// Response interceptor — handle token expiry
api.interceptors.response.use(
(response) => response,
async (error) => {
const originalRequest = error.config;
if (error.response?.status === 401 && !originalRequest._retry) {
originalRequest._retry = true;
try {
const refreshToken = localStorage.getItem('refreshToken');
const response = await axios.post('/api/auth/refresh-token', { refreshToken });
localStorage.setItem('accessToken', response.data.accessToken);
originalRequest.headers.Authorization = `Bearer ${response.data.accessToken}`;
return api(originalRequest);
} catch (refreshError) {
localStorage.removeItem('accessToken');
localStorage.removeItem('refreshToken');
window.location = '/login';
}
}
return Promise.reject(error);
}

⚠️ Common JWT Mistakes
  • ❌ Storing sensitive data in JWT payload (it's base64 encoded, not encrypted!)
  • ❌ Using too long expiry for access tokens (security risk)
  • ❌ Hardcoding JWT secret in code (use environment variables)
  • ❌ Not validating token signature on every request
💼 Common Interview Questions

❓ What are the parts of a JWT?

Header, Payload, Signature (Base64 encoded, separated by dots).

❓ Why use refresh tokens?

Access tokens are short-lived for security; refresh tokens allow obtaining new access tokens without re-login.

❓ Where to store JWT on frontend?

localStorage (simple) or httpOnly cookies (more secure for production).

❓ Can JWT be decoded without secret?

Yes, payload is base64 encoded, not encrypted. Never store passwords in JWT!
