📡 Axios — Complete HTTP Client
GET, POST, PUT, DELETE Requests · Interceptors · Error Handling

1️⃣ What is Axios?

Axios is a promise-based HTTP client for making API requests from browser (React) or Node.js. It's the most popular alternative to fetch() with better features.

React Component → Axios → API Server → Response → React Updates UI
💡 Why Axios over fetch? Auto JSON parsing, better error handling, request/response interceptors, supports older browsers, shorter syntax, cancels requests.

3️⃣ Axios vs Fetch — Syntax Comparison
OperationFetchAxios
GETfetch(url).then(res => res.json())axios.get(url).then(res => res.data)
POSTfetch(url, { method:'POST', body: JSON.stringify(data), headers })axios.post(url, data)
Error HandlingNeed to check response.okAutomatic (catches 4xx/5xx)
TimeoutsNot built-inaxios.defaults.timeout = 5000
4️⃣ GET Request — Fetch Data
import axios from 'axios';

// Basic GET
const fetchUsers = async () => {
try {
const response = await axios.get('https://jsonplaceholder.typicode.com/users');
console.log(response.data); // Data is automatically parsed
} catch (error) {
console.error('Error:', error.message);
}
};

// GET with query parameters
const fetchUser = async (id) => {
const response = await axios.get(`/api/users/${id}`);
// OR: axios.get('/api/users', { params: { id } })
return response.data;
};
5️⃣ POST Request — Send Data
const addUser = async (userData) => {
try {
const response = await axios.post('https://jsonplaceholder.typicode.com/users', {
name: userData.name,
email: userData.email,
});
console.log('User created:', response.data);
return response.data;
} catch (error) {
console.error('POST error:', error.response?.data || error.message);
}
};
6️⃣ PUT Request — Update Entire Resource
const updateUser = async (id, userData) => {
try {
const response = await axios.put(`https://jsonplaceholder.typicode.com/users/${id}`, {
name: userData.name,
email: userData.email,
});
console.log('User updated:', response.data);
return response.data;
} catch (error) {
console.error('PUT error:', error.message);
}
};
7️⃣ DELETE Request — Remove Resource
const deleteUser = async (id) => {
try {
await axios.delete(`https://jsonplaceholder.typicode.com/users/${id}`);
console.log(`User ${id} deleted`);
return true;
} catch (error) {
console.error('DELETE error:', error.message);
return false;
}
};

9️⃣ Axios Instance — DRY Configuration
// services/api.js
import axios from 'axios';

const api = axios.create({
baseURL: 'http://localhost:5000/api',
timeout: 10000,
headers: { 'Content-Type': 'application/json' }
});

// Add auth token to all requests
api.interceptors.request.use(config => {
const token = localStorage.getItem('token');
if (token) config.headers.Authorization = `Bearer ${token}`;
return config;
});

export const getUsers = () => api.get('/users');
export const getUser = (id) => api.get(`/users/${id}`);
export const createUser = (data) => api.post('/users', data);
export const updateUser = (id, data) => api.put(`/users/${id}`, data);
export const deleteUser = (id) => api.delete(`/users/${id}`);
🔟 Interceptors — Global Request/Response Handling
// Request Interceptor (before request is sent)
api.interceptors.request.use(
config => {
console.log(`Making request to ${config.url}`);
// Add loading indicator
return config;
},
error => Promise.reject(error)
);

// Response Interceptor (after response is received)
api.interceptors.response.use(
response => {
console.log('Response received:', response.status);
return response;
},
error => {
if (error.response?.status === 401) {
// Redirect to login
window.location = '/login';
}
return Promise.reject(error);
}
);
🚨 Interceptors use case: Add auth tokens to all requests, log errors globally, show/hide loading spinners automatically.
1️⃣1️⃣ Axios Error Handling
try {
const response = await api.get('/users');
} catch (error) {
if (error.response) {
// Server responded with error status (4xx, 5xx)
console.log('Status:', error.response.status);
console.log('Data:', error.response.data);
} else if (error.request) {
// Request was made but no response
console.log('Network error:', error.message);
} else {
// Something else
console.log('Error:', error.message);
}
}
1️⃣2️⃣ Concurrent Requests (Promise.all)
const fetchMultiple = async () => {
try {
const [users, posts, comments] = await Promise.all([
api.get('/users'),
api.get('/posts'),
api.get('/comments')
]);
console.log('All data loaded:', { users: users.data, posts: posts.data });
} catch (error) {
console.error('One of the requests failed:', error);
}
};

⚠️ Common Axios Mistakes
  • ❌ Forgetting await or .then() → Promise not resolved
  • ❌ Not handling errors with try/catch → app crashes silently
  • ❌ Not using Axios instance → repeated base URL configuration
  • ❌ Not checking error.response before accessing → TypeError
💼 Common Interview Questions

❓ What is Axios?

Promise-based HTTP client for browser and Node.js, popular alternative to fetch.

❓ Axios vs fetch?

Axios auto-parses JSON, better error handling, interceptors, cancel requests, timeout support.

❓ What are interceptors?

Functions that intercept requests/responses globally — for auth tokens, logging, loading states.

❓ How to set default headers in Axios?

axios.defaults.headers.common['Authorization'] = token or use instance config.
