👑 Role-Based Access Control (RBAC)
Admin vs User Permissions · Frontend & Backend Authorization · Role Middleware

1️⃣ What is Role-Based Access Control (RBAC)?

RBAC is a method of restricting system access to authorized users based on their role (e.g., admin, user, moderator, guest). Different roles have different permissions.

User Login → Check Role → Admin → Access Admin Dashboard, Delete Users, View Analytics User Login → Check Role → Regular User → Access Profile, View Products, Add to Cart
💡 Analogy: Hotel key cards — guests can only access their room and lobby, staff can access maintenance areas, managers can access all areas.

👑 Admin

Full system access — manage users, view analytics, delete content, configure settings

👤 Regular User

Basic access — view profile, edit own data, create content, leave comments

✍️ Editor/Moderator

Content management — edit/delete posts, manage comments, approve content

👁️ Guest

View-only — read public content, cannot interact or modify anything

5️⃣ Backend Role Middleware — Protect API Routes
// middleware/roleMiddleware.js
const authorize = (...allowedRoles) => {
return (req, res, next) => {
if (!req.user) {
return res.status(401).json({ error: 'Not authenticated' });
}
if (!allowedRoles.includes(req.user.role)) {
return res.status(403).json({ error: 'Access denied. Insufficient permissions.' });
}
next();
};
};

// Usage in routes
router.get('/admin/users', authenticate, authorize('admin'), async (req, res) => {
const users = await User.find({});
res.json(users);
});

router.delete('/admin/users/:id', authenticate, authorize('admin'), async (req, res) => {
await User.findByIdAndDelete(req.params.id);
res.json({ message: 'User deleted' });
});

router.put('/posts/:id', authenticate, authorize('admin', 'moderator'), async (req, res) => {
// Admin OR moderator can edit posts
});
6️⃣ Frontend — Conditional Rendering by Role
// components/Dashboard.jsx
import { useAuth } from '../context/AuthContext';

function Dashboard() {
const { user } = useAuth();

return (
<div>
<h1>Welcome, {user.name}!</h1>
<p>Your role: {user.role}</p>

{/* Admin-only section */}
{user.role === 'admin' && (
<div className="admin-section">
<h2>Admin Controls</h2>
<button>Manage Users</button>
<button>View Analytics</button>
<button>System Settings</button>
</div>
)}

{/* Moderator + Admin section */}
{(user.role === 'admin' || user.role === 'moderator') && (
<div className="moderator-section">
<h2>Content Management</h2>
<button>Edit Posts</button>
<button>Manage Comments</button>
</div>
)}

{/* All authenticated users */}
<div className="user-section">
<h2>My Profile</h2>
<button>Edit Profile</button>
</div>
</div>
);
}

8️⃣ ProtectedRoute Component with Role Checking
// components/RoleBasedRoute.jsx
function RoleBasedRoute({ children, allowedRoles }) {
const { user, loading } = useAuth();
const navigate = useNavigate();

if (loading) return <div>Loading...</div>;
if (!user) return <Navigate to="/login" replace />;
if (!allowedRoles.includes(user.role)) {
return <Navigate to="/unauthorized" replace />;
}
return children;
}

// Usage in App.jsx
<Route path="/admin" element={
<RoleBasedRoute allowedRoles={['admin']}>
<AdminDashboard />
</RoleBasedRoute>
} />

<Route path="/moderator" element={
<RoleBasedRoute allowedRoles={['admin', 'moderator']}>
<ModeratorPanel />
</RoleBasedRoute>
} />
9️⃣ Permission-Based Rendering Hook
// hooks/usePermissions.js
export function usePermissions() {
const { user } = useAuth();

const hasRole = (roles) => {
if (!user) return false;
return roles.includes(user.role);
};

const isAdmin = () => user?.role === 'admin';
const isModerator = () => user?.role === 'moderator';
const isUser = () => user?.role === 'user';

return { hasRole, isAdmin, isModerator, isUser };
}

// Usage in component
function DeleteButton({ postId }) {
const { isAdmin } = usePermissions();
if (!isAdmin()) return null;
return <button onClick={() => deletePost(postId)}>Delete Post</button>;
}

⚠️ Common RBAC Mistakes
  • ❌ Only checking roles on frontend — backend must also validate!
  • ❌ Hardcoding role checks everywhere (create reusable middleware/hooks)
  • ❌ Not defaulting new users to 'user' role — accidentally creating admins
  • ❌ Storing role in localStorage without backend verification
💼 Common Interview Questions

❓ What is RBAC?

Role-Based Access Control — restricting access based on user role (admin, user, etc).

❓ How to implement RBAC in backend?

Create role middleware that checks user.role from JWT and allows/denies access.

❓ Why check roles on both frontend AND backend?

Frontend for UX (hide buttons), backend for security (prevent API abuse).
