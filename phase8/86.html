🔐 Password Hashing with bcrypt
Secure Password Storage · Salt Rounds · Hash Comparison · Security Best Practices

1️⃣ Why Hash Passwords? (Never Store Plain Text!)

NEVER store passwords in plain text! If your database is hacked, all user passwords are compromised. Hashing converts passwords into irreversible strings.

Plain Text Password → Hash Function → Irreversible Hash String → Store in Database
💡 Analogy: Hashing is like a meat grinder. You put in a password, you get hash. But you cannot turn hash back into password (one-way).

3️⃣ What is bcrypt?

bcrypt is a password hashing function designed to be slow and computationally expensive, making brute-force attacks extremely difficult. It automatically includes a salt to prevent rainbow table attacks.

🧂 Salt

Random data added to password before hashing — ensures same password produces different hashes

⚙️ Salt Rounds

Number of hashing iterations (2^rounds). Higher = more secure but slower

🛡️ Adaptive

Can increase salt rounds over time as computers get faster

5️⃣ Hashing a Password — Registration
const bcrypt = require('bcryptjs');
const saltRounds = 10; // 2^10 iterations = 1024 hashing rounds

// Hash password during registration
const hashPassword = async (plainPassword) => {
const salt = await bcrypt.genSalt(saltRounds);
const hashedPassword = await bcrypt.hash(plainPassword, salt);
// OR directly: const hashedPassword = await bcrypt.hash(plainPassword, saltRounds);
return hashedPassword;
};

// Example
const password = "mySecretPassword123";
const hashed = await bcrypt.hash(password, 10);
// Store 'hashed' in database

6️⃣ Comparing Passwords — Login Verification
const bcrypt = require('bcryptjs');

const comparePassword = async (plainPassword, hashedPasswordFromDB) => {
const isValid = await bcrypt.compare(plainPassword, hashedPasswordFromDB);
return isValid; // true or false
};

// Login route example
router.post('/login', async (req, res) => {
const { email, password } = req.body;
const user = await User.findOne({ email });
if (!user) return res.status(401).json({ error: 'Invalid credentials' });

const isMatch = await bcrypt.compare(password, user.password);
if (!isMatch) return res.status(401).json({ error: 'Invalid credentials' });

// Password correct — generate JWT and login

8️⃣ Complete User Model (Mongoose + bcrypt)
// models/User.js
const mongoose = require('mongoose');
const bcrypt = require('bcryptjs');

const userSchema = new mongoose.Schema({
name: { type: String, required: true },
email: { type: String, required: true, unique: true },
password: { type: String, required: true }
});

// Hash password before saving
userSchema.pre('save', async function(next) {
if (!this.isModified('password')) return next();
this.password = await bcrypt.hash(this.password, 10);
next();
});

// Method to compare passwords
userSchema.methods.comparePassword = async function(candidatePassword) {
return await bcrypt.compare(candidatePassword, this.password);
};

const User = mongoose.model('User', userSchema);
module.exports = User;
9️⃣ bcrypt vs SHA-256 — Why Not Just SHA?

❌ SHA-256 is FAST

Fast = bad for passwords. Attackers can try billions of passwords per second.

✅ bcrypt is SLOW

Designed to be slow. 1 million bcrypt hashes/second vs 100 billion SHA/second.

🧂 bcrypt has built-in salt

Same password = different hash every time. Prevents rainbow table attacks.

🔟 Password Security Best Practices
  • ✅ Always use bcrypt (or argon2) for password hashing — never MD5 or SHA-1
  • ✅ Use salt rounds = 10-12 for production
  • ✅ Require strong passwords (min 8 chars, mix of cases, numbers, symbols)
  • ✅ Implement rate limiting on login attempts (prevent brute force)
  • ✅ Never log or expose hashed passwords
  • ✅ Use HTTPS everywhere (prevent password interception)

  • ❌ Using low salt rounds (like 4) → too fast, easier to brute force

💼 Common Interview Questions

❓ Why hash passwords?

To protect user passwords if database is compromised — hashes are irreversible.

❓ Why bcrypt over SHA?

bcrypt is intentionally slow and includes salt, making brute-force attacks expensive.

❓ What are salt rounds?

Number of hashing iterations (2^rounds). Higher = more secure but slower.

❓ Can two same passwords have different hashes?

Yes! bcrypt adds a random salt, so same password produces different hashes.
