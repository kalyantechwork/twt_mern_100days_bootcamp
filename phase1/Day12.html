1️⃣ What is an Object? (Storing Related Data)

📦 Understanding Objects

🎯 Purpose: An object is a collection of key-value pairs (properties) that describe a single entity. Unlike arrays (numbered list), objects use meaningful names (keys) to store data. Think of an object as a real-world thing — a student, a car, a product — with multiple attributes.

🏠 Analogy: A student object has properties like name, age, roll number. Each property has a value. Objects are the foundation of how JavaScript handles complex data.

// Creating an object using curly braces {}
let student = {
name: "Kalyan", // key: "name", value: "Kalyan"
age: 25, // key: "age", value: 25
city: "Hyderabad",
isEnrolled: true
};

2️⃣ Accessing & Updating Object Properties

🔍 Dot Notation vs Bracket Notation

🎯 Purpose: You can access object values using dot notation (object.key) or bracket notation (object["key"]). Use dot notation when you know the property name. Use brackets when the property name is stored in a variable or contains special characters.

💡 Adding/Updating: Assign a value to a new or existing key. Use delete to remove a property.

let user = { name: "Priya", role: "Developer" };
console.log(user.name); // "Priya" (dot notation)
console.log(user["role"]); // "Developer" (bracket notation)
user.city = "Chennai"; // Add new property
delete user.role; // Remove property

3️⃣ Nested Objects (Object Inside Object)

🏠 Creating Hierarchical Data

🎯 Purpose: Objects can contain other objects as property values. This is called nesting — useful for representing complex relationships like a user with an address, a company with multiple departments, etc.

💡 Access nested properties: Use chained dot notation: user.address.city

let user = {
name: "Kalyan",
address: {
city: "Hyderabad",
pincode: 500001,
street: "MG Road"
}
};
console.log(user.address.city); // "Hyderabad"

4️⃣ Arrays of Objects (Real API Data)

🛍️ List of Products

🎯 Purpose: Most real-world data from APIs comes as an array of objects. Each object represents one item (product, user, post). This structure allows you to use array methods like map(), filter(), and forEach() to work with the data.

let products = [
{ id: 1, name: "Laptop", price: 50000 },
{ id: 2, name: "Phone", price: 20000 },
{ id: 3, name: "Tablet", price: 15000 }
];
console.log(products[0].name); // "Laptop"

5️⃣ Object.keys() & Object.values() (Introspection)

🔑 Getting All Keys or Values

🎯 Purpose: Object.keys() returns an array of all property names (keys). Object.values() returns an array of all property values. These are useful for looping through objects when you don't know the property names in advance.

let student = { name: "Kalyan", age: 25, city: "Hyderabad" };
let keys = Object.keys(student); // ["name", "age", "city"]
let values = Object.values(student); // ["Kalyan", 25, "Hyderabad"]

6️⃣ Object Methods & 'this' Keyword

📞 Functions Inside Objects

🎯 Purpose: Objects can also contain functions as properties, called methods. Inside a method, the this keyword refers to the object itself, allowing you to access other properties of the same object.

let user = {
name: "Kalyan",
greet() {
return `Hello, my name is ${this.name}!`;
}
};
user.greet(); // "Hello, my name is Kalyan!"

7️⃣ Object Destructuring (Clean Variable Extraction)

📦 Unpacking Object Properties

🎯 Purpose: Destructuring allows you to extract multiple properties from an object into variables in one line — no need for repetitive person.name, person.age. You can also rename variables and set default values.

💡 Syntax: let { property1, property2 } = object

let student = { name: "Kalyan", age: 25, city: "Hyderabad" };
let { name, age } = student; // name = "Kalyan", age = 25
let { name: studentName } = student; // rename to studentName
let { country = "India" } = student; // default value

8️⃣ Array Destructuring (Unpacking Arrays)

🎯 Destructuring Arrays

🎯 Purpose: Array destructuring works similarly to object destructuring but uses square brackets [] and relies on position (index) rather than property names. Perfect for swapping variables or extracting values from returned arrays.

let fruits = ["Apple", "Mango", "Orange"];
let [first, second] = fruits; // first = "Apple", second = "Mango"
let [a, , c] = fruits; // skip second element
let [x, ...rest] = fruits; // rest = ["Mango", "Orange"]

9️⃣ Spread Operator (...) with Objects (Copy & Merge)

✨ Copying and Combining Objects

🎯 Purpose: The spread operator ... can also be used with objects to create shallow copies or merge multiple objects. This is the modern way to update state in React and other frameworks.

💡 Important: Later properties overwrite earlier ones when merging.

let user = { name: "Kalyan", age: 25 };
let copy = { ...user }; // independent copy
let updated = { ...user, city: "Hyderabad" }; // add property
let merged = { ...user, ...{ age: 26, role: "dev" } }; // merge & override

⚠️ Common Object Mistakes for Beginners
🚫 Avoid These Errors:
  • ❌ Using dot notation with variables: user[variable] not user.variable (brackets needed when key is dynamic)
  • ❌ Accessing undefined nested properties: user.address.city when address is undefined → use optional chaining: user?.address?.city
  • ❌ Confusing arrays of objects vs objects with array properties → different structures for different use cases
  • ❌ Destructuring undefined variables: provide default values: let { name = "Guest" } = user
  • ❌ Forgetting commas between properties: objects need commas between key-value pairs
  • ❌ Modifying objects when you meant to copy: use spread { ...obj } to create a copy
