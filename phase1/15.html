1️⃣ What is an API?
📚 API (Application Programming Interface) allows applications to communicate. Like a waiter in a restaurant: Customer (Frontend) → Waiter (API) → Kitchen (Backend). APIs power everything from weather apps to payment systems.
Frontend → API Request → Backend Server → Database → API Response → Frontend
2️⃣ JSON (JavaScript Object Notation) – Data Transfer Format

📌 Purpose & Usage

JSON is the standard format for sending data between client and server. It looks like JavaScript objects but with double-quoted keys. JSON.stringify() converts JS objects to JSON strings. JSON.parse() converts JSON strings back to JS objects.

Where it's used: Every API request/response, configuration files, local storage data, real-time communication.

📦 JSON Stringify & Parse

let user = { name: "Kalyan", age: 25 };
let jsonStr = JSON.stringify(user); // '{"name":"Kalyan","age":25}'
let parsed = JSON.parse(jsonStr); // { name: "Kalyan", age: 25 }

3️⃣ Fetch API – Make HTTP Requests

📌 Purpose & Usage

Fetch API is modern JavaScript's built-in way to make HTTP requests. It returns a Promise, allowing async/await or .then() syntax. Used to GET data from servers, POST new data, PUT updates, and DELETE records.

Where it's used: Loading user data, submitting forms, fetching products, real-time updates, integrating third-party services (payment, maps, weather).

🌐 Basic Fetch GET Request

fetch("https://jsonplaceholder.typicode.com/users")
.then(response => response.json())
.then(data => console.log(data))
.catch(error => console.log(error));

    5️⃣ HTTP Methods (CRUD Operations)

    📌 Purpose & Usage

    HTTP methods define the action to perform on a resource. GET retrieves, POST creates, PUT updates, DELETE removes. Understanding these is essential for full-stack development.

    MethodPurposeExample Use Case
    GETFetch dataView products, user profiles
    POSTCreate new dataRegister user, add comment
    PUTUpdate existing dataEdit profile, update order
    DELETERemove dataDelete account, remove item
    6️⃣ POST Request – Send Data to Server

    📌 Purpose & Usage

    POST requests send data to the server to create new resources. Common for form submissions, user registrations, adding new items. Requires setting method: "POST" and body with JSON data.

    Where it's used: Signup forms, blog post creation, adding products to cart, sending messages.

    📨 POST Request Example

    fetch("https://jsonplaceholder.typicode.com/posts", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ title: "My Post", body: "Hello World" })
    });

    ⚠️ Common Beginner Mistakes
    • ❌ Forgetting response.json() → fetch returns Response object, need to parse JSON. Use await response.json().
    • ❌ Using fetch without catch → unhandled promise rejections. Always use try...catch or .catch().
    • ❌ Not checking response.ok → 404/500 errors don't reject fetch. Check if(!response.ok) throw new Error().
    • ❌ Forgetting to stringify body in POST requests → body must be a JSON string.
    • ❌ Missing "Content-Type": "application/json" header → server may not parse request correctly.
