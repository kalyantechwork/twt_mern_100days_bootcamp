1️⃣ Higher-Order Functions (Functions that accept/return functions)

📌 Purpose & Usage

Higher-Order Functions either take a function as an argument or return a function. They enable functional programming patterns like map, filter, reduce, and callbacks.

Where it's used: Array methods (map, filter, reduce), event handlers, setTimeout, Promises, custom hooks in React.

⚡ Higher-Order Function Example

function greet(name) { return `Hello ${name}`; }
function processUser(name, callback) {
return callback(name);
}
processUser("Kalyan", greet); // "Hello Kalyan"

2️⃣ Closures (Function remembers outer variables)

📌 Purpose & Usage

Closures occur when a function "remembers" variables from its outer scope even after the outer function has finished executing. This enables data privacy and stateful functions.

Where it's used: React hooks (useState, useEffect), module patterns, event handlers, debounce/throttle functions, currying.

🔒 Closure Counter Example

function createCounter() {
let count = 0;
return function() {
count++;
return count;
};
}
const counter = createCounter();
counter(); // 1, counter(); // 2

3️⃣ Lexical Scope (Inner functions can access outer variables)

📌 Purpose & Usage

Lexical Scope means that inner functions can access variables defined in their outer scope. This is the foundation of closures.

let global = "Global";
function outer() {
let outerVar = "Outer";
function inner() {
console.log(outerVar); // ✅ accessible
}
inner();
}
✅ inner() can access outerVar from outer() function.
✅ inner() can also access global variables.
✅ This creates the "scope chain" — inner → outer → global.
4️⃣ Event Bubbling (Child → Parent → Document)

📌 Purpose & Usage

Event Bubbling is when an event starts from the target element and bubbles up to the root. Useful for event delegation, but can cause unexpected behavior if not managed.

Where it's used: Modal clicks, dropdown menus, event delegation for dynamic lists, preventing unwanted parent triggers.

5️⃣ Debouncing (Wait before execution — Search optimization)

📌 Purpose & Usage

Debouncing delays function execution until after a pause in events. Prevents excessive API calls during typing.

Where it's used: Search bars (Google search), autocomplete, form validation, resize event handlers, save drafts.

🔍 Debounced Search Input

function debounce(func, delay) {
let timer;
return function() {
clearTimeout(timer);
timer = setTimeout(func, delay);
};
}

6️⃣ Throttling (Limit execution frequency — Scroll/Resize)

📌 Purpose & Usage

Throttling ensures a function is called at most once in a specified time interval. Prevents excessive execution during rapid events.

Where it's used: Scroll event handlers, window resize, button spamming prevention, game controls, infinite scroll.

📊 Debouncing vs Throttling — When to Use

🛑 Debouncing

Waits until user stops typing — saves API calls. Best for search, autocomplete.

⏱️ Throttling

Executes at regular intervals — limits frequency. Best for scroll, resize, game controls.

8️⃣ call(), apply(), bind() (Control `this`)

📌 Purpose & Usage

These methods explicitly set the this value for a function. call/apply invoke immediately, bind returns a new function.

function greet() { return `Hello ${this.name}`; }
let user = { name: "Kalyan" };
greet.call(user); // "Hello Kalyan"
let bound = greet.bind(user);
bound(); // "Hello Kalyan"
✅ call() — invokes with given this and arguments
✅ apply() — similar but takes array of arguments
✅ bind() — returns a new function with this bound permanently
💡 Used in React for event handlers, currying, function borrowing.
9️⃣ `this` Keyword & Arrow Functions

📌 Purpose & Usage

Arrow functions don't have their own this — they inherit from the surrounding scope. This makes them perfect for callbacks but unsuitable for object methods that need their own this.

let obj = {
name: "Kalyan",
regular: function() { console.log(this.name); }, // ✅ "Kalyan"
arrow: () => console.log(this.name) // ❌ undefined
};
✅ Regular functions: 'this' refers to the calling object.
❌ Arrow functions: 'this' refers to parent scope (lexical this).
💡 Use regular functions for object methods, arrow functions for callbacks.

⚠️ Common Mistakes
  • ❌ Forgetting clearTimeout in debounce → multiple timers run. Always clear previous timeout.
  • ❌ Assuming arrow functions bind own this → they don't. Use regular functions for object methods.
  • ❌ Not stopping propagation when needed (modal closes unexpectedly) → use e.stopPropagation().
  • ❌ Overusing throttling where debounce is better (search needs debounce, scroll needs throttle).
  • ❌ Creating closures in loops without block scope → use let or IIFE.
