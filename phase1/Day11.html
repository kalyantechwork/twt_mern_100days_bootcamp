1️⃣ What is an Array? (Storing Multiple Values)

📦 Understanding Arrays

🎯 Purpose: An array is a special variable that can hold multiple values at once. Instead of creating separate variables for each item (like fruit1, fruit2, fruit3), you use one array: let fruits = ["Apple", "Mango", "Orange"].

💡 Key points: Arrays are ordered, each item has an index (position) starting from 0. The first item is at index 0, second at index 1, etc.

🏠 Analogy: An array is like a train with multiple compartments. Each compartment can hold different items (numbers, text, even other arrays).

// Creating arrays
let fruits = ["Apple", "Mango", "Orange"];
let numbers = [10, 20, 30, 40];
let mixed = ["Kalyan", 25, true];

// Accessing elements (index starts at 0)
console.log(fruits[0]); // "Apple"
console.log(fruits.length); // 3 (number of items)

2️⃣ Accessing & Updating Elements by Index

🔢 Working with Index Positions

🎯 Purpose: Each item in an array has a numeric index (position). The first item is index 0, second is index 1, and so on. You can read or change any item using its index.

💡 Important: Trying to access an index that doesn't exist returns undefined (not an error).

let colors = ["Red", "Green", "Blue"];
console.log(colors[1]); // "Green" (second item)
colors[2] = "Yellow"; // Change third item
console.log(colors); // ["Red", "Green", "Yellow"]

3️⃣ Basic Array Methods (Adding & Removing Items)

➕➖ Modify Arrays

🎯 Purpose: These methods help you add or remove items from arrays:

  • push() - adds item to the END
  • pop() - removes item from the END
  • unshift() - adds item to the BEGINNING
  • shift() - removes item from the BEGINNING
let arr = [1, 2];
arr.push(3); // [1, 2, 3] (add to end)
arr.pop(); // [1, 2] (remove from end)
arr.unshift(0); // [0, 1, 2] (add to start)
arr.shift(); // [1, 2] (remove from start)

4️⃣ forEach() — Looping Through Arrays

🔄 Iterating Without a For Loop

🎯 Purpose: forEach() executes a function once for each array element. It's a cleaner, more readable way to loop through arrays compared to traditional for loops.

💡 Syntax: array.forEach((element, index) => { // do something })

let fruits = ["Apple", "Mango", "Orange"];
fruits.forEach((fruit, index) => {
console.log(`${index}: ${fruit}`);
});
// Output: 0: Apple, 1: Mango, 2: Orange

5️⃣ map() — Transform Every Element

📐 Creating a New Array from an Existing One

🎯 Purpose: map() creates a brand new array by applying a function to each element of the original array. The original array remains unchanged.

💡 Use case: Converting prices to include tax, squaring numbers, extracting specific properties from objects.

let numbers = [1, 2, 3, 4];
let doubled = numbers.map(n => n * 2);
// doubled = [2, 4, 6, 8]
// original numbers still [1, 2, 3, 4]

6️⃣ filter() — Select Matching Elements

🎯 Creating a Subset Based on Conditions

🎯 Purpose: filter() creates a new array with only the elements that pass a test (return true from the callback function). Elements that return false are excluded.

💡 Use case: Finding all products under a certain price, getting only active users, filtering search results.

let numbers = [10, 20, 30, 40, 50];
let greaterThan30 = numbers.filter(n => n > 30);
// greaterThan30 = [40, 50]

7️⃣ reduce() — Accumulate into Single Value

💰 Calculating Sums, Averages, Totals

🎯 Purpose: reduce() reduces an array to a single value by applying a function that accumulates results. It's perfect for calculating totals, averages, or combining data.

💡 Parameters: reduce((accumulator, currentValue) => accumulator + currentValue, initialValue)

let cart = [100, 200, 50];
let total = cart.reduce((sum, price) => sum + price, 0);
// total = 350

8️⃣ find() and includes() — Searching Arrays

🔎 Finding Elements and Checking Existence

🎯 Purpose: find() returns the first element that satisfies a condition (unlike filter which returns all). includes() returns true/false if an element exists.

💡 Difference: find() → returns the element, includes() → returns boolean.

let nums = [5, 12, 8, 130];
let firstBig = nums.find(n => n > 10); // returns 12
let hasEight = nums.includes(8); // returns true

9️⃣ sort() and reverse() — Ordering Arrays

📋 Sorting and Reversing Order

🎯 Purpose: sort() arranges array elements in order (alphabetical for strings, numeric with compare function). reverse() reverses the order. Important: Both methods modify the original array.

let letters = ["c", "a", "b"];
letters.sort(); // ["a", "b", "c"]
letters.reverse(); // ["c", "b", "a"]

let numbers = [40, 5, 100];
numbers.sort((a,b) => a - b); // [5, 40, 100] (numeric sort)

🔟 Spread Operator (...) — Copy and Combine Arrays

✨ The Three Dots Magic

🎯 Purpose: The spread operator ... "spreads" array elements into individual items. It's perfect for copying arrays (without affecting original) and combining multiple arrays.

💡 Why copy? Arrays are reference types. Changing a copied array shouldn't change the original. Spread creates a shallow copy.

let arr1 = [1, 2];
let arr2 = [3, 4];
let combined = [...arr1, ...arr2]; // [1, 2, 3, 4]
let copy = [...arr1]; // independent copy

📦 Array Destructuring (Unpacking Values)

🎁 Extracting Array Values into Variables

🎯 Purpose: Destructuring allows you to unpack array values into distinct variables in one line. It's cleaner than accessing each element with index.

💡 Syntax: let [first, second, third] = myArray

let fruits = ["Apple", "Mango", "Orange"];
let [first, second] = fruits;
console.log(first); // "Apple"
console.log(second); // "Mango"

⚠️ Common Array Mistakes for Beginners
🚫 Avoid These Errors:
  • ❌ Forgetting map() needs return: arr.map(n => n * 2) works; arr.map(n => { n * 2 }) returns undefined (curly braces need explicit return)
  • ❌ Confusing filter() and find(): filter() returns ALL matching elements (array), find() returns FIRST matching element (single)
  • ❌ Accessing index out of bounds: arr[10] when length is 3 returns undefined (not error)
  • ❌ sort() on numbers: default sort is alphabetical: [1, 10, 2] becomes [1, 10, 2]. Use sort((a,b) => a - b) for numbers
  • ❌ Mutating original array accidentally: Use spread [...arr] to create a copy before modifying
  • ❌ reduce() without initial value: Can cause error on empty arrays. Always provide initial value: reduce((sum, n) => sum + n, 0)
