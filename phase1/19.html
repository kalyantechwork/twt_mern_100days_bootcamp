1️⃣ Why Browser Storage? (Problem & Solution)

📌 Purpose & Usage

Normally, JavaScript variables disappear after page refresh. Browser Storage (Local Storage & Session Storage) keeps data even after refresh — perfect for dark mode preferences, shopping cart items, user settings, and form data persistence.

Where it's used: Dark mode themes, saved usernames, shopping carts, notes apps, form auto-save, game scores, and user preferences across millions of websites.

Page Refresh → Without Storage: Data lost ❌
Page Refresh → With Local Storage: Data persists ✅
2️⃣ Local Storage (Permanent Storage)

📌 What is Local Storage?

localStorage stores data with no expiration date. Data persists across browser sessions, tabs, and even after computer restart. ~5MB limit per domain.

// Store data
localStorage.setItem("username", "Kalyan");

// Get data
const name = localStorage.getItem("username");

// Remove one item
localStorage.removeItem("username");

// Clear all data for this domain
localStorage.clear();
📋 localStorage API Methods:
✅ setItem(key, value) — stores data
✅ getItem(key) — retrieves data (returns null if not found)
✅ removeItem(key) — deletes specific key
✅ clear() — removes all data for this domain
⚠️ Data persists until manually cleared!

5️⃣ Session Storage (Tab-Specific, Cleared on Tab Close)

📌 What is Session Storage?

sessionStorage stores data for the duration of the page session. Data survives page reload but is cleared when the tab or browser is closed. Each tab has its own separate session storage.

sessionStorage.setItem("tempData", "value");
const temp = sessionStorage.getItem("tempData");
sessionStorage.removeItem("tempData");

7️⃣ Storing Objects (JSON.stringify & JSON.parse)

📌 Important: Storage Only Accepts Strings

localStorage and sessionStorage only store strings. To store objects/arrays, use JSON.stringify() before saving and JSON.parse() after retrieving.

const user = { name: "Kalyan", age: 25, city: "Hyderabad" };
// Store object
localStorage.setItem("user", JSON.stringify(user));
// Retrieve object
const storedUser = JSON.parse(localStorage.getItem("user"));
console.log(storedUser.name); // "Kalyan"

⚠️ Security & Common Mistakes
  • ❌ Forgetting JSON.stringify() → stores "[object Object]" instead of data.
  • ❌ Forgetting JSON.parse() → returns string instead of object.
  • ❌ Storing sensitive data (passwords, tokens, payment info) → NOT SECURE! Vulnerable to XSS attacks.
  • ❌ Assuming storage works in private/incognito mode (it does, but cleared after closing).
  • ❌ Not handling null when item doesn't exist → always check if(storage.getItem("key")).
🔐 Security: Never store JWT tokens, passwords, or payment info in local/session storage — it's vulnerable to XSS attacks. Use HTTP-only cookies for authentication.
💼 Common Interview Questions

❓ Q1: Difference?

Local: persists until cleared. Session: cleared when tab closes.

❓ Q2: Store objects?

Not directly → use JSON.stringify/parse.

❓ Q3: Storage limit?

~5MB per domain.

❓ Q4: Can other domains access?

No — same-origin policy.
