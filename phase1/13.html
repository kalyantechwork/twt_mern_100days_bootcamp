⚡ ES6+ Modern JavaScript

1️⃣ What is ES6 (ECMAScript 2015)?
📚 ES6 (ECMAScript 2015) is a major update to JavaScript that introduced modern features making code cleaner, shorter, and more readable. Frameworks like React, Node.js, Express heavily use ES6+ syntax.
Old JavaScript (var, function) → Modern ES6 (let/const, arrow functions, spread) → Professional Code
2️⃣ let & const (Block-scoped variables)

📌 Purpose & Usage

let and const replace the old var. They provide block-level scope (inside { }), preventing accidental variable leaks. const is for values that never change, let for values that need reassignment. Used everywhere in modern JavaScript — React state, loops, conditionals.

Where it's used: All JavaScript projects (React, Node.js, Vue, Angular) — always use let/const instead of var.

📦 let & const Examples

let score = 100; // can be changed later
score = 200; // ✅ allowed

const PI = 3.14159; // cannot be changed
// PI = 3.14; // ❌ Error!

3️⃣ Template Literals (Backticks ` `)

📌 Purpose & Usage

Template literals allow embedding variables and expressions directly inside strings using ${} syntax. They support multi-line strings without concatenation.

Where it's used: Dynamic HTML generation, console logs, API URL building, error messages, React JSX, email templates.

📝 Template Literal Examples

let name = "Kalyan";
let age = 25;
// Old way: "Hello " + name + ", age " + age
// ES6 way:
let message = `Hello ${name}, age ${age}`;
let multiLine = `Line 1
Line 2`;

4️⃣ Arrow Functions (Shorter Syntax)

📌 Purpose & Usage

Arrow functions provide a shorter function syntax using =>. They don't bind their own this, making them perfect for callbacks, array methods (map, filter, reduce), and React components.

Where it's used: Array methods (map, filter), event handlers, setTimeout/setInterval, React function components.

🏹 Arrow Function Examples

// Traditional function
function add(a,b) { return a + b; }

// Arrow function (implicit return)
const addArrow = (a,b) => a + b;

// Arrow with multiple lines
const multiply = (a,b) => {
return a * b;
};

5️⃣ Spread Operator (...) (Expand / Copy)

📌 Purpose & Usage

Spread operator (...) expands an array or object into individual elements. Perfect for copying arrays/objects without mutation, combining data, and passing arguments.

Where it's used: React state updates (immutability), merging objects, converting NodeList to array, function arguments.

✨ Spread Operator Examples

let arr1 = [1, 2];
let arr2 = [...arr1, 3, 4]; // [1,2,3,4]

let user = { name: "Kalyan" };
let updated = { ...user, city: "Hyderabad" };
// { name: "Kalyan", city: "Hyderabad" }

6️⃣ Destructuring (Extract values easily)

📌 Purpose & Usage

Destructuring unpacks values from arrays or properties from objects into distinct variables. Reduces repetitive code and makes data extraction clean.

Where it's used: Extracting props in React, API response handling, function parameters, importing modules, useState hook.

📦 Object & Array Destructuring

// Object destructuring
let user = { name: "Kalyan", age: 25 };
let { name, age } = user;

// Array destructuring
let colors = ["red", "blue"];
let [first, second] = colors;

7️⃣ Rest Operator (Collect remaining arguments)

📌 Purpose & Usage

Rest operator (...) collects remaining arguments into an array. Opposite of spread — it "gathers" values.

Where it's used: Functions with variable number of arguments, destructuring to collect remaining items, React hooks dependencies.

📥 Rest Operator Example

function sum(...numbers) {
return numbers.reduce((acc, n) => acc + n, 0);
}
sum(1,2,3,4); // 10

8️⃣ Optional Chaining (?.) — Safe nested access

📌 Purpose & Usage

Optional Chaining (?.) allows reading property values deep inside an object without checking if each reference exists. Returns undefined instead of throwing error.

Where it's used: API responses, user data with optional fields, configuration objects, preventing "Cannot read property of undefined" errors.

🛡️ Optional Chaining Example

let user = { profile: { name: "Kalyan" } };
console.log(user.profile?.name); // "Kalyan"
console.log(user.address?.city); // undefined (no error)

9️⃣ Default Parameters (Fallback values)

📌 Purpose & Usage

Default parameters allow functions to have preset values when no argument is passed or undefined is passed.

Where it's used: Function arguments that are optional, API wrapper functions, configuration objects, React component props defaults.

⚙️ Default Parameters Example

function greet(name = "Guest") {
return `Hello ${name}`;
}
greet(); // "Hello Guest"
greet("Kalyan"); // "Hello Kalyan"

⚠️ Common Beginner Mistakes (ES6)
  • ❌ Using const for variable that needs reassignment → use let.
  • ❌ Forgetting backticks in template literals: "Hello ${name}" → must be `Hello ${name}`.
  • ❌ Confusing spread (...) and rest: spread expands, rest collects.
  • ❌ Using arrow functions when this binding is required (arrow functions don't bind their own this).
  • ❌ Destructuring null/undefined → provide default values: let { name = "Guest" } = user || {};
