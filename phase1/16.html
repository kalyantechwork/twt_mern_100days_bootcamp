1️⃣ What is Git? (Version Control System)

📌 Purpose & Usage

Git is a Version Control System (VCS) that tracks changes in your code over time. Like "save points" in video games — you can always go back to a previous version. Git enables collaboration, experiment safely with branches, and maintain complete project history.

Where it's used: Every professional software project — from small personal projects to large teams at Google, Microsoft, and startups.

📦 What Git Does

git init # Start tracking a folder
git add index.html # Stage changes
git commit -m "message" # Save snapshot
git log # View history
📋 Git Core Concept:
✅ Git tracks every change to your files.
✅ Each commit is like a photograph of your code at that moment.
✅ You can jump back to any commit using git checkout.
✅ Teams can work simultaneously without overwriting each other's work.
2️⃣ Git vs GitHub (Difference)

📌 Purpose & Usage

Git is the tool that runs on your computer. GitHub is a cloud platform that hosts Git repositories online, enabling collaboration, code review, and portfolio hosting.

Git (Local)GitHub (Cloud)
Runs on your computerWebsite: github.com
Tracks changes locallyStores repositories remotely
git init, add, commitgit push, pull, clone
Local Git Repository → git push → GitHub Cloud Repository
3️⃣ Install Git & Basic Configuration

📌 Purpose & Usage

Before using Git, you must install it from git-scm.com and configure your identity. Every commit will be linked to your name and email — essential for team collaboration.

# Download from git-scm.com and install
git --version # Verify installation

git config --global user.name "Your Name"
git config --global user.email "your@email.com"

git config --list # View all settings
📋 Setup Commands:
✅ git config --global user.name "Kalyan"
✅ git config --global user.email "kalyan@example.com"
✅ These settings are stored in ~/.gitconfig
⚠️ Required before making commits!
4️⃣ Creating a Repository (git init)

📌 Purpose & Usage

git init creates a hidden .git folder inside your project, turning it into a Git repository. This folder stores all version history and metadata.

mkdir my-project
cd my-project
git init # Initializes empty Git repository
$ git init
Initialized empty Git repository in /my-project/.git/

✅ A hidden .git folder is created — contains all version history.
✅ Your project is now tracked by Git!
5️⃣ Git Workflow: Working → Staging → Commit

📌 Purpose & Usage

Files move through three stages: Working Directory (where you edit), Staging Area (git add), Commit (git commit saves to history). This gives you control over which changes to include.

git status # See which files changed
git add index.html # Stage specific file
git add . # Stage all changes
git commit -m "Add homepage" # Save snapshot
Working Directory (modified files)
↓ git add
Staging Area (files to commit)
↓ git commit
Commit → Saved in History
6️⃣ Check Status & History
git status # See modified/staged files
git log # Full commit history
git log --oneline # Compact history (one line per commit)
$ git status
On branch main
Changes not staged for commit:
  modified: index.html

$ git log --oneline
a1b2c3d Add navbar feature
d4e5f6g Initial commit
✅ git status shows what changed, git log shows commit history.
7️⃣ Connect to GitHub (Remote) & Push

📌 Purpose & Usage

git remote add links your local repo to a GitHub repository. git push uploads your commits to GitHub, making them available online.

# Create repo on GitHub first
git remote add origin https://github.com/username/repo.git
git branch -M main
git push -u origin main # First push
$ git remote add origin https://github.com/kalyan/my-app.git
$ git push -u origin main
✅ origin = nickname for your GitHub repository
✅ After this, just "git push" works!
8️⃣ Pull & Clone (Download from GitHub)
git pull origin main # Download latest changes
git clone https://github.com/username/repo.git # Full copy
✅ git pull = fetch + merge — gets latest changes from GitHub
✅ git clone creates a local copy of any GitHub repository
$ git clone https://github.com/facebook/react.git
→ Downloads React's entire codebase!
🌿 Branches (Parallel Development)

📌 Purpose & Usage

Branches allow you to work on features independently without affecting the main codebase. This is how teams collaborate — each developer works on their own branch, then merges.

git branch feature-login # Create branch
git checkout feature-login # Switch to branch
git checkout -b new-feature # Create & switch (shortcut)
git merge feature-login # Merge into current branch
main (stable) ←─── feature-login
                ↓ git merge
main now includes feature-login
✅ Branches enable safe experimentation
✅ git branch -a shows all branches
✅ git merge combines changes
📁 .gitignore (Exclude Files from Git)

📌 Purpose & Usage

.gitignore tells Git which files/folders to ignore. Prevents committing temporary files, dependencies, secrets, and large binaries.

# .gitignore file example
node_modules/
.env
.DS_Store
dist/
*.log
✅ node_modules/ → never commit (can be reinstalled via npm install)
✅ .env → contains API keys/secrets (security risk)
✅ .DS_Store → macOS system file
⚠️ Always add .gitignore BEFORE committing node_modules!
📋 Complete Git Workflow Example
# 1. Create project
mkdir mern-app && cd mern-app
git init

# 2. Create file and commit
echo "# My MERN App" > README.md
git add .
git commit -m "Initial commit"

# 3. Connect to GitHub & push
git remote add origin https://github.com/username/mern-app.git
git branch -M main
git push -u origin main
✅ This is the exact workflow used by developers daily!
1. git init → start tracking
2. git add + commit → save changes
3. git remote add → link to GitHub
4. git push → upload to cloud

🔁 Repeat: edit → add → commit → push
⚠️ Common Git Mistakes & Solutions
  • ❌ Forgetting git add . before commit → files not saved. Use git status to check.
  • ❌ Using wrong remote URL → check with git remote -v, fix with git remote set-url origin new-url.
  • ❌ Committing node_modules → huge repo size. Add to .gitignore immediately.
  • ❌ Writing vague commit messages → "fixed stuff" is bad. Use "Add login validation" or "Fix navbar responsive issue".
  • ❌ Merge conflicts fear → practice! Conflicts are normal. Read conflict markers and resolve carefully.
