⏳ Asynchronous JavaScript
setTimeout · setInterval · Callbacks · Promises · Async/Await · Event Loop

1️⃣ What is Asynchronous JavaScript?
📚 Asynchronous JavaScript allows tasks to run without blocking other operations. Like ordering food online — you continue working while food is being prepared. This is essential for API calls, timers, and user interactions.
Task Starts → JavaScript Continues Other Tasks → Task Completes Later
2️⃣ Synchronous vs Asynchronous Code

📌 Purpose & Usage

Synchronous code executes line by line — each operation waits for the previous to finish. Asynchronous code doesn't block — it starts a task and continues executing other code, handling the result later when ready.

Where it's used: API calls, file reading, database queries, timers, event handlers — any operation that takes time.

🔄 Async vs Sync Demo

console.log("Start");
setTimeout(() => console.log("Async Task"), 2000);
console.log("End");
// Output: Start → End → (after 2s) Async Task

3️⃣ setTimeout() — Run Code After Delay

📌 Purpose & Usage

setTimeout() executes a function once after a specified delay (in milliseconds). It returns a timer ID that can be used with clearTimeout() to cancel.

Where it's used: Delayed notifications, splash screens, loading states, debouncing search inputs, auto-save features.

⏰ setTimeout Example

setTimeout(() => {
console.log("Runs after 2 seconds");
}, 2000);

4️⃣ setInterval() & clearInterval() — Repeated Execution

📌 Purpose & Usage

setInterval() repeatedly executes a function at specified intervals. clearInterval() stops it using the returned timer ID.

Where it's used: Countdown timers, stopwatches, real-time clocks, live data refresh, animations, carousels.

⏲️ setInterval Counter

let count = 0;
let interval = setInterval(() => {
count++;
console.log(count);
if(count === 5) clearInterval(interval);
}, 1000);

5️⃣ Callback Functions (Function as Argument)

📌 Purpose & Usage

Callback functions are functions passed as arguments to other functions, to be executed later. They are the foundation of async programming in JavaScript.

Where it's used: Event listeners, setTimeout/setInterval, array methods (map, filter, forEach), API calls (older style).

📞 Callback Example

function greet(name, callback) {
console.log(`Hello ${name}`);
callback();
}
greet("Kalyan", () => console.log("Callback executed!"));

6️⃣ Callback Hell (The Problem)

📌 Purpose & Usage

Callback Hell occurs when multiple nested callbacks create pyramid-like code that's hard to read, debug, and maintain. This led to Promises and Async/Await.

Where it's seen: Legacy code, complex async workflows with many dependent steps.

🔗 Callback Hell Example

step1(() => {
step2(() => {
step3(() => {
step4(() => {
// Hard to read! });
});
});
});
⚠️ Callback Hell makes code difficult to maintain. Solution: Promises and Async/Await flatten the structure and improve readability.
7️⃣ Promises (Future Result)

📌 Purpose & Usage

Promises represent a future value that will be available at some point. They have three states: Pending, Resolved (fulfilled), Rejected. Chain with .then() and .catch().

Where it's used: Modern JavaScript APIs, Fetch API, database operations, file reading, any async operation with success/failure states.

🤝 Promise Example

let promise = new Promise((resolve, reject) => {
let success = true;
if(success) resolve("Data received");
else reject("Error");
});
promise.then(data => console.log(data)).catch(err => console.log(err));

8️⃣ Async/Await (Modern Async Code)

📌 Purpose & Usage

async declares a function that returns a Promise. await pauses execution until the Promise resolves, making async code look synchronous.

Where it's used: Modern React apps, Node.js backend, API calls, database queries — preferred over raw Promises for readability.

🚀 Async/Await Example

async function fetchData() {
let result = await Promise.resolve("Data loaded");
console.log(result);
}
fetchData();

9️⃣ Error Handling with try...catch

📌 Purpose & Usage

try...catch blocks handle errors in async functions, preventing application crashes and providing graceful failure handling.

Where it's used: All async operations that might fail — network requests, database operations, file system access.

🛡️ try...catch Example

async function safeFetch() {
try {
let data = await riskyOperation();
console.log(data);
} catch(error) {
console.log("Error handled:", error);
}
}
✅ try...catch catches errors in async/await without crashing the application. Use it for all API calls and async operations.
🔄 Event Loop (How Async Works in JS)

📌 Purpose & Usage

Event Loop is what allows JavaScript (single-threaded) to handle async operations. It continuously checks the call stack and callback queue.

Why it matters: Understanding the event loop helps debug timing issues and write efficient async code.

Call Stack → Web APIs (setTimeout, DOM events) → Callback Queue → Event Loop → Call Stack
💡 Simple explanation: setTimeout, promises, clicks go to the queue. Event Loop moves them to call stack when it's empty.

⚠️ Common Beginner Mistakes
  • ❌ Forgetting clearInterval → intervals run forever (memory leak). Always clean up.
  • ❌ Using await without async function → SyntaxError. Every await must be inside an async function.
  • ❌ Ignoring promise rejections (no .catch or try/catch) → unhandled errors.
  • ❌ Assuming setTimeout runs exactly after delay (it can be delayed by call stack).
  • ❌ Callback hell → switch to Promises or async/await for readability.
