1️⃣ What is State? (Dynamic Data)

📌 Purpose & Usage

State is data that changes over time inside a React component. Unlike props (which are read-only from parent), state is managed internally and can be updated. When state changes, React automatically re-renders the component to reflect the new UI.

Where it's used: Every interactive React application — counters, forms, toggles, shopping carts, user authentication, data fetching, and anything that changes over time.

User Action → State Updates → Component Re-renders → UI Updates Automatically
2️⃣ useState Hook — The Syntax
import { useState } from 'react';

function Counter() {
// Array destructuring: [currentState, setterFunction]
const [count, setCount] = useState(0);

return (
<div>
<p>Count: {count}</p>
<button onClick={() => setCount(count + 1)}>Increase</button>
</div>
);
}
useState(initialValue) → returns [state, setStateFunction]
✅ useState is a Hook — special function that "hooks into" React features
✅ Returns an array with two items: current state and a function to update it
✅ Array destructuring: const [count, setCount] = useState(0)
💡 'count' is the current value, 'setCount' updates it and triggers re-render

4️⃣ Updating State — setState Function
const [count, setCount] = useState(0);

// Direct new value
setCount(10); // count becomes 10

// Using previous value (safer for async updates)
setCount(prevCount => prevCount + 1);

// With objects
const [user, setUser] = useState({ name: "Kalyan", age: 25 });
setUser({ ...user, age: 26 }); // Spread operator to keep other properties
✅ setCount(10) — directly sets new value
✅ setCount(prev => prev + 1) — functional update (safer for async)
✅ For objects: spread (...) to preserve existing properties
✅ NEVER mutate state directly: count++ ❌ use setCount(count + 1) ✅
5️⃣ Multiple State Variables
function UserForm() {
const [name, setName] = useState("");
const [email, setEmail] = useState("");
const [isSubscribed, setIsSubscribed] = useState(false);

return (<div>...</div>);
}
✅ You can use useState multiple times in one component!
✅ Each state variable is independent
✅ Good practice: separate state for separate concerns
💡 Example: name, email, and isSubscribed are separate state variables
6️⃣ State with Objects (Remember to Spread!)
const [form, setForm] = useState({ username: "", password: "" });

// ✅ Correct — preserves other fields
const handleChange = (e) => {
setForm({ ...form, [e.target.name]: e.target.value });
};

// ❌ Wrong — overwrites entire object
setForm({ username: "newName" }); // password is lost!
🔑 When updating object state, ALWAYS spread existing properties!
✅ Correct: setForm({ ...form, name: "New" })
❌ Wrong: setForm({ name: "New" }) — deletes other properties
💡 This is one of the most common React mistakes!
7️⃣ State with Arrays (Adding/Removing Items)
const [items, setItems] = useState(["Apple", "Mango"]);

// Add item
const addItem = () => {
setItems([...items, "Orange"]);
};

// Remove item by index
const removeItem = (index) => {
setItems(items.filter((_, i) => i !== index));
};

// Update item
const updateItem = (index, newValue) => {
const updated = [...items];
updated[index] = newValue;
setItems(updated);
};
✅ Add: setItems([...items, newItem]) — spread then add
✅ Remove: setItems(items.filter(item => item !== target))
✅ Update: create new array with [...items], modify, then set
💡 Never mutate arrays directly — always create new arrays!

⚠️ Common Beginner Mistakes
  • ❌ Directly mutating state: state.count++ ❌ → always use setter function: setCount(count + 1) ✅
  • ❌ Forgetting to spread objects when updating: setUser({ age: 26 }) ❌ loses other properties → use setUser({ ...user, age: 26 }) ✅
  • ❌ Calling setState in a loop without batching → React handles this, but be mindful of performance
  • ❌ Using state value immediately after setState (setState is async) → use functional update or useEffect
💼 Common Interview Questions

❓ What is state in React?

Data that changes over time inside a component. Triggers re-render when updated.

❓ What is the useState hook?

A React hook that lets you add state to functional components. Returns [state, setState].

❓ Difference between state and props?

Props are passed from parent (read-only). State is internal (mutable).

❓ Why use spread operator with object state?

To preserve existing properties when updating only one field.
