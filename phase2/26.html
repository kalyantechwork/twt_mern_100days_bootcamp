1️⃣ What is JSX? (JavaScript + XML)

📌 Purpose & Usage

JSX (JavaScript XML) is a syntax extension that allows you to write HTML-like code directly inside JavaScript. It makes React code readable, intuitive, and easy to write. JSX gets compiled by Babel into React.createElement() calls.

Where it's used: Every React component — JSX is the standard way to define UI in React. It's what makes React so developer-friendly.

// Without JSX (plain JavaScript)
const element = React.createElement('h1', null, 'Hello World');

// With JSX (much cleaner!)
const element = <h1>Hello World</h1>;
JavaScript + HTML-like syntax → JSX → Babel compiles to React.createElement → Browser understands
✅ JSX is NOT HTML — it's JavaScript with XML-like syntax!
✅ Babel transforms JSX into regular JavaScript that browsers can understand.
✅ JSX makes React code more readable and maintainable.

5️⃣ Embedding JavaScript Expressions ({ })
function Greeting() {
const userName = "Kalyan";
const age = 25;
const isLoggedIn = true;

return (
<div>
<h1>Hello, {userName}!</h1>
<p>Age: {age}</p>

</div>
);
}

✅ Anything inside { } is evaluated as JavaScript — variables, operators, ternary, function calls!
6️⃣ Fragments (<> ) — Returning Multiple Elements
// ❌ Wrong: multiple adjacent elements without wrapper
return (
<h1>Title</h1>
<p>Paragraph</p>
); // Error!

// ✅ Correct using Fragment (short syntax)
return (
<>
<h1>Title</h1>
<p>Paragraph</p>
</>
);
✅ Fragments solve the "multiple elements" problem without adding extra DOM nodes!
• Short syntax: <>...</>
• Long syntax: <React.Fragment>...</React.Fragment>
• Use when you need to return multiple elements from a component
7️⃣ Conditional Rendering in JSX
// Using ternary operator
{isLoggedIn ? (<h1>Welcome back!</h1>) : (<h1>Please login</h1>)}

// Using && operator (render if true)
{hasMessage && <p>You have a new message!</p>}
🎯 Conditional Rendering Examples:

✅ Logged In User: Welcome back, Kalyan!

🎉 Notification: You have a new message!

❌ Guest User: Please login to continue

✅ Ternary: condition ? trueValue : falseValue
✅ &&: condition && element — renders element only if condition is true
8️⃣ Styling in JSX
// Inline style (double curly braces — one for JS, one for object)
<div style={{color: 'red', fontSize: '20px'}}>Text</div>

// CSS Classes (use className, not class)
<h1 className="title">Heading</h1>

// Import CSS file
import './App.css'
🎨 Styling Results:
Red text with inline style (20px)
This uses className approach with CSS
✅ Inline: style={{property: 'value'}} (camelCase)
✅ Classes: className="myClass"
💡 External CSS is cleaner for complex styling!

📁 Complete JSX Component Example
function UserCard() {
const user = {
name: "Kalyan",

return (

<h2>{user.name}</h2>

</div>

}

• Variables in JSX {user.name}

⚠️ Common Beginner Mistakes
  • ❌ Using class instead of className — class is a JavaScript keyword.
  • ❌ Forgetting to close tags — <img /> and <input /> must be self-closing.
  • ❌ Returning multiple elements without a fragment or wrapper div — use <> </>.
  • ❌ Writing comments inside JSX without braces — use {/* comment */}.
  • ❌ Using for instead of htmlFor in labels.

💼 Common Interview Questions

❓ What is JSX?

JavaScript XML — syntax extension that allows writing HTML-like code in JavaScript.

❓ Why use className instead of class?

"class" is a reserved keyword in JavaScript.

❓ Can you use if/else directly in JSX?

No — use ternary operators or && operator, or extract logic outside return.

❓ What are React Fragments?

Allow returning multiple elements without adding extra DOM nodes (<> </>).
