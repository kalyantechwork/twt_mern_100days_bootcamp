1️⃣ What is GitHub? (Cloud for Git Repositories)

📌 Purpose & Usage

GitHub is a cloud platform that hosts Git repositories online — like Google Drive for code, but built specifically for developers. It adds collaboration features (pull requests, issues, project boards), and serves as your professional portfolio.

Where it's used: Every tech company — Microsoft, Google, Netflix, Spotify, Airbnb. Open source projects (React, Vue, TensorFlow) live on GitHub.

Local Git Repository ↔ git push → GitHub Cloud ↔ git clone/pull → Other Developers

💡 Git vs GitHub

Git = version control tool (runs on your computer)
GitHub = cloud platform (stores repositories online)

🏆 Why GitHub?

✓ Portfolio for recruiters
✓ Team collaboration
✓ Backup your code
✓ Open source contributions

2️⃣ Create GitHub Account & First Repository

📌 Steps to Create Your First Repository

Go to github.com, sign up (free account), then click the "+" icon → "New repository". Name it (e.g., "my-first-project"), keep it public, and click "Create Repository". GitHub will show commands to push your local code.

# Steps:
1. Go to github.com and sign up (free)
2. Click "New Repository"
3. Name your repo (e.g., "my-first-project")
4. Click "Create Repository"
5. Copy the repository URL (https://github.com/username/repo.git)
✅ After creating, GitHub shows these commands:

git remote add origin https://github.com/yourusername/my-first-project.git
git branch -M main
git push -u origin main

📌 Always copy the repository URL — you'll need it for git remote add.
3️⃣ Connect Local Git to GitHub (git remote add origin)

📌 What is git remote?

git remote add links your local repository to a GitHub repository. "origin" is the conventional name for the main remote. After this, Git knows where to push/pull from.

# Inside your project folder (after git init)
git remote add origin https://github.com/username/repo-name.git

# Verify remote connection
git remote -v
$ git remote add origin https://github.com/kalyan/my-portfolio.git
$ git remote -v
origin https://github.com/kalyan/my-portfolio.git (fetch)
origin https://github.com/kalyan/my-portfolio.git (push)

✅ Your local repo is now linked to GitHub!
⚠️ "origin" is just a nickname — you can name it anything, but "origin" is standard.
4️⃣ Push Code to GitHub (git push)

📌 What does git push do?

git push uploads your local commits to GitHub. The first push needs -u to set the upstream branch. After that, just git push works.

# First push (set upstream)
git push -u origin main

# Or if your default branch is master
git push -u origin master

# Subsequent pushes
git push
$ git push -u origin main
Enumerating objects: 5, done.
Counting objects: 100% (5/5), done.
Writing objects: 100% (5/5), done.
To https://github.com/username/repo.git
* [new branch] main -> main
Branch 'main' set up to track remote branch 'main' from 'origin'.

✅ Your code is now LIVE on GitHub!
🔑 Authentication: Use Personal Access Token (not password) — Settings → Developer settings → Tokens.
5️⃣ git clone — Download Repository from GitHub

📌 What does git clone do?

git clone downloads an entire repository from GitHub to your local machine, including all commit history and branches. It automatically sets up the remote connection.

# Clone a repository (downloads to folder with repo name)
git clone https://github.com/username/repo-name.git

# Clone into specific folder name
git clone https://github.com/username/repo-name.git my-custom-folder
$ git clone https://github.com/facebook/react.git
Cloning into 'react'...
remote: Enumerating objects: 250000, done.
Receiving objects: 100% (250000/250000), done.

✅ After cloning, you have a complete local copy!
💡 No need to run git init — clone creates the .git folder automatically.
6️⃣ git pull — Download Latest Changes from GitHub

📌 What does git pull do?

git pull fetches the latest changes from GitHub and merges them into your local branch. Always pull before starting work to avoid conflicts.

# Inside your local repository
git pull origin main

# Or if upstream is configured
git pull
$ git pull origin main
remote: Enumerating objects: 5, done.
From https://github.com/username/repo
* branch main -> FETCH_HEAD
Updating a1b2c3d..e4f5g6h
Fast-forward
index.html | 2 +-
1 file changed, 1 insertion(+), 1 deletion(-)

✅ Your local repository is now up to date with GitHub!

8️⃣ Complete Professional GitHub Workflow
# 1. Create project locally
mkdir awesome-app && cd awesome-app
git init
echo "# Awesome App" > README.md
git add .
git commit -m "Initial commit"

# 2. Create repo on GitHub (no README, no .gitignore)
# 3. Connect and push
git remote add origin https://github.com/username/awesome-app.git
git branch -M main
git push -u origin main

# 4. Make changes, commit, push
echo "new feature" > feature.js
git add .
git commit -m "Add new feature"
git push

# 5. Another developer clones and pulls
git clone https://github.com/username/awesome-app.git
cd awesome-app
git pull
Write Code → git add → git commit → git push → GitHub → git clone/git pull → Latest Code
✅ This is the exact workflow used by developers at companies worldwide!
🔁 Repeat daily: pull → work → commit → push

🔐 GitHub Authentication — Personal Access Token (PAT)

📌 Important: Passwords No Longer Work

GitHub no longer accepts account passwords for Git operations. You MUST use a Personal Access Token (PAT) or SSH key. Generate one at: GitHub Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new token.

🔑 When git push asks for password, enter your Personal Access Token (not your GitHub password).
💡 Save the token somewhere safe — you won't see it again!
⚠️ Common Beginner Mistakes & Solutions
  • ❌ Wrong repository URL → use exact URL from GitHub. Check with git remote -v.
  • ❌ Forgetting to commit before push → git status shows uncommitted changes.
  • ❌ Authentication failed → use Personal Access Token instead of password. Generate one in GitHub Settings.
  • ❌ Branch name mismatch (master vs main) → use git branch -M main to rename.
  • ❌ Push rejected because remote has changes → git pull first, then push.
