🌤️ Project 1: Weather App (Fetch API + Async/Await)

📌 What This Project Teaches

Weather App uses the Fetch API to get real-time weather data from OpenWeatherMap. It teaches async/await, API integration, JSON parsing, and dynamic DOM updates.

Where it's used: Weather widgets, travel apps, smart home dashboards, news websites, agriculture apps.

🌍 Weather App Demo (with Simulated API)

async function getWeather() {
const city = document.getElementById("cityInput").value;
const response = await fetch(`api.openweathermap.org/data/2.5/weather?q=${city}`);
const data = await response.json();
displayWeather(data);
}

💼 Common Interview Questions (After These Projects)

❓ Why async/await?

To handle asynchronous API calls cleanly without callback hell or complex .then chains.

❓ Why localStorage?

To persist data across page reloads (dark mode, notes, cart, user preferences).

❓ What does fetch() return?

A Promise that resolves to a Response object, which we then parse with .json().

❓ Difference between localStorage and sessionStorage?

localStorage persists until cleared; sessionStorage clears when tab closes.

🐛 Debugging Tips for Beginners
  • ✅ Use console.log(data) to inspect API responses — see what data you're getting!
  • ✅ Check browser DevTools → Application tab to view Local Storage contents.
  • ✅ Use try...catch for fetch errors to handle network issues gracefully.
  • ✅ Verify DOM element IDs match exactly — one typo breaks the app.
  • ✅ Use debugger; statement to pause code execution and inspect variables.
