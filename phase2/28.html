1️⃣ What are Props? (Properties)

📌 Purpose & Usage

Props (short for properties) are read-only data passed from a parent component to a child component. They allow components to be dynamic, configurable, and reusable. Think of props like function arguments — the parent component "calls" the child with specific data.

Where it's used: Every React application — passing user data, configuration, event handlers, styles, and more from parent to child components.

Parent Component → passes props → Child Component → displays dynamic data
✅ Props = Properties = data passed from parent to child
✅ Props are read-only (immutable) — child cannot modify them
✅ Props make components reusable — same component, different data
💡 Analogy: Function parameters in JavaScript!
2️⃣ Passing Props — Basic Syntax
// Parent Component (App.jsx)
function App() {
return (
<div>
<Greeting name="Kalyan" age={25} />
<Greeting name="Priya" age={22} />
</div>
);
}

// Child Component (Greeting.jsx)
function Greeting(props) {
return (
<div>
<h2>Hello, {props.name}!</h2>
<p>Age: {props.age}</p>
</div>
);
}

Hello, Kalyan!

Age: 25

Hello, Priya!

Age: 22

✅ Same Greeting component used twice with different props!
✅ String props: name="Kalyan" (quotes)
✅ Number props: age={25} (curly braces)
3️⃣ Props are Read-Only (Never Modify Props Directly)
// ❌ WRONG - Never modify props directly
function Button(props) {
props.label = "Changed"; // ❌ This causes error!
return <button>{props.label}</button>
}

// ✅ CORRECT - Props are for reading only
function Button(props) {
return <button>{props.label}</button>
}
🔒 Props are immutable — they cannot be changed by the child component!
✅ If you need to change data, use State (coming in Day 29)
✅ Props flow one way: Parent → Child (unidirectional data flow)
💡 This makes React apps predictable and easier to debug.
4️⃣ Destructuring Props (Cleaner Code)
// Without destructuring
function UserCard(props) {
return <div>{props.name} - {props.email}</div>;
}

// With destructuring (cleaner!)
function UserCard({ name, email }) {
return <div>{name} - {email}</div>;
}

// With default values
function Button({ label = "Click Me", variant = "primary" }) {
return <button className={variant}>{label}</button>;
}
✅ Destructuring makes code cleaner and more readable!
✅ Instead of props.name, just use name directly
✅ Default values: { label = "Click Me" } — used if prop not provided
💡 This is a standard pattern in React development
5️⃣ Different Types of Props
function App() {
const user = { name: "Kalyan", city: "Hyderabad" };
const colors = ["red", "blue", "green"];
return (
<div>
<Message text="Hello World" /> {/* String */}
<Counter initialValue={10} /> {/* Number */}
<Toggle isEnabled={true} /> {/* Boolean */}
<ColorList colors={colors} /> {/* Array */}
<UserProfile user={user} /> {/* Object */}
</div>
);
}
✅ String: text="Hello" (quotes)
✅ Number: count={5} (curly braces, no quotes)
✅ Boolean: isActive={true} or isActive
✅ Array: items={[1,2,3]}
✅ Object: data={{name: "Kalyan"}}
💡 Any JavaScript value can be passed as a prop!
6️⃣ Passing Functions as Props (Child → Parent Communication)
// Parent Component
function App() {
const handleClick = () => {
alert("Button clicked from child!");
};
return <Button onClick={handleClick} label="Click Me" />;
}

// Child Component
function Button({ onClick, label }) {
return <button onClick={onClick}>{label}</button>;
}
✅ Parent passes a function to child via props
✅ Child executes the function when clicked
✅ This is how children communicate with parents!
💡 This pattern is essential for forms, modals, and user interactions
7️⃣ Special Prop: children (Nested Content)
// Card component that wraps children
function Card({ children, title }) {
return (
<div className="card">
<h2>{title}</h2>
<div className="content">{children}</div>
</div>
);
}

// Using Card component
function App() {
return (
<Card title="Welcome">
<p>This content is passed as children prop!</p>
<button>Learn More</button>
</Card>
);
}

Welcome

This content is passed as children prop!
✅ children is a special prop for nested content
✅ Enables component composition and layout components
✅ Commonly used for modals, cards, layouts, and wrappers
📁 Complete Example: Product List with Props
// ProductCard.jsx (child)
function ProductCard({ name, price, onBuy }) {
return (
<div className="product-card">
<h3>{name}</h3>
<p>₹{price}</p>
<button onClick={onBuy}>Buy Now</button>
</div>
);
}

✅ Same ProductCard component used 3 times with different props!
✅ Each product has its own name and price
✅ onBuy function prop allows child to communicate with parent
💡 This is the standard pattern for rendering lists in React

⚠️ Common Beginner Mistakes
  • ❌ Trying to modify props directly → props are read-only! Use state instead.
  • ❌ Forgetting curly braces for numbers/booleans: <Component count=5 /> ❌ → <Component count={5} /> ✅
  • ❌ Misspelling prop names → component receives undefined. Check spelling!
  • ❌ Passing functions without handling parameters correctly → use arrow functions: onClick={() => handleClick(id)}
  • ❌ Forgetting to pass required props → component breaks. Use default values or PropTypes.
💼 Common Interview Questions

❓ What are props in React?

Read-only data passed from parent to child components, like function parameters.

❓ Are props mutable?

No, props are immutable. Children cannot modify props received from parent.

❓ How to pass a function as a prop?

<Button onClick={handleClick} /> — then call it in child with onClick={onClick}

❓ What is the children prop?

Special prop that contains content between component tags — enables composition.
