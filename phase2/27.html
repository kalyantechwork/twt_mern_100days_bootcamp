1️⃣ What are React Components?

📌 Purpose & Usage

Components are independent, reusable pieces of UI — like LEGO blocks that you assemble to build a complete website. Every React app is essentially a tree of components. A component can be as small as a button or as large as an entire page.

Where it's used: Every React application — from simple portfolios to massive enterprise apps like Facebook, Instagram, Netflix.

App (Root Component)
├── Header
│ ├── Logo
│ └── Navbar
├── Main
│ ├── ProductList
│ │ └── ProductCard (reused 10 times)
│ └── Sidebar
└── Footer
✅ Components are the heart of React
✅ Each component is a JavaScript function that returns JSX
✅ Components can be nested inside other components
✅ Components are reusable — write once, use everywhere
2️⃣ Why Use Components?

♻️ Reusability

Write once, use anywhere. A Button component can be used 100 times across your app.

🔧 Maintainability

Fix a bug in one component → fixes everywhere it's used.

👥 Team Collaboration

Different developers can work on different components simultaneously.

🧪 Testability

Test each component independently for better code quality.

3️⃣ Functional Components (The Modern Way)

📌 What are Functional Components?

Functional components are plain JavaScript functions that return JSX. They are the standard way to write React components today (with hooks).

// Basic functional component
function Welcome() {
return <h1>Hello, React!</h1>;
}

// Arrow function version
const Greeting = () => {
return <p>Welcome to React Components</p>;
};

// Export so other files can import
export default Welcome;

Hello, React! 👋

Welcome to React Components

✅ Functional components are simple JavaScript functions
✅ They return JSX (HTML-like syntax)
✅ Always start with capital letter: Welcome, not welcome
4️⃣ How to Use Components (Import/Export)
// File: Header.jsx
function Header() {
return <header>My Website Header</header>;
}
export default Header;

// File: App.jsx (main component)
import Header from './Header';
import Footer from './Footer';

function App() {
return (
<div>
<Header />
<main>Content goes here</main>
<Footer />
</div>
);
}
export default App;
Component 1 (Header.jsx) → export → import → Component 2 (App.jsx) → render
✅ export default — makes component available for import
✅ import ComponentName from './path' — brings component into current file
✅ Use component with JSX tag: <ComponentName />
5️⃣ Important Component Naming Rules

✅ Start with Capital Letter

function MyComponent() — not myComponent

React treats lowercase as HTML tags

✅ PascalCase

UserProfile, ShoppingCart, ProductCard, Button

✅ One Component per File

Organized and easy to find. File name = component name (Button.jsx)

6️⃣ Component Composition (Building Complex UIs)
// Button component (reusable)
function Button() {
return <button className="btn">Click Me</button>;
}

// Card component (uses Button)
function Card() {
return (
<div className="card">
<h3>Product Title</h3>
<Button />
</div>
);
}

Product Title

✅ Card component composes (uses) Button component
✅ Composition = building complex UIs from smaller components
✅ Reusable components can be nested inside each other

8️⃣ Organizing Components (Folder Structure)
src/
├── components/ (all reusable components)
│ ├── Header.jsx
│ ├── Footer.jsx
│ ├── Button.jsx
│ ├── ProductCard.jsx
│ └── Navbar.jsx
├── pages/ (page-level components)
│ ├── Home.jsx
│ ├── About.jsx
│ └── Contact.jsx
├── App.jsx
└── main.jsx
📁 Best Practice: Keep each component in its own file with the same name (e.g., Button.jsx for Button component).
⚠️ Common Beginner Mistakes
  • ❌ Forgetting to export default the component → import fails with "is not exported" error.
  • ❌ Using lowercase for component names → function button() ❌ vs function Button() ✅ (React treats lowercase as HTML tags).
  • ❌ Creating components inside other components (causes performance issues and state problems).
  • ❌ Not importing the component before using it → "Component is not defined" error.
  • ❌ Putting too much logic inside one component → break it into smaller, focused components.

💼 Common Interview Questions

❓ What is a React component?

Independent, reusable piece of UI that returns JSX.

❓ Types of components?

Functional components (modern) and Class components (legacy, less common now).

❓ Why use components?

Reusability, maintainability, separation of concerns, easier testing.

❓ Component naming convention?

PascalCase (e.g., UserProfile, ShoppingCart, ProductCard).
