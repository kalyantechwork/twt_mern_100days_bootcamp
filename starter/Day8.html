1️⃣ What are Functions? (Reusable Code Blocks)

🎯 Why Functions Matter

📌 Purpose: A function is a reusable block of code that performs a specific task. Instead of writing the same code multiple times, you write it once inside a function and call it whenever needed. Functions help organize code, reduce repetition, and make debugging easier.

💡 Key parts: function keyword, function name, parentheses () for parameters, curly braces {} for code block, and optional return statement.

// Basic function declaration
function sayHello() {
return "Hello, World!";
}
console.log(sayHello()); // Call the function

2️⃣ Parameters vs Arguments (Inputs to Functions)

👤 Passing Data to Functions

🎯 Purpose: Parameters are placeholders defined in the function declaration (like name). Arguments are actual values passed when calling the function (like "Kalyan"). Parameters make functions flexible and reusable with different inputs.

💡 Pro tip: You can have multiple parameters separated by commas.

function greetUser(name, age) {
return `Hello ${name}, you are ${age} years old.`;
}
greetUser("Kalyan", 25);

3️⃣ Return Statement (Sending Output Back)

📤 Getting Results from Functions

🎯 Purpose: The return statement specifies the value a function sends back to the caller. Without return, a function returns undefined. Return stops function execution immediately.

💡 Remember: A function can only return one value, but that value can be an array or object containing multiple pieces of data.

function addNumbers(a, b) {
return a + b; // sends result back
}
let result = addNumbers(10, 20); // result = 30

4️⃣ Function Expression (Function as a Value)

📜 Storing Functions in Variables

🎯 Purpose: A function expression stores a function in a variable. Unlike declarations, function expressions are NOT hoisted (cannot be called before definition). Useful for conditional function creation and passing functions as arguments.

💡 Syntax: const multiply = function(x, y) { return x * y; };

const multiply = function(x, y) {
return x * y;
};
console.log(multiply(4, 6)); // 24

5️⃣ Arrow Functions (=>) - Modern JavaScript

🏹 Shorter Function Syntax

🎯 Purpose: Arrow functions provide a concise syntax. If there's a single expression, you can omit return and curly braces (implicit return). Great for short callbacks and array methods. However, they behave differently with the this keyword.

💡 Syntax: (param1, param2) => param1 + param2 or param => param * 2 for single parameter.

// Single line implicit return
const square = (n) => n * n;
// Multi-line arrow (needs explicit return and {})
const greet = (name) => {
return `Hello ${name}!`;
};

6️⃣ Scope (Where Variables Live)

🌍 Understanding Variable Accessibility

🎯 Purpose: Scope defines where variables are accessible. Global scope variables are accessible everywhere. Local scope (inside functions) are only accessible within that function. This prevents naming conflicts.

💡 Best practice: Always use let or const instead of var to avoid accidental global variables.

let globalVar = "I am global";
function showScope() {
let localVar = "I am local";
return globalVar + " & " + localVar;
}
// console.log(localVar); // Error! Not accessible

7️⃣ Block Scope (let & const inside {})

📦 Scope in if/for blocks

🎯 Purpose: Variables declared with let and const have block scope — they exist only inside the nearest {} (if statements, loops, etc.). This is safer than var, which ignores block scope.

if (true) {
let blockScoped = "Only inside if";
const alsoBlock = "Constant block";
}
// console.log(blockScoped); // Error!

8️⃣ Hoisting (Moving Declarations to Top)

📤 Hoisting Behavior in JavaScript

🎯 Purpose: Hoisting is JavaScript's behavior of moving declarations to the top of their scope during compilation. Function declarations are fully hoisted (can be called before definition). var variables are hoisted but initialized as undefined. let/const are hoisted but not initialized (Temporal Dead Zone).

// Function hoisting works!
sayHi(); // "Hi there!"
function sayHi() { console.log("Hi there!"); }

// var hoisting (undefined, not error)
console.log(myVar); // undefined
var myVar = 5;

9️⃣ Callback Functions (Function as Argument)

🔄 Passing Functions to Functions

🎯 Purpose: A callback is a function passed as an argument to another function. The outer function then calls (executes) the callback. This is fundamental for asynchronous operations (like timers, event listeners, API calls).

💡 Real-world: setTimeout, addEventListener, array methods (map, filter) all use callbacks.

function processUser(name, callback) {
return callback(name);
}
const sayHello = (n) => `Hello ${n}!`;
processUser("Kalyan", sayHello); // "Hello Kalyan!"
🔄 Callback Execution:
⏱️ setTimeout (Asynchronous Callback)

⏰ Delayed Execution with Callbacks

🎯 Purpose: setTimeout is a built-in function that executes a callback after a specified delay (in milliseconds). It's a perfect example of asynchronous JavaScript — the code continues running while waiting.

setTimeout(() => {
console.log("Runs after 2 seconds");
}, 2000);

⚠️ Common Function Mistakes for Beginners
🚫 Avoid These Errors:
  • ❌ Forgetting parentheses when calling: myFunction vs myFunction() — parentheses execute!
  • ❌ Missing return statement: function returns undefined by default
  • ❌ Confusing parameters with arguments: parameters are placeholders, arguments are actual values
  • ❌ Trying to access local variable outside its scope → ReferenceError
  • ❌ Arrow function with multiple lines missing return and {}: (a,b) => a+b works, but (a,b) => { a+b } returns undefined
  • ❌ Callback hell: nesting too many callbacks — use promises/async-await later
