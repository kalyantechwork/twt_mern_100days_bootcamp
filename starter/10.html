📊 Project 1: Counter App (Increase / Decrease / Reset)

📚 Concepts Used in Counter App

  • Variables (let): let count = 0; stores the current number.
  • DOM Selection: document.getElementById() or querySelector() to access HTML elements.
  • addEventListener: Listens for "click" events on buttons and triggers functions.
  • textContent: Updates the displayed number dynamically.
  • State Management: Keeping track of count value and updating UI when it changes.

🔢 Counter with Explanation

// Step 1: Create a variable to store count
let count = 0;

// Step 2: Select the display element and buttons
const display = document.getElementById("counterDisplay");
const incBtn = document.getElementById("incCounter");

// Step 3: Add event listener to increase button
incBtn.addEventListener("click", () => {
count++; // increment count
display.textContent = count; // update UI
});

0
✅ How it works: Every time you click "+", the count increases by 1 and the display updates. Clicking "-" decreases. Reset sets count back to 0.
🧮 Project 2: Calculator (Add, Subtract, Multiply, Divide)

📚 Concepts Used in Calculator

  • Input Values & Type Conversion: .value returns a string, so we use Number() to convert to number.
  • Arithmetic Operators: +, -, *, / for calculations.
  • Conditional Logic: Check division by zero to avoid errors.
  • Multiple Event Listeners: Different buttons perform different operations on the same input values.
  • toFixed(): Format decimal results to 2 decimal places for cleaner display.

📟 Calculator with Explanation

// Step 1: Get input values and convert to numbers
let a = Number(num1.value);
let b = Number(num2.value);

// Step 2: Perform calculation based on button clicked
function add() { return a + b; }
function subtract() { return a - b; }

// Step 3: Display result on the page

✅ Project 3: To-Do App (Add & Delete Tasks)

📚 Concepts Used in To-Do App

  • createElement(): Dynamically creates new HTML elements (li, button, span).
  • appendChild(): Adds created elements to the DOM tree.
  • remove(): Deletes an element from the DOM when delete button is clicked.
  • Event Delegation / Individual Listeners: Each delete button gets its own click listener.
  • Keyboard Events: keypress and Enter key to add tasks without clicking button.
  • Input Validation: Check if task is empty before adding.

📝 To-Do List with Explanation

// Step 1: Create new elements dynamically
const li = document.createElement("li");
const span = document.createElement("span");
const deleteBtn = document.createElement("button");

// Step 2: Add content and styling
span.textContent = taskText;
deleteBtn.textContent = "Delete";

// Step 3: Add delete functionality
deleteBtn.addEventListener("click", () => li.remove());

// Step 4: Append to parent
li.appendChild(span);
li.appendChild(deleteBtn);
todoList.appendChild(li);

    ✅ Dynamic Creation: Each task is a new <li> element created with JavaScript. Delete button removes that specific task using li.remove().
    🎨 Bonus Project: Random Color Generator

    📚 Concepts Used in Color Generator

    • Math.random(): Generates a random decimal between 0 and 1.
    • Math.floor(): Rounds down to get an integer.
    • toString(16): Converts number to hexadecimal (base 16) for hex color codes.
    • padStart(): Ensures 6 digits for hex color (e.g., #0f0f0f).
    • Dynamic Style Change: element.style.backgroundColor to change color.

    🌈 Random Hex Color Generator

    // Generate random hex color: # + 6 random hex digits
    const randomColor = `#${Math.floor(Math.random()*16777215).toString(16).padStart(6, '0')}`;
    element.style.backgroundColor = randomColor;

    ⚠️ Common Beginner Mistakes & Fixes
    • ❌ Wrong selector IDs: always match HTML id exactly → use getElementById or correct querySelector syntax.
    • ❌ Forgetting Number() conversion: "10" + "5" = "105" → fix with Number() or parseFloat().
    • ❌ Not clearing input after adding task: bad UX → set input.value = "" after adding.
    • ❌ Script running before DOM loads: place script before closing </body> or use DOMContentLoaded.
    • ❌ Creating duplicate IDs: IDs must be unique on a page.
