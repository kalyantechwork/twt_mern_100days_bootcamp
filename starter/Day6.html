1️⃣ Display Property (block, inline, flex, grid)

🔲 Understanding How Elements Behave

🎯 Purpose: The display property determines how an element is shown on the page. block elements take full width (like <div>), inline elements only take as much width as needed (like <span>), flex enables powerful one-dimensional layouts, and grid enables two-dimensional layouts.

💡 Key insight: Changing display changes how other elements interact with it. Flex and Grid are the foundation of modern responsive designs.

.block { display: block; background: orange; width: 100%; }
.inline { display: inline; background: lightblue; }
.flex { display: flex; gap: 10px; }

2️⃣ Flexbox (One-Dimensional Layout - Rows OR Columns)

↔️ What is Flexbox?

🎯 Purpose: Flexbox is a layout model designed for one direction at a time — either a row (horizontal) or a column (vertical). It makes distributing space and aligning items super easy. Perfect for navigation bars, card groups, and centering content.

💡 Key properties: justify-content (main axis alignment), align-items (cross axis), flex-wrap (allow items to wrap to next line), gap (space between items).

.flex-container {
display: flex;
justify-content: center; /* centers horizontally */
gap: 20px; /* space between items */
flex-wrap: wrap; /* wrap on small screens */
}

3️⃣ justify-content (Aligning Items on Main Axis)

🎯 Space-Between vs Center vs Space-Around

🎯 Purpose: justify-content controls how extra space is distributed between flex items. space-between puts first item at start, last at end, space evenly in between. center groups items in the middle. space-around gives equal space around each item.

.space-between { justify-content: space-between; }
.center { justify-content: center; }

🔸 center (all items together in middle):

✅ Use space-between for navbars, center for hero sections.

4️⃣ CSS Grid (Two-Dimensional Layout - Rows AND Columns)

🔲 What is CSS Grid?

🎯 Purpose: Grid is for layouts that need both rows and columns simultaneously. You define a grid with grid-template-columns and grid-template-rows. It's ideal for overall page structure, image galleries, dashboards, and complex responsive designs.

💡 Key insight: With Flexbox you control one direction; with Grid you control both. Use fr unit for fractional space (like 1fr = 1 part).

.grid {
display: grid;
grid-template-columns: 1fr 1fr 1fr; /* 3 equal columns */
gap: 16px;
}

5️⃣ Position Property (relative, absolute, fixed, sticky)

📍 Controlling Element Placement

🎯 Purpose: The position property changes how an element is placed in the document flow. relative moves element relative to its normal position. absolute positions relative to nearest positioned ancestor. fixed stays in place while scrolling. sticky toggles between relative and fixed when scrolled past.

💡 Use case: Absolute for badges, fixed for back-to-top buttons, sticky for headers that stick on scroll.

.relative { position: relative; left: 20px; }
.absolute-parent { position: relative; }
.absolute-child { position: absolute; top: 0; right: 0; }

6️⃣ z-index (Stacking Order of Overlapping Elements)

📚 Controlling Which Element Appears On Top

🎯 Purpose: When elements overlap, z-index decides who's on top. Higher z-index values appear above lower ones. Only works on positioned elements (relative, absolute, fixed).

💡 Common use: Dropdown menus, modals, tooltips, floating action buttons.

.box1 { position: relative; z-index: 2; }
.box2 { position: relative; z-index: 1; top: -20px; }

7️⃣ Media Queries (Making Websites Responsive)

📱 How Websites Adapt to Any Screen Size

🎯 Purpose: @media queries apply different CSS styles based on screen width. They are the heart of responsive design — allowing layouts to change from desktop (3 columns) to tablet (2 columns) to mobile (1 column).

💡 Best practice: Design for mobile first, then use min-width for larger screens. Always include viewport meta tag: <meta name="viewport" content="width=device-width">

/* Default: mobile layout */
.container { display: flex; flex-direction: column; }
/* Tablet and up */
@media (min-width: 768px) {
.container { flex-direction: row; }
}
🔄 Resize your browser window to see changes!
📏 Current layout behavior

Desktop (>768px) → horizontal arrangement
Mobile (≤768px) → stacked vertically

⚖️ Flexbox vs Grid: When to Use Each?
📐 Choose the right tool:
ScenarioBest ChoiceReason
Navigation barFlexboxOne-dimensional row alignment
Card group (wrapping)Flexbox + wrapItems flow naturally
Full page layoutGridRows and columns control
Image galleryGridPerfect 2D arrangement
Centering a buttonFlexboxSimple justify/align
⚠️ Common CSS Layout Mistakes (Avoid These!)
  • ❌ Forgetting display: flex or display: grid → no layout appears
  • ❌ Using absolute positioning everywhere → breaks responsiveness and flow
  • ❌ Not testing media queries on actual mobile sizes → always use DevTools device emulation
  • ❌ Missing viewport meta tag → <meta name="viewport"> is essential for responsive design
  • ❌ Confusing justify-content (main axis) with align-items (cross axis) in Flexbox
  • ❌ Using pixels instead of fr or % in Grid → fr adapts to screen size beautifully
