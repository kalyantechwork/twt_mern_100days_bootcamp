1️⃣ What is the DOM? (Document Object Model)

📌 Purpose: The DOM is a programming interface for HTML. When a browser loads a webpage, it creates a tree-like structure of all HTML elements. JavaScript can access and modify this tree — adding, removing, or changing elements — to make webpages dynamic and interactive.

🌳 Analogy: Think of the DOM as a family tree of your HTML. Each element (like <div>, <p>, <button>) is a node. JavaScript can find any node, read it, change it, or delete it, and even create new nodes.

💡 Key insight: The DOM is not HTML — it's a live representation that JavaScript can manipulate. When you change the DOM, the page updates instantly without refreshing.

2️⃣ Selecting Elements (querySelector, getElementById)

🔍 Finding Elements in the DOM

🎯 Purpose: Before you can change an element, you need to select it. document.querySelector() is the most powerful method — it uses CSS selectors. Use #id for IDs, .class for classes.

💡 Common selectors: querySelector('#myId'), querySelector('.myClass'), querySelector('p') (first paragraph).

// Select by ID
let title = document.querySelector("#mainTitle");
// Select by class
let boxes = document.querySelectorAll(".box");
// Change content
title.textContent = "New Text!";

✅ querySelector returns the first matching element. Use querySelectorAll for multiple.

3️⃣ Changing Content (textContent vs innerHTML)

📝 Modifying What Users See

🎯 Purpose: textContent changes plain text (safe, faster). innerHTML parses HTML tags (powerful but can be risky with user input). Use textContent for regular text, innerHTML when you need to insert HTML elements.

element.textContent = "Plain text only";
element.innerHTML = "<strong>Bold text</strong>";

4️⃣ Changing CSS Styles with JavaScript

🎨 Style Property Manipulation

🎯 Purpose: Use element.style.property to change CSS dynamically. Property names become camelCase (e.g., backgroundColor instead of background-color).

element.style.color = "blue";
element.style.fontSize = "24px";
element.style.backgroundColor = "#facc15";

5️⃣ Events (addEventListener)

🖱️ Responding to User Actions

🎯 Purpose: Events like click, mouseover, keydown let you run code when users interact with your page. addEventListener is the modern way to attach event handlers.

💡 Common events: click, dblclick, mouseenter, mouseleave, input, submit, keyup.

button.addEventListener("click", () => {
alert("Button clicked!");
});

6️⃣ Getting Form Input Values

📝 Reading What Users Type

🎯 Purpose: Use .value property to get what's inside text inputs, textareas, and select dropdowns.

let userInput = document.querySelector("#myInput").value;
console.log(userInput);

7️⃣ Form Handling (preventDefault)

🚫 Stop Page Reload on Form Submit

🎯 Purpose: When a form submits, the browser reloads the page by default. Use e.preventDefault() to stop this and handle the data with JavaScript instead.

form.addEventListener("submit", (e) => {
e.preventDefault();
// process data here
});

8️⃣ Creating and Removing Elements Dynamically

➕ Building Elements with JavaScript

🎯 Purpose: createElement() creates a new element in memory. appendChild() adds it to the DOM. removeChild() or remove() deletes elements. This is how to-do lists, shopping carts, and dynamic content work.

let newDiv = document.createElement("div");
newDiv.textContent = "Hello!";
parent.appendChild(newDiv);

9️⃣ classList: Managing CSS Classes

🎭 Toggle, Add, Remove Classes

🎯 Purpose: classList makes it easy to add, remove, or toggle CSS classes. This is better than directly modifying styles because you keep styling in CSS and behavior in JavaScript.

element.classList.add("dark-mode");
element.classList.remove("dark-mode");
element.classList.toggle("active");

⚠️ Common DOM Manipulation Mistakes
🚫 Avoid These Errors:
  • ❌ Script runs before DOM loads → use DOMContentLoaded event or put script at bottom of body
  • ❌ Wrong selector syntax: querySelector('.myclass') needs dot, querySelector('#myid') needs hash
  • ❌ Forgetting event.preventDefault() → form reloads the page
  • ❌ Using innerHTML with user input → security risk (XSS) — use textContent for untrusted data
  • ❌ Not checking if element exists: if(element) { ... } before manipulating
  • ❌ Confusing textContent (plain text) with innerHTML (parses HTML)
