📅 Day 2 · Dev Environment Setup

🛠️ Install Development Environment
VS Code, Node.js, Git, npm & MERN Setup for Absolute Beginners

1️⃣ What is a Development Environment?

A development environment means all the tools required to build applications. Just like a mechanic needs tools to repair vehicles, developers also need tools to create websites and applications.

🔧 Essential Tools

VS Code (editor), Node.js (run JS), npm (package manager), Git (version control), GitHub (cloud storage), Chrome browser (testing).

🖥️ Why Setup Matters

Proper environment saves time, prevents errors, and helps you build professional projects easily.

Developer
↓
VS Code (code editor)
↓
Node.js + npm (runtime + packages)
↓
Git + GitHub (version control & backup)
↓
Browser Testing (Chrome DevTools)
2️⃣ Install Visual Studio Code (VS Code)

VS Code is the most popular code editor used by millions. It supports JavaScript, React, Node.js, HTML, CSS, Python, Java and many more languages.

⭐ Why VS Code?

  • ✅ Free & open source
  • 🚀 Fast & lightweight
  • 🧩 Huge extensions support
  • 👶 Beginner-friendly

📥 Installation Steps

1. Open code.visualstudio.com
2. Download for Windows/Mac/Linux
3. Install with default settings
4. ✅ Enable "Add to PATH", "Open with Code" during installation.

Download VS Code → Install → Open Editor → Start Coding
3️⃣ Install Node.js (JavaScript Runtime)

Node.js allows JavaScript to run outside the browser. Without Node.js, JavaScript works only inside browsers. With Node.js, you can create servers, build APIs, connect databases, and create backends.

💡 Real Example: Frontend JavaScript (browser) vs Backend JavaScript (Node.js). MERN stack uses JavaScript everywhere → Frontend (React) + Backend (Node + Express) = easier development.
  • 1 Open nodejs.org
  • 2 Download LTS Version (recommended for beginners)
  • 3 Run installer, keep default settings, click Next → Finish
  • 4 ✅ Verify installation: Open terminal (Command Prompt / PowerShell / Git Bash)
    • node -v
    # Expected output: v22.x.x or any LTS version

    npm -v
    # Expected output: 10.x.x (npm comes with Node.js)

    📦 What is npm?

    npm = Node Package Manager. It installs libraries like React, Express, Axios, Nodemon. Just like Play Store for JavaScript packages.

    npm → Install Packages → Use in Projects
    4️⃣ Understanding Packages & npm

    A package is prewritten code created by developers. Instead of building everything from scratch, we install packages via npm.

    # Example: install React
    npm install react

    # install Express (backend framework)
    npm install express
    5️⃣ Install Git (Version Control)

    Git is a version control system that tracks changes in your code. It helps to save history, restore previous versions, work in teams, and upload to GitHub.

    📀 Real-Life Analogy: Git is like "Save Game Progress" for coding projects. Every change is recorded.
  • 1 Go to git-scm.com/downloads
  • 2 Download for your OS (Windows / Mac / Linux)
  • 3 Install with default settings, keep "Git from the command line" option
    • git --version
    # Output: git version 2.x.x
    6️⃣ GitHub — Cloud Storage for Code

    GitHub.com is a platform to store code online. Benefits: backup projects, share, build portfolio, collaborate, apply for jobs.

    Local Computer (Git) → GitHub Cloud → Access from anywhere

    9️⃣ Terminal / Command Line Basics

    Terminal allows you to run commands: install packages, create folders, run apps.

    CommandAction
    pwd (or cd on Windows)Print current directory
    ls / dirList files/folders
    mkdir folder-nameCreate new folder
    cd folder-nameChange directory
    code .Open current folder in VS Code
    Developer → Terminal Commands → System executes
    📁 Create Your First MERN Project Folder
    mkdir mern-learning
    cd mern-learning
    code .
    # VS Code opens with your workspace

    ✅ Your first development workspace is ready!

    🌐 Browser Developer Tools (Essential)

    DevTools help to inspect HTML, debug CSS, test JavaScript, check API requests. Shortcut: F12 or Right Click → Inspect.

    TabUse
    ElementsView/edit HTML/CSS live
    ConsoleSee JavaScript logs & errors
    NetworkMonitor API calls and requests
    ApplicationLocal storage, cookies, sessions
    Browser → F12 / Inspect → Debug & optimize
    📚 Important Beginner Concepts (Paths & Extensions)

    📂 Absolute vs Relative Path

    Absolute: C:\Users\ADMIN\Desktop
    Relative: ./src/components

    📄 Common File Extensions

    .html (HTML) , .css (styles) , .js (JavaScript) , .jsx (React components) , .json (config data)

    ⚠️ Common Beginner Mistakes (Avoid These)
    • ❌ Installing wrong Node.js version → always LTS version
    • ❌ Not restarting terminal after installation → close and reopen terminal
    • ❌ Not adding Node.js to PATH → during installation, select "Add to PATH"
    • ❌ Opening wrong folder in VS Code → always open project root folder
    • ❌ Forgetting to verify installations with node -v / git --version
    🖥️ Mini Practical Demo — Show These Steps
