4️⃣ Contact Form with Validation
function ContactForm() {
const [formData, setFormData] = useState({ name: '', email: '', message: '' });
const handleSubmit = (e) => { e.preventDefault(); alert('Thank you!'); };
return (<form onSubmit={handleSubmit} className="max-w-lg mx-auto space-y-4">
<input type="text" placeholder="Name" className="w-full px-4 py-2 border rounded-lg" onChange={e => setFormData({...formData, name: e.target.value})} />
<input type="email" placeholder="Email" className="w-full px-4 py-2 border rounded-lg" onChange={e => setFormData({...formData, email: e.target.value})} />
<textarea rows="4" placeholder="Message" className="w-full px-4 py-2 border rounded-lg" onChange={e => setFormData({...formData, message: e.target.value})} />
<button type="submit" className="w-full bg-blue-600 text-white py-3 rounded-lg font-semibold">Send Message</button>
</form>);
}

⚠️ Common Landing Page Mistakes
  • ❌ Not making the navbar sticky → users lose navigation when scrolling (use sticky top-0)
  • ❌ Forgetting mobile menu on small screens → bad UX on phones (use md:hidden with hamburger)
  • ❌ Poor color contrast between text and background → use Tailwind's accessibility-friendly colors
  • ❌ No call-to-action (CTA) above the fold → visitors don't know what to do
  • ❌ Images not optimized → slow loading → use CDN or optimize images
💼 Common Interview Questions

❓ How do you make a responsive navbar?

Use flex with justify-between, hide desktop links on mobile with hidden md:flex, add hamburger menu button for mobile with useState toggle.

❓ What is mobile-first design?

Design for mobile screens first (default styles), then use breakpoint prefixes (sm:, md:, lg:) for larger screens.

❓ How to handle forms in React?

Controlled components with useState for each input, handleSubmit with e.preventDefault(), validation before submission.
