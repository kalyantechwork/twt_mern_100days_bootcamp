1️⃣ Why Build Reusable Components with Tailwind?

📌 Purpose & Usage

Tailwind makes it easy to build consistent, reusable components. Instead of writing custom CSS for every button or card, you compose utility classes. This creates a design system that ensures consistency across your entire application and speeds up development dramatically.

Where it's used: Every professional React project — component libraries are the foundation of scalable applications. Companies like Shopify, GitHub, and Vercel all have their own design systems built with Tailwind.

Utility Classes → Composed into Components → Consistent Design → Faster Development → Less Bug Fixes

8️⃣ @apply Directive — Creating Custom Classes
/* In your main CSS file */
@layer components {
.btn-primary {
@apply bg-blue-500 hover:bg-blue-600 text-white font-bold py-2 px-4 rounded-lg transition duration-200;
}
.card {
@apply bg-white rounded-lg shadow-md overflow-hidden hover:shadow-lg transition-shadow;
}
}
⚠️ Use @apply sparingly — Tailwind works best with utility classes directly in JSX.
✅ @apply is useful for complex repeated patterns like buttons across multiple components.
📁 Complete E-commerce Component Example
export function ProductCard({ product, onAddToCart }) {
return (<div className="bg-white rounded-xl shadow-md overflow-hidden hover:shadow-xl transition-all duration-300">
<img src={product.image} alt={product.name} className="w-full h-48 object-cover" />
<div className="p-4">
<h3 className="font-semibold text-lg mb-2">{product.name}</h3>
<p className="text-gray-600 text-sm mb-3">{product.description}</p>
<div className="flex justify-between items-center">
<span className="text-2xl font-bold text-blue-600">${product.price}</span>
<button onClick={() => onAddToCart(product)} className="bg-blue-500 hover:bg-blue-600 text-white px-4 py-2 rounded-lg">Add to Cart</button>
</div>
</div>
</div>);
}

⚠️ Common Component Mistakes
  • ❌ Making components too specific → harder to reuse. Make components configurable via props.
  • ❌ Not using props for customization → component becomes inflexible, leads to duplicate code.
  • ❌ Hardcoding colors and values instead of using Tailwind utilities → defeats the purpose.
  • ❌ Forgetting responsive design in components → components break on mobile.
  • ❌ Not extracting repeated patterns into components → code duplication.
💼 Common Interview Questions

❓ How do you create reusable components in React?

Use props for configuration, children for composition, and consistent styling with Tailwind classes.

❓ What is the @apply directive?

Tailwind feature to compose utility classes into custom CSS classes — use sparingly.

❓ Why build a component library?

Consistency, faster development, easier maintenance, and better team collaboration.
