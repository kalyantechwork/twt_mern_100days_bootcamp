1️⃣ Why Performance Matters in React

📌 Purpose & Usage

React re-renders components when state or props change. Unnecessary re-renders can slow down your app, especially with large component trees, expensive calculations, or frequent updates. Performance optimization helps your app feel fast and responsive.

Where it's used: Large lists, complex dashboards, real-time data apps, e-commerce sites, and any component that does expensive calculations.

State Change → Component Re-renders → Child Components Re-render → (Potentially hundreds of components)
2️⃣ React.memo — Prevent Child Re-renders

📌 What is React.memo?

React.memo is a higher-order component that memoizes a component — it only re-renders when its props actually change. Without memo, child components re-render every time the parent re-renders, even if their props are identical.

import { memo } from 'react';
const MemoizedChild = memo(function Child({ data }) {
return <div>{data}</div>;
});
🧠 React.memo Simulation

Parent state: 0

Child renders: 0 (only when prop changes)
✅ React.memo prevents unnecessary child re-renders
✅ Child only re-renders when its props actually change
💡 Great for expensive components or components that re-render frequently
3️⃣ useCallback — Memoize Functions

📌 What is useCallback?

useCallback returns a memoized version of a function that only changes if dependencies change. Without useCallback, functions are re-created on every render, causing memoized child components to re-render unnecessarily.

import { useCallback } from 'react';
const memoizedCallback = useCallback(() => {
doSomething(a, b);
}, [a, b]); // Only re-creates when a or b change
📦 useCallback Simulation

Count: 0

Function re-creations: 0
✅ useCallback memoizes functions — same reference across renders
✅ Prevents child components from re-rendering unnecessarily
💡 Always use when passing functions to memoized child components!
4️⃣ useMemo — Memoize Expensive Calculations

📌 What is useMemo?

useMemo memoizes the result of an expensive calculation, only recomputing when dependencies change. Without useMemo, expensive calculations run on EVERY render, slowing down your app.

import { useMemo } from 'react';
const expensiveResult = useMemo(() => {
return expensiveCalculation(data);
}, [data]); // Only recomputes when data changes
🧮 useMemo Simulation (Expensive Calculation)

Number: 0

Result: 0
Calculations: 0 (with useMemo, only when number changes)
✅ useMemo caches calculation results
✅ Only recalculates when dependencies change
💡 Perfect for: filtering large arrays, sorting, complex math, data transformations

6️⃣ Performance Anti-Patterns (What NOT to Do)
  • ❌ Memoizing EVERY component — adds memory overhead, only memoize expensive components
  • ❌ Using useCallback/useMemo for simple values (over-optimization hurts readability)
  • ❌ Creating new objects/arrays in props without memoization → breaks React.memo
  • ❌ Not using keys in lists (causes full re-renders of all items)
  • ❌ Optimizing before measuring — always profile first with React DevTools!
7️⃣ React DevTools Profiler — Find Performance Issues
🔧 How to profile:
1. Install React DevTools browser extension
2. Open DevTools → Components → Profiler tab
3. Click "Start profiling" → Interact with app → "Stop profiling"
4. Analyze which components re-rendered unnecessarily → Optimize those first!

⚠️ Common Performance Mistakes
  • ❌ Using index as key in dynamic lists → re-renders wrong items or loses state. Use unique IDs!
  • ❌ Creating functions inside render that get passed to children → breaks React.memo
  • ❌ Creating new objects/arrays in props: value={{ name: "test" }} → new object every render
  • ❌ Not using React.memo for expensive components that re-render often
  • ❌ Optimizing prematurely — always measure first, then optimize!
💼 Common Interview Questions

❓ What is React.memo?

Higher-order component that memoizes a component — only re-renders when props actually change.

❓ When to use useCallback?

When passing functions to memoized child components to prevent unnecessary re-renders.

❓ useMemo vs useCallback?

useMemo memoizes values/results, useCallback memoizes functions themselves.

❓ When NOT to optimize?

Don't optimize prematurely — only when you see actual performance issues via profiling.
