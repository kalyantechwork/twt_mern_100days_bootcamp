1️⃣ Why Redux? (When Context API isn't enough)

📌 Purpose & Usage

Redux Toolkit is the official, modern way to write Redux logic. It's great for large applications with complex state that needs to be accessed by many components. Redux provides predictable state updates, powerful devtools for debugging, and middleware for async operations.

Where it's used: Large-scale applications like e-commerce platforms, dashboards, social media apps — any app with complex global state that many components need to access and update.

Context API → Good for medium apps (theme, auth)
Redux Toolkit → Great for large apps, complex state, frequent updates, devtools

3️⃣ Redux Core Concepts

🎯 Store

Single source of truth — contains the entire application state

📝 Slice

Piece of state + reducers + actions (all in one)

🔄 Reducer

Function that updates state based on action type

⚡ Action

Plain object describing what happened (type + payload)

📤 Dispatch

Function that sends actions to the store

👁️ Selector

Function that extracts specific data from store

Component → dispatch(action) → Reducer → Updates Store → Components re-render
4️⃣ Creating a Slice (Counter Example)
// counterSlice.js
import { createSlice } from '@reduxjs/toolkit';
const counterSlice = createSlice({
name: 'counter',
initialState: { value: 0 },
reducers: {
increment: (state) => { state.value += 1; },
decrement: (state) => { state.value -= 1; },
reset: (state) => { state.value = 0; }
}
});
export const { increment, decrement, reset } = counterSlice.actions;
export default counterSlice.reducer;

✅ createSlice generates actions and reducers automatically
✅ Immer allows "mutating" syntax — actually immutable under the hood

5️⃣ Configuring the Store
// store.js
import { configureStore } from '@reduxjs/toolkit';
import counterReducer from './features/counter/counterSlice';
import cartReducer from './features/cart/cartSlice';
export const store = configureStore({
reducer: { counter: counterReducer, cart: cartReducer }
});
✅ configureStore automatically sets up DevTools and middleware
✅ Each slice gets its own namespace in the store
✅ Reducer map defines the shape of your Redux state
6️⃣ Provide Store to React App
// main.jsx
import { Provider } from 'react-redux';
import { store } from './store';
ReactDOM.createRoot(document.getElementById('root')).render(
<Provider store={store}><App /></Provider>
);
✅ Provider makes Redux store available to all components
✅ Must wrap the entire app at the root level
💡 Without Provider, useSelector/useDispatch won't work!
7️⃣ Using Redux — useSelector & useDispatch
import { useSelector, useDispatch } from 'react-redux';
import { increment, decrement } from './counterSlice';
function Counter() {
const count = useSelector((state) => state.counter.value);
const dispatch = useDispatch();
return <button onClick={() => dispatch(increment())}>{count}</button>;
}
✅ useSelector — reads data from Redux store (automatically re-renders when data changes)
✅ useDispatch — returns the dispatch function to send actions
💡 This is the standard pattern for all Redux-connected components!
8️⃣ Complete Example: Shopping Cart with Redux
// cartSlice.js
const cartSlice = createSlice({
name: 'cart',
initialState: { items: [], totalQuantity: 0, totalPrice: 0 },
reducers: {
addItem: (state, action) => {
const existing = state.items.find(item => item.id === action.payload.id);
if (existing) existing.quantity++;
else state.items.push({ ...action.payload, quantity: 1 });
state.totalQuantity++;
state.totalPrice += action.payload.price;
},
removeItem: (state, action) => {
const item = state.items.find(item => item.id === action.payload);
if (item) {
state.totalQuantity -= item.quantity;
state.totalPrice -= item.price * item.quantity;
state.items = state.items.filter(item => item.id !== action.payload);
}
}
}
});

🔟 Async Actions — API Calls with createAsyncThunk
export const fetchUsers = createAsyncThunk('users/fetch', async () => {
const response = await fetch('https://jsonplaceholder.typicode.com/users');
return response.json();
});

const usersSlice = createSlice({
name: 'users',
initialState: { users: [], loading: false, error: null },
extraReducers: (builder) => {
builder
.addCase(fetchUsers.pending, (state) => { state.loading = true; })
.addCase(fetchUsers.fulfilled, (state, action) => {
state.users = action.payload;
state.loading = false;
})
.addCase(fetchUsers.rejected, (state, action) => {
state.error = action.error.message;
state.loading = false;
});
}
});
✅ createAsyncThunk handles async request lifecycle (pending/fulfilled/rejected)
✅ Automatic loading and error states management
💡 This is the standard pattern for API calls in Redux Toolkit!

⚠️ Common Redux Mistakes
  • ❌ Directly mutating state in reducers (Redux Toolkit's Immer allows "mutating", but only inside createSlice)
  • ❌ Forgetting to add reducer to store configuration → state slice missing
  • ❌ Not wrapping app with <Provider> → useSelector returns undefined, errors
  • ❌ Putting non-serializable values (functions, promises, symbols) in state
  • ❌ Trying to dispatch actions directly without calling the action creator
💼 Common Interview Questions

❓ What is Redux?

State management library with predictable state updates using actions and reducers.

❓ What is Redux Toolkit?

Official, modern way to write Redux — less boilerplate, includes Immer and Thunk.

❓ What is a slice?

Collection of reducer logic and actions for a single feature.

❓ Redux vs Context?

Redux has DevTools, middleware, better for complex state; Context is simpler for medium apps.
