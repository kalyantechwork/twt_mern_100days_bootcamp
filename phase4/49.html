1️⃣ What is Dark Mode?

📌 Purpose & Usage

Dark mode is a color scheme that uses light text on dark backgrounds. It reduces eye strain in low-light environments, saves battery on OLED screens, and is preferred by over 80% of users at night. Modern apps like YouTube, Twitter, WhatsApp, and Instagram all offer dark mode.

Where it's used: Every modern web application — from social media to productivity tools to e-commerce sites. Users expect this feature.

Light Mode → White background, dark text (default, best for daylight)
Dark Mode → Dark background, light text (easier on eyes at night, saves battery)
2️⃣ Enabling Dark Mode in Tailwind
// tailwind.config.js
export default {
darkMode: 'class', // 'media' follows system preference, 'class' manual toggle
content: ["./index.html", "./src/**/*.{js,ts,jsx,tsx}"],
}
✅ darkMode: 'class' — manual toggle by adding 'dark' class to <html> element
✅ darkMode: 'media' — follows system preference automatically (no toggle needed)
💡 'class' is recommended when you want to give users control over the theme.

4️⃣ Dark Mode Toggle Component (React)
function ThemeToggle() {
const [isDark, setIsDark] = useState(() => localStorage.getItem('theme') === 'dark');
useEffect(() => {
if (isDark) {
document.documentElement.classList.add('dark');
localStorage.setItem('theme', 'dark');
} else {
document.documentElement.classList.remove('dark');
localStorage.setItem('theme', 'light');
}
}, [isDark]);
return <button onClick={() => setIsDark(!isDark)}>{isDark ? '☀️' : '🌙'}</button>;
}

Current mode: Light

✅ useEffect adds/removes 'dark' class on html element
✅ localStorage saves preference — persists after page refresh
💡 This is the complete pattern used in production apps!
5️⃣ Complete Dark Mode Context (Global Theme)
// context/ThemeContext.jsx
const ThemeContext = createContext();
export function ThemeProvider({ children }) {
const [isDark, setIsDark] = useState(() => localStorage.getItem('theme') === 'dark');
useEffect(() => {
if (isDark) document.documentElement.classList.add('dark');
else document.documentElement.classList.remove('dark');
localStorage.setItem('theme', isDark ? 'dark' : 'light');
}, [isDark]);
return (<ThemeContext.Provider value={{ isDark, toggleTheme: () => setIsDark(!isDark) }}>{children}</ThemeContext.Provider>);
}
export const useTheme = () => useContext(ThemeContext);
✅ Context makes theme available to any component without prop drilling
✅ useTheme() custom hook provides clean API for consuming theme
💡 This is the professional pattern for global theme management!

8️⃣ Detecting System Preference (media query)
const getInitialTheme = () => {
const saved = localStorage.getItem('theme');
if (saved) return saved === 'dark';
return window.matchMedia('(prefers-color-scheme: dark)').matches;
};
✅ Respects user's system preference when no saved preference exists
✅ Listens for changes using media query event listener
✅ Provides best user experience — respects system settings!

⚠️ Common Dark Mode Mistakes
  • ❌ Forgetting to add dark: variant to all elements → some elements stay light in dark mode
  • ❌ Not persisting user preference in localStorage → theme resets on refresh
  • ❌ Hardcoding colors instead of using Tailwind's built-in dark mode utilities
  • ❌ Not testing in both modes → contrast issues, invisible elements in dark mode
  • ❌ Abrupt theme changes without transitions → jarring user experience
💼 Common Interview Questions

❓ How does Tailwind handle dark mode?

Using the dark: variant prefix and 'class' or 'media' strategy in tailwind.config.js.

❓ 'class' vs 'media' strategy?

'media' follows system preference automatically; 'class' allows manual toggle by adding 'dark' class to html.

❓ How to persist theme choice?

Save to localStorage on change, load on initial render in useState lazy initialization.

❓ How to detect system preference?

window.matchMedia('(prefers-color-scheme: dark)')
