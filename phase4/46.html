1️⃣ What is Tailwind CSS?

📌 Purpose & Usage

Tailwind CSS is a utility-first CSS framework. Instead of writing custom CSS, you use pre-built classes like bg-blue-500, text-center, p-4 directly in your HTML/JSX. This allows you to build custom designs without ever leaving your HTML.

Where it's used: Thousands of companies including GitHub, Netflix, Shopify, and Stripe. It's the fastest-growing CSS framework in modern web development.

Traditional CSS → Write custom styles in .css file → (slower development, naming conflicts)
Tailwind CSS → Use utility classes directly in JSX → (faster, consistent, no context switching)
2️⃣ Why Tailwind CSS? (Benefits)

⚡ Faster Development

No context switching between HTML and CSS files. Style directly in JSX.

📦 No Naming Fatigue

Never waste time thinking of class names (.card-container-wrapper).

🎨 Consistent Design

Built-in design system with consistent spacing, colors, typography.

📱 Responsive Built-in

Prefix like md:, lg: for responsive design without media queries.

🌙 Dark Mode Ready

Built-in dark mode with dark: prefix.

📦 Small Production Size

PurgeCSS removes unused styles → very small CSS file.

4️⃣ Installation in React (Vite)
# Step 1: Create React project
npm create vite@latest my-app -- --template react
cd my-app

# Step 2: Install Tailwind CSS
npm install -D tailwindcss postcss autoprefixer
npx tailwindcss init -p

# Step 3: Configure tailwind.config.js
export default { content: ["./index.html", "./src/**/*.{js,ts,jsx,tsx}"], ... }

# Step 4: Add to index.css
@tailwind base; @tailwind components; @tailwind utilities;

# Step 5: Run dev server
npm run dev
✅ After installation, you can use Tailwind classes like: className="text-red-500 text-2xl font-bold"
✅ Run npm run dev to see your styled components!
💡 Tailwind works with any React setup — Vite, Create React App, Next.js

⚠️ Common Tailwind Mistakes
  • ❌ Forgetting to configure content in tailwind.config.js → classes don't work
  • ❌ Not adding @tailwind utilities to CSS file
  • ❌ Using arbitrary values incorrectly (e.g., w-[373px] instead of standard spacing)
  • ❌ Overusing custom CSS instead of Tailwind utilities — defeats the purpose!
💼 Common Interview Questions

❓ What is Tailwind CSS?

Utility-first CSS framework for rapidly building custom designs without writing custom CSS.

❓ Benefits over traditional CSS?

Faster development, no naming conflicts, consistent design system, smaller production CSS.

❓ How to make responsive designs?

Use breakpoint prefixes: sm:, md:, lg:, xl:

❓ How does Tailwind purging work?

Removes unused CSS in production based on content configuration — keeps CSS tiny!
