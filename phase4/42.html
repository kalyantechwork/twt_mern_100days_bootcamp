1️⃣ Why Testing is Important

📌 Purpose & Usage

Testing ensures your code works correctly and prevents bugs. Professional developers write tests to catch errors before users see them. Tests give you confidence to refactor code, add new features, and deploy to production without fear of breaking existing functionality.

Where it's used: Every professional software project — from small startups to Google, Facebook, Netflix. Tests are a mandatory part of the development workflow.

Write Code → Write Tests → Run Tests → Find Bugs Early → Fix → Deploy with Confidence
2️⃣ Jest — JavaScript Testing Framework

📌 What is Jest?

Jest is a testing framework created by Facebook. It's a test runner, assertion library, and mocking library all in one. Vite React projects can be configured to use Jest for testing.

// Install Jest and React Testing Library
npm install --save-dev jest @testing-library/react @testing-library/jest-dom

// Run tests
npm test

// Run tests with coverage report
npm test -- --coverage

4️⃣ React Testing Library — Testing Components

📌 What is React Testing Library?

React Testing Library (RTL) helps you test React components by simulating user interactions. It encourages testing from the user's perspective — what users see and do — rather than testing implementation details.

// Button.test.jsx
import { render, screen, fireEvent } from '@testing-library/react';
import Button from './Button';

test('button calls onClick when clicked', () => {
const handleClick = jest.fn();
render(<Button onClick={handleClick}>Click Me</Button>);
fireEvent.click(screen.getByText('Click Me'));
expect(handleClick).toHaveBeenCalledTimes(1);
});

7️⃣ Mocking with jest.fn()
// Mock a function to track calls
const mockFunction = jest.fn();
mockFunction('hello');
expect(mockFunction).toHaveBeenCalledWith('hello');

// Mock API responses
jest.mock('axios');
axios.get.mockResolvedValue({ data: { name: 'Kalyan' } });
✅ jest.fn() — creates a mock function that tracks calls
✅ toHaveBeenCalledWith() — verifies function was called with specific args
✅ mockResolvedValue() — mock async function that resolves
💡 Mocking isolates code and makes tests predictable!

⚠️ Common Testing Mistakes
  • ❌ Testing implementation details instead of user behavior (e.g., testing state directly)
  • ❌ Not using waitFor for async operations → flaky tests
  • ❌ Forgetting to mock API calls → real network requests in tests (slow and unreliable)
  • ❌ Using getByText when element may not exist → use queryByText
  • ❌ Not cleaning up mocks between tests → test pollution
💼 Common Interview Questions

❓ What is Jest?

JavaScript testing framework by Facebook — test runner and assertion library.

❓ What is React Testing Library?

Library for testing React components by simulating user interactions.

❓ Difference between getBy and queryBy?

getBy throws error if not found, queryBy returns null.

❓ What is mocking?

Replacing real functions with fake ones to control test behavior.
