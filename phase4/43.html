1️⃣ What are Nested Routes?

📌 Purpose & Usage

Nested Routes allow you to have a layout component that contains child routes. This is perfect for dashboards, admin panels, and any UI where the outer structure (header, sidebar, footer) stays constant while the inner content changes.

Where it's used: Dashboards, e-commerce checkouts, settings pages, any app with shared UI elements across multiple pages.

/dashboard → Layout (Header + Sidebar + Outlet) ├── /dashboard/profile → Profile Page ├── /dashboard/settings → Settings Page └── /dashboard/analytics → Analytics Page
2️⃣ Nested Routes Syntax (Outlet Component)
// DashboardLayout.jsx (parent layout)
import { Outlet } from 'react-router-dom';
function DashboardLayout() {
return (<div><h1>Dashboard</h1><nav><Link to="/dashboard/profile">Profile</Link></nav><Outlet /></div>);
}

// App.jsx — route configuration
<Route path="/dashboard" element={<DashboardLayout />}>
<Route index element={<DashboardHome />} />
<Route path="profile" element={<Profile />} />
<Route path="settings" element={<Settings />} />
</Route>

👤 Profile
⚙️ Settings
Welcome to Dashboard Home!
✅ Outlet component is where child routes render
✅ Layout (tabs) stays the same — only content changes
💡 This pattern is used by every dashboard in production!
3️⃣ Index Routes (Default Child Route)
<Route path="/dashboard" element={<DashboardLayout />}>
<Route index element={<DashboardHome />} /> {/* Shows at /dashboard */}
<Route path="profile" element={<Profile />} /> {/* Shows at /dashboard/profile */}
</Route>
✅ index route renders when parent path matches exactly
✅ No need to specify path attribute for index routes
💡 Example: /dashboard renders DashboardHome, /dashboard/profile renders Profile
4️⃣ Protected Routes (Authentication Required)

📌 What are Protected Routes?

Protected Routes restrict access to certain pages unless the user is authenticated. This is essential for dashboards, admin panels, user settings, and any page that requires login.

// ProtectedRoute.jsx
function ProtectedRoute({ children }) {
const { user } = useAuth();
if (!user) return <Navigate to="/login" replace />;
return children;
}

// Usage in App.jsx
<Route path="/dashboard" element={
<ProtectedRoute><DashboardLayout /></ProtectedRoute>
}>...</Route>

✅ Protected routes check authentication before rendering
✅ Unauthenticated users are redirected to /login
💡 This pattern is used in EVERY production app with authentication!
5️⃣ Navigate Component (Programmatic Redirect)
import { Navigate } from 'react-router-dom';
function LoginPage() {
const [shouldRedirect, setShouldRedirect] = useState(false);
if (shouldRedirect) return <Navigate to="/dashboard" />;
return <button onClick={() => setShouldRedirect(true)}>Login</button>;
}
✅ Navigate component immediately redirects when rendered
✅ Use 'replace' prop to replace history entry: <Navigate to="/" replace />
✅ Great for conditional redirects based on state
6️⃣ useNavigate Hook (Programmatic Navigation)
import { useNavigate } from 'react-router-dom';
function LoginPage() {
const navigate = useNavigate();
return <button onClick={() => navigate('/dashboard')}>Login</button>;
}
✅ useNavigate() returns a function for programmatic navigation
✅ navigate('/path') — go to route
✅ navigate(-1) — go back, navigate(1) — go forward
💡 Preferred over Navigate component for event handlers
7️⃣ Layout Routes — Shared UI Components
// RootLayout.jsx (shared header/footer for all pages)
function RootLayout() {
return (<><Header /><Outlet /><Footer /></>);
}

// App.jsx — wrap multiple routes with shared layout
<Route element={<RootLayout />}>
<Route path="/" element={<Home />} />
<Route path="/about" element={<About />} />
</Route>
✅ Layout routes eliminate duplicate code across pages
✅ Header and Footer appear on every route inside the layout
💡 This is the standard pattern for multi-page apps!

⚠️ Common Router Mistakes
  • ❌ Forgetting <Outlet /> in parent route → child routes never render
  • ❌ Not handling redirect after login → user stays on login page
  • ❌ Using Navigate inside conditional that re-renders → infinite redirect loops
  • ❌ Not using absolute paths for nested routes (missing leading slash)
  • ❌ Forgetting to protect routes that should be private → security risk
💼 Common Interview Questions

❓ What are nested routes?

Routes that render inside parent layout using Outlet component — perfect for dashboards.

❓ How to protect routes?

Create a ProtectedRoute component that checks auth and redirects to login using Navigate.

❓ What is Outlet?

Component where child routes render in a parent route — the "slot" for nested content.

❓ How to redirect programmatically?

useNavigate() hook (for event handlers) or <Navigate /> component (for conditional rendering).
