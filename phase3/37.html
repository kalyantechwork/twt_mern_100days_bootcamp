1️⃣ What is React Router? (Navigation in React)

📌 Purpose & Usage

React Router is the standard library for routing in React apps. It allows you to create multi-page experiences without actually reloading the page — true Single Page Application (SPA) navigation. React Router synchronizes your UI with the URL, making bookmarking and sharing links work naturally.

Where it's used: Every multi-page React app — dashboards, e-commerce sites, blogs, admin panels, portfolio websites, and any app with multiple views/screens.

URL changes → React Router matches route → Renders corresponding component → No page reload

# Install React Router DOM (v6 is current)
npm install react-router-dom

# Or with yarn
yarn add react-router-dom
✅ react-router-dom version 6.x is the latest stable version
✅ After installation, you can import BrowserRouter, Routes, Route, Link, etc.
💡 Always check package.json to confirm installation
3️⃣ Basic Setup — BrowserRouter
// main.jsx or index.js (entry point)
import { BrowserRouter } from 'react-router-dom';
import App from './App';

ReactDOM.createRoot(document.getElementById('root')).render(
<BrowserRouter>
<App />
</BrowserRouter>
);
✅ BrowserRouter wraps your entire app — enables routing functionality across all components
✅ Alternative routers: HashRouter (for static file hosting), MemoryRouter (for testing)
💡 Without BrowserRouter, useNavigate() and Link won't work!
4️⃣ Defining Routes — Routes & Route
// App.jsx
import { Routes, Route } from 'react-router-dom';
import Home from './pages/Home';
import About from './pages/About';
import Contact from './pages/Contact';

function App() {
return (<Routes>
<Route path="/" element={<Home />} />
<Route path="/about" element={<About />} />
<Route path="/contact" element={<Contact />} />
</Routes>);
}

📖 About
📞 Contact
🏠 Welcome to the Home Page!
✅ Routes component wraps all route definitions
✅ Route component maps URL path to a component
✅ path="/" is the home route
💡 Order matters — more specific routes should come before general ones
5️⃣ Link & NavLink — Navigation Components
import { Link, NavLink } from 'react-router-dom';

// Basic Link (like <a> tag but no reload)
<Link to="/about">About</Link>

// NavLink (adds 'active' class when route matches)
<NavLink to="/" className={({ isActive }) => isActive ? 'active' : ''}>Home</NavLink>
✅ Link component prevents full page reload — client-side navigation!
✅ NavLink adds an "active" class automatically when route matches (great for styling)
✅ Use "end" prop on NavLink for exact matching: <NavLink to="/" end>Home</NavLink>
💡 Never use <a href> in React Router apps — it causes full page reloads!
6️⃣ Dynamic Routing — URL Parameters (:id)

📌 Dynamic Routes

Dynamic routes use URL parameters (like :id) to create flexible paths. The useParams() hook extracts these values, allowing you to show different content based on the URL.

// Route definition
<Route path="/users/:userId" element={<UserProfile />} />

// UserProfile component
import { useParams } from 'react-router-dom';
function UserProfile() {
const { userId } = useParams();
return <div>Showing profile for user: {userId}</div>;
}

👤 User 2
👤 User 3
Click a user to see profile
✅ useParams() extracts URL parameters like :id, :userId, etc.
✅ Perfect for product pages, user profiles, blog posts
💡 Example URL: /products/123 → useParams().productId = "123"
7️⃣ Programmatic Navigation (useNavigate)

📌 Navigate Programmatically

The useNavigate() hook allows you to navigate to different routes from within functions — perfect for redirecting after form submissions, logins, or button clicks.

import { useNavigate } from 'react-router-dom';

function LoginPage() {
const navigate = useNavigate();
const handleLogin = () => {
// After successful login, redirect to dashboard
navigate('/dashboard');
};
return <button onClick={handleLogin}>Login</button>;
}
🎯 Programmatic Navigation Demo
Click the button — it will simulate navigation
✅ useNavigate() returns a function to navigate programmatically
✅ navigate('/path') — goes to specified route
✅ navigate(-1) — go back, navigate(1) — go forward
💡 Essential for login redirects, form submissions, and error handling
8️⃣ Nested Routes — Layouts & Sub-Pages
// Dashboard layout component
function Dashboard() {
return (<div><h1>Dashboard</h1><Outlet /></div>);
}

// Routes configuration
<Route path="/dashboard" element={<Dashboard />}>
<Route index element={<DashboardHome />} />
<Route path="profile" element={<Profile />} />
<Route path="settings" element={<Settings />} />
</Route>
✅ Outlet component is where child routes render
✅ Nested routes share the same layout
💡 Perfect for dashboards, admin panels, and multi-step forms
9️⃣ 404 Not Found Page
<Route path="*" element={<NotFound />} />

function NotFound() {
return <div>404 - Page Not Found</div>;
}
✅ path="*" catches any undefined routes
✅ Always add this as the last route
💡 Provides better user experience when users visit non-existent pages

⚠️ Common React Router Mistakes
  • ❌ Forgetting to wrap app with <BrowserRouter> → routes won't work, useNavigate() throws error
  • ❌ Using <a href> instead of <Link to> → causes full page reloads (defeats SPA purpose)
  • ❌ Path order: specific paths before general paths (e.g., /users/:id before /users/new)
  • ❌ Forgetting to import Routes and Route components → component not found errors
  • ❌ Not adding trailing slash handling → inconsistent matching
💼 Common Interview Questions

❓ What is React Router?

Standard routing library for React — enables navigation without page reloads in SPAs.

❓ Difference between Link and NavLink?

NavLink adds 'active' class when route matches — ideal for navigation bars.

❓ How to get URL parameters?

Use the useParams() hook inside the component.

❓ How to navigate programmatically?

Use the useNavigate() hook, then call navigate('/path').
