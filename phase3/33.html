1️⃣ What are Controlled Components?

📌 Purpose & Usage

Controlled components are form elements (input, textarea, select) whose value is controlled by React state. Instead of the DOM maintaining the value, React state is the "single source of truth". When the user types, an onChange event updates the state, which then updates the input value.

Where it's used: Every form in React — login forms, registration forms, search bars, comment boxes, settings panels, data entry forms.

State → Input Value → User Types → onChange → Updates State → Input Shows New Value
2️⃣ Simple Controlled Input Example
function SimpleForm() {
const [name, setName] = useState("");
return (<div>
<input value={name} onChange={(e) => setName(e.target.value)} placeholder="Enter your name"/>
<p>You typed: {name}</p>
</div>);
}
📝 Controlled Input:

You typed:

✅ value is controlled by state (simulated here)
✅ onChange updates state on every keystroke
✅ This is called "two-way binding" — state → input, input → state
3️⃣ Handling Multiple Inputs (One State Object)
function RegistrationForm() {
const [form, setForm] = useState({ name: "", email: "", password: "" });
const handleChange = (e) => {
setForm({ ...form, [e.target.name]: e.target.value });
};
return (<div>
<input name="name" value={form.name} onChange={handleChange} placeholder="Name"/>
<input name="email" value={form.email} onChange={handleChange} placeholder="Email"/>
</div>);
}
📋 Registration Form (Controlled):
Live Preview:


✅ One state object for all form fields
✅ One handleChange function using computed property name: [e.target.name]
✅ Spread operator (...) preserves other fields when updating

6️⃣ Checkbox (Controlled)
function NewsletterSignup() {
const [isSubscribed, setIsSubscribed] = useState(false);
return (<label>
<input type="checkbox" checked={isSubscribed} onChange={(e) => setIsSubscribed(e.target.checked)}/>
Subscribe to newsletter
</label>);
}

Status: Not subscribed

✅ Checkbox uses 'checked' prop (not 'value')
✅ onChange reads e.target.checked (boolean)
✅ Great for toggles, agreements, preferences
7️⃣ Form Submission (preventDefault)
function LoginForm() {
const [email, setEmail] = useState("");
const handleSubmit = (e) => {
e.preventDefault(); // CRITICAL: prevents page refresh
alert(`Logging in with ${email}`);
};
return (<form onSubmit={handleSubmit}>...</form>);
}
🔐 Login Form (No Page Reload):

✅ e.preventDefault() is CRITICAL — prevents page refresh!
✅ Without it, React apps would reload and lose state
✅ Always use onSubmit on form, not onClick on button
8️⃣ Basic Form Validation
function SignupForm() {
const [email, setEmail] = useState("");
const [error, setError] = useState("");
const handleSubmit = (e) => {
e.preventDefault();
if (!email.includes("@")) setError("Valid email required");
else setError("");
};
return (<form onSubmit={handleSubmit}>...</form>);
}
✅ Signup Form (with Validation):

✅ Validation happens before submission
✅ Error messages shown conditionally (&& operator)
✅ User gets immediate feedback

⚠️ Common Beginner Mistakes
  • ❌ Forgetting value={state} on input → becomes uncontrolled (React loses control)
  • ❌ Not using e.preventDefault() on form submit → page refreshes, state lost
  • ❌ Mutating state directly: form.name = value ❌ → always use setter function
  • ❌ Forgetting to spread existing state when updating object: setForm({ [name]: value }) ❌ loses other fields
  • ❌ Using onClick on submit button instead of onSubmit on form → Enter key won't work
💼 Common Interview Questions

❓ What is a controlled component?

Form element whose value is controlled by React state, with onChange handler updating state.

❓ Why use controlled components?

For validation, dynamic inputs, single source of truth, and conditional UI updates.

❓ How to handle multiple inputs?

Use one state object and a single handleChange with computed property names [e.target.name].

❓ What does preventDefault() do?

Prevents default browser behavior — essential for preventing page refresh on form submit.
