1️⃣ What is useEffect? (Side Effects in React)

📌 Purpose & Usage

useEffect is a React Hook that lets you perform side effects in functional components. Side effects include: API calls, DOM manipulation, timers, subscriptions, localStorage, and anything that interacts with the outside world. useEffect runs after React updates the DOM.

Where it's used: Every React app — fetching data, updating page title, setting up event listeners, starting timers, saving to localStorage, and cleaning up resources.

Component Renders → useEffect Runs → Side Effect Happens → Component Re-renders (if state changes)
2️⃣ Basic Syntax — Three Patterns
// Pattern 1: Run after EVERY render
useEffect(() => { console.log("Runs every render"); });

// Pattern 2: Run ONCE (on mount) — empty dependency array
useEffect(() => { console.log("Runs once when component mounts"); }, []);

// Pattern 3: Run when dependencies change
useEffect(() => { console.log(`Count: ${count}`); }, [count]);
✅ useEffect replaces lifecycle methods: componentDidMount, componentDidUpdate, componentWillUnmount
✅ No dependency array → runs after every render (rarely used)
✅ Empty array [] → runs once after initial mount (MOST COMMON for API calls)
✅ With dependencies → runs when any dependency changes (search, filters)
3️⃣ Pattern 1: Run After Every Render (No Dependency Array)
useEffect(() => {
console.log("Component rendered or state updated!");
document.title = `Count: ${count}`;
});

4️⃣ Pattern 2: Run Once (Empty Dependency Array — [])

📌 Most Common Pattern — API Calls

Empty dependency array means the effect runs only once, after the component mounts. This is perfect for fetching initial data from an API.

useEffect(() => {
console.log("Component mounted — fetching data...");
fetchUsers();
}, []); // Empty array = run once

Effect hasn't run yet
✅ useEffect with [] runs ONLY ONCE when component first appears
✅ Perfect for: API calls, initial data fetch, subscriptions
💡 This is the most common useEffect pattern in React apps!
5️⃣ Pattern 3: Run When Dependencies Change

📌 Reactive Effects

When you include dependencies, the effect re-runs whenever any of those values change. Perfect for search inputs, filters, or any reactive data.

useEffect(() => {
console.log(`Searching for: ${query}`);
fetchResults(query);
}, [query]); // Re-runs when 'query' changes

✅ Dependencies can be state, props, or any variable
💡 Perfect for search inputs, filters, pagination, and form validation
6️⃣ Cleanup Function — Preventing Memory Leaks

📌 What is Cleanup?

The cleanup function runs when the component unmounts or before the next effect runs. It's essential for cleaning up intervals, event listeners, and subscriptions to prevent memory leaks.

useEffect(() => {
const interval = setInterval(() => {
setSeconds(s => s + 1);
}, 1000);
return () => clearInterval(interval); // Cleanup!
}, []);

7️⃣ Real Example — Fetching API Data with useEffect
useEffect(() => {
const fetchUsers = async () => {
const response = await fetch('https://jsonplaceholder.typicode.com/users');
const data = await response.json();
setUsers(data);
};
fetchUsers();
}, []); // Runs once when component mounts

📊 Page Title Update

💾 Save to localStorage

useEffect(() => { localStorage.setItem("theme", theme); }, [theme]);

🖱️ Event Listener

useEffect(() => { window.addEventListener("scroll", handle); return () => window.removeEventListener(...); }, []);

⏱️ Timer/Interval

useEffect(() => { const id = setInterval(...); return () => clearInterval(id); }, []);

⚠️ Common useEffect Mistakes
  • ❌ Missing dependencies → causes stale closures (always include used state/props in dependency array)
  • ❌ Forgetting cleanup for intervals/event listeners → memory leaks, use return () => cleanup
  • ❌ Making async function directly in useEffect — wrap inside function: useEffect(() => { async function fetch() {...}; fetch(); }, [])
  • ❌ Updating state in useEffect without dependency array → infinite loop
💼 Common Interview Questions

❓ What is useEffect?

Hook for performing side effects in functional components (API calls, DOM, timers).

❓ What does empty dependency array [] do?

Effect runs only once after initial mount — perfect for API calls.

❓ When to use cleanup function?

For subscriptions, intervals, event listeners to prevent memory leaks.

❓ Can useEffect be async?

No directly, but you can define an async function inside and call it.
