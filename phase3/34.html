1️⃣ useState — Quick Review

📌 What is useState?

useState is a React Hook that lets you add state to functional components. It returns an array with two values: the current state and a function to update it. When state changes, React re-renders the component.

useState(initialValue) → [state, setStateFunction] → Component re-renders on state change
2️⃣ Functional Updates — When New State Depends on Previous

📌 Why Functional Updates?

When updating state based on previous state, use the functional form setCount(prev => prev + 1). This ensures you're working with the most recent state, especially important when multiple updates happen in the same event or when state updates are batched.

// ❌ Wrong for multiple updates in same event
setCount(count + 1); setCount(count + 1); // both use same old value

// ✅ Correct: functional update
setCount(prev => prev + 1); setCount(prev => prev + 1); // each gets latest

3️⃣ State with Objects — Always Spread!

📌 Updating Object State

When updating object state, you must spread (... ) existing properties to avoid losing data. Never mutate state directly — always use the setter function with a new object.

const [user, setUser] = useState({ name: "Kalyan", age: 25, city: "Hyderabad" });
// ✅ Correct — spread existing properties
setUser({ ...user, age: 26 });
// ❌ Wrong — overwrites entire object
setUser({ age: 26 }); // name and city are lost!
👤 User Profile (Object State)

Name: Kalyan | Age: 25 | City: Hyderabad

✅ Spread operator preserves other properties when updating one field
✅ Never mutate state directly — always use setter function
💡 This is one of the most common mistakes in React!
4️⃣ State with Arrays — Common Operations
const [items, setItems] = useState(["Apple", "Mango"]);
// Add to end: setItems([...items, "Orange"]);
// Add to beginning: setItems(["Banana", ...items]);
// Remove: setItems(items.filter((_, i) => i !== index));
// Update: setItems(items.map((item, i) => i === index ? "New" : item));
📋 Todo List (Array State)
✅ Spread to add items: [...items, newItem]
✅ filter() to remove items
✅ map() to update items
💡 Always create new arrays — never mutate directly!
5️⃣ Multiple State Variables vs One Object

✅ Separate States (Recommended)

const [name, setName] = useState("");
const [age, setAge] = useState(0);
Easier to update, simpler to read, fewer bugs

⚠️ Single Object State

const [form, setForm] = useState({});
Good for related fields, must spread on updates

6️⃣ Lazy Initial State (For Expensive Computations)
// ❌ Runs on every render (expensive if complex)
const [data, setData] = useState(expensiveComputation());

// ✅ Runs only once (lazy initialization)
const [data, setData] = useState(() => expensiveComputation());
✅ Lazy initialization prevents expensive calculations on every render
✅ Pass a function: useState(() => heavyCalculation())
✅ The function runs only once when component mounts
💡 Useful for reading from localStorage, complex math, data transformation
7️⃣ Rules of Hooks (Must Follow!)
// ❌ Wrong — conditional hook
if (isLoggedIn) { const [user, setUser] = useState(null); }

// ✅ Correct — always called, condition inside
const [user, setUser] = useState(null);
if (isLoggedIn) { /* use user */ }
  • ✅ Only call Hooks at the top level (not inside loops, conditions, or nested functions)
  • ✅ Only call Hooks from React functions (components or custom Hooks)
  • ✅ Hooks must be called in the same order every render
  • 💡 Violating these rules causes bugs and unpredictable behavior!

⚠️ Common useState Mistakes
  • ❌ Direct mutation: state.count++ ❌ → always use setter function: setCount(count + 1) ✅
  • ❌ Not using functional update when state depends on previous value → use setCount(prev => prev + 1) ✅
  • ❌ Forgetting spread operator with object state → losing properties → use { ...state, key: value } ✅
  • ❌ Using array index as key for list (causes issues with reordering) → use unique ID ✅
💼 Common Interview Questions

❓ What does useState return?

An array with current state value and a setter function.

❓ Why use functional updates?

To ensure you're updating based on the latest state, especially with multiple updates.

❓ Can useState be used in loops/conditions?

No — violates Rules of Hooks. Hooks must be called at top level.

❓ How to update object state?

Spread existing properties: setState({ ...state, key: newValue })
