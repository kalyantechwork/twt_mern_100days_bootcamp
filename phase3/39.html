1️⃣ What are Custom Hooks? (Reusable Logic)

📌 Purpose & Usage

Custom Hooks are JavaScript functions that start with "use" and can call other React hooks. They let you extract component logic into reusable functions, making your code cleaner, more maintainable, and shareable across multiple components.

Where it's used: Form handling, API calls, localStorage persistence, window size tracking, online status detection, timer logic — any repeating logic that uses React hooks.

Duplicate Logic in multiple components → Extract to Custom Hook → One source of truth → Reuse everywhere
2️⃣ Why Custom Hooks? (Benefits)

♻️ Reusability

Write logic once, use in multiple components without duplication

🧹 Clean Code

Components become smaller and focused only on UI rendering

🧪 Testability

Easier to test isolated logic separate from UI components

👥 Team Collaboration

Share common logic across the team via a hooks library

3️⃣ Rules for Custom Hooks
  • ✅ Must start with "use" (e.g., useFetch, useLocalStorage, useWindowSize)
  • ✅ Can call other hooks inside (useState, useEffect, useContext, other custom hooks)
  • ✅ Only call hooks at the top level (not inside conditions, loops, or nested functions)
  • ✅ Two components using same custom hook do NOT share state — each gets isolated state
✅ Naming: starts with "use" — React uses this to check hook rules
✅ Can use other hooks — this is what makes them powerful!
✅ Top level only — no conditional hook calls inside custom hooks
✅ Isolated state — each component using the hook gets its own state
4️⃣ Example 1: useWindowSize (Track Window Dimensions)
function useWindowSize() {
const [windowSize, setWindowSize] = useState({ width: window.innerWidth, height: window.innerHeight });
useEffect(() => {
const handleResize = () => setWindowSize({ width: window.innerWidth, height: window.innerHeight });
window.addEventListener('resize', handleResize);
return () => window.removeEventListener('resize', handleResize);
}, []);
return windowSize;
}

5️⃣ Example 2: useLocalStorage (Persist State to localStorage)
function useLocalStorage(key, initialValue) {
const [storedValue, setStoredValue] = useState(() => {
const item = localStorage.getItem(key);
return item ? JSON.parse(item) : initialValue;
});
useEffect(() => localStorage.setItem(key, JSON.stringify(storedValue)), [key, storedValue]);
return [storedValue, setStoredValue];
}

✅ useEffect automatically saves to localStorage when value changes

6️⃣ Example 3: useFetch (API Calls Made Easy)
function useFetch(url) {
const [data, setData] = useState(null);
const [loading, setLoading] = useState(true);
const [error, setError] = useState(null);
useEffect(() => {
const fetchData = async () => {
try { const res = await fetch(url); const result = await res.json(); setData(result); }
catch (err) { setError(err.message); } finally { setLoading(false); }
}; fetchData();
}, [url]);
return { data, loading, error };
}

7️⃣ Example 4: useToggle (Boolean Toggle Helper)
function useToggle(initialValue = false) {
const [value, setValue] = useState(initialValue);
const toggle = () => setValue(prev => !prev);
return [value, toggle];
}

⚠️ Common Custom Hook Mistakes
  • ❌ Not starting hook name with "use" → React can't enforce hook rules, ESLint won't check
  • ❌ Calling hooks conditionally inside custom hook → violates Rules of Hooks
  • ❌ Forgetting to return values from the hook → component can't use the hook's data
  • ❌ Using custom hooks for static values (no state/effects needed) → just use regular functions
  • ❌ Creating infinite loops by updating state in useEffect without proper dependencies
💼 Common Interview Questions

❓ What are custom hooks?

Functions that start with "use" and encapsulate reusable stateful logic using React hooks.

❓ Why create custom hooks?

To reuse logic across multiple components, keep components clean, and improve testability.

❓ Can custom hooks call other hooks?

Yes! They can call useState, useEffect, useContext, and other custom hooks.

❓ Do custom hooks share state between components?

No — each component using the hook gets its own isolated state.
