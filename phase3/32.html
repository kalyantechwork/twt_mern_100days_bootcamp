1️⃣ Why Do We Need Lists in React?

📌 Purpose & Usage

In real applications, data comes as arrays (users, products, todos, comments). React allows you to render arrays as lists using the map() function, transforming each array item into a JSX element.

Where it's used: Product listings, user directories, todo lists, comment sections, shopping carts, search results — every dynamic list in a React app.

Array of Data → map() → Array of JSX elements → Rendered to UI
2️⃣ Basic List Rendering — Using map()
function FruitList() {
const fruits = ["Apple", "Mango", "Orange", "Banana"];
return (<ul>
{fruits.map((fruit) => (<li>{fruit}</li>))}
</ul>);
}
🍎 Rendered List (map()):
  • Apple
  • Mango
  • Orange
  • Banana
✅ map() transforms array into array of JSX elements
✅ Each item becomes an <li> element
⚠️ Without keys, React shows a warning!
3️⃣ What are Keys? (Very Important!)

📌 What are Keys?

Keys are special string attributes that help React identify which items have changed, been added, or been removed. Keys must be unique among their siblings. They are essential for React's reconciliation algorithm to update the DOM efficiently.

// ❌ Without key (warning in console)
{fruits.map((fruit) => (<li>{fruit}</li>))}

// ✅ With key (React happy!)
{fruits.map((fruit, index) => (<li key={index}>{fruit}</li>))}

// ✅ Best practice: using unique ID
{users.map((user) => (<li key={user.id}>{user.name}</li>))}
🔑 What are Keys?
✅ Keys help React identify which items changed, added, or removed
✅ React uses keys to optimize re-rendering (only update changed items)
✅ Keys must be UNIQUE among siblings
⚠️ React shows a warning in console when keys are missing!
4️⃣ Why Keys are Important (Performance)

✅ Efficient Updates

Keys help React identify which items changed. Without keys, React re-renders the entire list.

✅ Correct State Preservation

Keys preserve component state when items are reordered (e.g., checkboxes stay checked).

✅ Better Performance

React skips re-rendering items with the same key — only updates changed ones.

Without Keys → Re-render everything → Slow & buggy
With Keys → Update only changed items → Fast & correct
5️⃣ Choosing the Right Key — Best Practices
Key SourceGood/BadExample
Unique ID from database✅ Best<li key={user.id}>{user.name}</li>
Index (map second parameter)⚠️ Only for static lists (no reorder)<li key={index}>{item}</li>
Random value (Math.random)❌ Bad — changes every render<li key={Math.random()}>{item}</li>
6️⃣ Rendering List of Objects (Real Data)
function ProductList() {
const products = [
{ id: 1, name: "Laptop", price: 50000 },

7️⃣ Dynamic List — Adding and Removing Items
function TodoList() {
const [todos, setTodos] = useState([]);
const addTodo = () => {
setTodos([...todos, { id: Date.now(), text: input }]);
};
return todos.map((todo) => (
<li key={todo.id}>{todo.text}<button onClick={() => removeTodo(todo.id)}>Delete</button></li>
));
}
📝 Todo List (with Add/Delete)
✅ Each todo has unique ID (Date.now()) as key
✅ Keys remain stable even when items are added/deleted
💡 This is how real todo apps work!
8️⃣ Common Key Mistakes to Avoid
  • ❌ Using array index as key when list can reorder (items shift, state gets mixed up — checkboxes, form inputs)
  • ❌ Using random keys like Math.random() or Date.now() (creates new key every render → defeats purpose)
  • ❌ Forgetting keys completely (React shows console warning and performance suffers)
  • ❌ Using duplicate keys (causes rendering bugs — React can't distinguish items)
9️⃣ Keys Only Need to be Unique Among Siblings
// ✅ This is fine — keys are unique within each list
function App() {
return (<>
<ul>{fruits.map(f => <li key={f}>{f}</li>)}</ul>
<ul>{vegetables.map(v => <li key={v}>{v}</li>)}</ul>
</>);
}
🔑 Remember: Keys only need to be unique among their siblings, not globally across the whole app. Same key can be used in different lists!

💼 Common Interview Questions

❓ What are keys in React?

Special attributes that help React identify which items changed in a list for efficient updates.

❓ Why do we need keys?

For efficient DOM updates, correct state preservation, and better performance when lists change.

❓ Can we use index as key?

Only for static lists that don't reorder, add, or delete. Avoid for dynamic lists.

❓ What happens without keys?

React shows a console warning and may have performance issues with list updates.
