1️⃣ What is Conditional Rendering?

📌 Purpose & Usage

Conditional rendering in React means showing different UI elements based on certain conditions (like user login status, data loading state, or error messages). Just like if/else in JavaScript, but adapted for JSX. This is essential for building dynamic, user-friendly applications.

Where it's used: Login/Logout buttons, loading spinners, error messages, user role-based dashboards, form validation feedback, feature toggles, and dark/light mode.

Condition → True → Show Component A
Condition → False → Show Component B
2️⃣ Method 1: if/else (Outside return statement)

📌 When to Use if/else

Use if/else when you have complex logic or multiple conditions. The condition must be evaluated outside the JSX return statement, and you store the result in a variable.

function UserGreeting({ isLoggedIn }) {
let message;
if (isLoggedIn) {
message = <h1>Welcome back, User!</h1>;
} else {
message = <h1>Please log in to continue</h1>;
}
return <div>{message}</div>;
}

3️⃣ Method 2: Ternary Operator (condition ? true : false)

📌 When to Use Ternary

The ternary operator is the most popular method in React. Use it when you have two simple options (true/false). It's concise and can be used directly inside JSX.

function UserGreeting({ isLoggedIn }) {
return (<div>
{isLoggedIn ? (
<h1>✨ Welcome back, User!</h1>
) : (
<h1>🔐 Please log in</h1>
)}
</div>);
}

4️⃣ Method 3: && Operator (Show or Nothing)

📌 When to Use &&

Use the && operator when you want to render something OR nothing (no else case). It's perfect for conditional badges, error messages, notifications, and loading spinners.

function Notification({ hasMessage }) {
return (<div>
{hasMessage && <p>🔔 You have a new message!</p>}
<h1>Dashboard</h1>
</div>);
}

⚠️ Warning: Don't use && with numbers (0 is falsy and renders 0!)

⚠️ Common Beginner Mistakes
  • ❌ Using if/else directly inside JSX return → not allowed! Move logic outside return or use ternary.
  • ❌ Forgetting curly braces around ternary: {condition ? true : false} — braces are required.
  • ❌ Using && with numbers: {count && <p>You have {count} items</p>} — if count is 0, it renders "0". Fix: {count > 0 && ...}
  • ❌ Too many nested ternaries → hard to read. Extract logic to a variable or use if/else.
💼 Common Interview Questions

❓ How to conditionally render in React?

Using if/else (outside JSX), ternary operator, or && operator.

❓ Can we use if/else inside JSX?

No, but you can use ternary or extract logic outside return.

❓ When to use && operator?

When you want to render something OR nothing (no else case).

❓ What is the most common conditional rendering method?

Ternary operator — it's concise and works directly inside JSX.
