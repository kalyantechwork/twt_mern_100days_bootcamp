1️⃣ What are API Calls in React?

📌 Purpose & Usage

API calls allow your React app to communicate with backend servers to fetch, create, update, or delete data. Real-world apps always use APIs to get data from databases, external services, or your own backend. Without APIs, React apps can only show static data.

Where it's used: User profiles, product listings, weather apps, social media feeds, e-commerce carts, search results — every dynamic web app.

React Component → useEffect (on mount) → fetch/axios → API Server → Data → Update State → Re-render UI
2️⃣ Fetch API — Basic GET Request
useEffect(() => {
fetch('https://jsonplaceholder.typicode.com/users')
.then(response => response.json())
.then(data => setUsers(data))
.catch(err => setError(err.message))
.finally(() => setLoading(false));
}, []);
📡 Fetch API GET Demo (JSONPlaceholder)
✅ fetch() returns a Promise
✅ response.json() parses JSON from response
✅ useEffect with [] ensures API call happens once on mount
💡 This is the standard pattern for data fetching in React!
3️⃣ Async/Await — Cleaner Syntax
useEffect(() => {
const fetchData = async () => {
try {
const response = await fetch('https://api.example.com/users');
if (!response.ok) throw new Error('Network response was not ok');
const data = await response.json();
setUsers(data);
} catch (err) {
setError(err.message);
} finally {
setLoading(false);
}
};
fetchData();
}, []);
✅ async/await makes async code look synchronous — easier to read!
✅ try/catch handles errors gracefully
✅ finally block runs regardless of success/failure — perfect for loading state
💡 This is the recommended pattern for modern React apps
4️⃣ Loading & Error States — Professional UX

📌 Professional Data Fetching

Always implement three states: loading (show spinner), error (show user-friendly message), and success (display data). This provides clear feedback to users during network requests.

if (loading) return <div>⏳ Loading...</div>;
if (error) return <div>⚠️ {error}</div>;
return <div>{data.map(item => ...)}</div>;
🎯 API States Demo (Loading → Success → Error)
Click a button to see state change
✅ Loading state: shows spinner or "Loading..." message
✅ Error state: shows user-friendly error message
✅ Success state: displays fetched data
💡 Users appreciate knowing what's happening during network requests!
5️⃣ POST Request — Sending Data to API
const addUser = async (newUser) => {
const response = await fetch('https://api.example.com/users', {
method: 'POST',
headers: { 'Content-Type': 'application/json' },
body: JSON.stringify(newUser)
});
const data = await response.json();
setUsers(prev => [...prev, data]);
};
📤 POST Request Demo
✅ POST method sends data to create new resources
✅ Always set Content-Type: 'application/json' header
✅ body must be JSON stringified
💡 After POST, update state to show new data without refetching
6️⃣ Axios — Popular Alternative to Fetch
// Install: npm install axios

import axios from 'axios';

useEffect(() => {
axios.get('https://jsonplaceholder.typicode.com/users')
.then(response => setUsers(response.data))
.catch(error => setError(error.message));
}, []);
✅ Axios automatically transforms JSON — no need for response.json()
✅ Better error handling (non-2xx status codes throw errors)
✅ Supports request/response interceptors
✅ Works in older browsers
💡 Axios is used in many production React apps!

8️⃣ Environment Variables — Keep API URLs Safe
🔐 Best Practice:
Create a .env file in your project root:
VITE_API_URL=https://jsonplaceholder.typicode.com

Use in React component:
const API_URL = import.meta.env.VITE_API_URL;

⚠️ Never commit .env file to GitHub (add to .gitignore)!

⚠️ Common API Mistakes
  • ❌ Forgetting to add await response.json() → getting Promise object instead of data
  • ❌ Not handling loading state → UI feels unresponsive, users don't know what's happening
  • ❌ Not catching errors → app crashes silently with no user feedback
  • ❌ Making API calls without useEffect → infinite re-render loops
  • ❌ Hardcoding API URLs → use environment variables instead for different environments
💼 Common Interview Questions

❓ How to fetch data in React?

Use fetch() or axios inside useEffect with empty dependency array [] to run once on mount.

❓ Why useEffect for API calls?

To avoid infinite loops and ensure API calls happen after component mounts in the DOM.

❓ Difference between fetch and axios?

Axios auto-parses JSON, has better error handling, supports request/response interceptors.

❓ What are loading and error states?

UI states to show progress or errors during async API calls — essential for good UX.
