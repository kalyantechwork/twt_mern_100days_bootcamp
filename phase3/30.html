1️⃣ What are Events in React?

📌 Purpose & Usage

Events are user actions like clicks, typing, form submission, mouse movements. React events are similar to HTML events but use camelCase syntax (onClick instead of onclick) and are passed as functions (not strings). Events are the bridge between user interaction and React state changes.

Where it's used: Every interactive React application — buttons, forms, search inputs, modals, dropdowns, drag-and-drop, and more.

User Action → React Event Handler → State Update → UI Re-renders
2️⃣ onClick Event — Handling Button Clicks
function ClickDemo() {
const handleClick = () => {
alert("Button was clicked!");
};
return (<button onClick={handleClick}>Click Me</button>);
}
✅ React uses onClick (camelCase) — not onclick (HTML)
✅ Pass the function reference, NOT function call: onClick={handleClick}
❌ Wrong: onClick={handleClick()} — this would run on render!
3️⃣ Passing Arguments to Event Handlers
function UserList() {
const handleDelete = (userId) => {
console.log(`Deleting user: ${userId}`);
};
return <button onClick={() => handleDelete(1)}>Delete User 1</button>;
}
✅ Use arrow function to pass arguments: onClick={() => handleClick(id)}
✅ This creates a new function that calls your handler with the argument
💡 This pattern is very common for lists and dynamic data
4️⃣ onChange Event — Handling Input Changes
function InputDemo() {
const [text, setText] = useState("");
const handleChange = (event) => {
setText(event.target.value);
};
return (
<>
<input value={text} onChange={handleChange} />
<p>You typed: {text}</p>
</>
);
}

You typed:

✅ onChange triggers on every keystroke (or input change)
✅ event.target.value gives the current input value
✅ This pattern is called "controlled component" — React controls the input value
5️⃣ onSubmit Event — Handling Form Submission
function FormDemo() {
const handleSubmit = (event) => {
event.preventDefault(); // PREVENTS PAGE REFRESH
alert("Form submitted!");
};
return (
<form onSubmit={handleSubmit}>
<button type="submit">Submit</button>
</form>
);
}

✅ event.preventDefault() is CRITICAL — prevents page refresh!
✅ Without preventDefault(), React apps would reload on every form submit
✅ Forms without page refresh = SPA (Single Page Application)

8️⃣ Best Practices for Event Handlers

✅ Define outside JSX

Define handlers outside return statement for cleaner, more readable code

✅ Use arrow functions for arguments

onClick={() => handleDelete(id)} — avoids immediate execution

✅ Always preventDefault on forms

Prevents page refresh which would break SPA behavior

✅ Use descriptive function names

handleClick, handleSubmit, handleChange — clear intent

⚠️ Common Beginner Mistakes
  • ❌ Forgetting preventDefault() on form submission → page refreshes, state lost. Always use e.preventDefault().
  • ❌ Calling function instead of passing reference: onClick={handleClick()} ❌ → onClick={handleClick} ✅
  • ❌ Using lowercase event names: onclick ❌ → onClick ✅ (camelCase is required)
  • ❌ Not using event.target.value correctly in onChange — remember: event.target.value
💼 Common Interview Questions

❓ How are React events different from HTML events?

React uses camelCase (onClick) and Synthetic Events (cross-browser wrapper).

❓ Why use preventDefault() in forms?

Prevents default browser behavior (page refresh) so React can handle submission without reload.

❓ How to pass arguments to event handlers?

Use arrow function: onClick={() => handleClick(id)}

❓ What is the event object in React?

SyntheticEvent — React's cross-browser wrapper around native browser event.
