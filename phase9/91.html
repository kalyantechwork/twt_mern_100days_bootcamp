📅 Day 91 · Redux Toolkit (Advanced State Management)

📦 Redux Toolkit
Global State Management · Slices · useSelector · useDispatch · Async Thunks

1️⃣ What is Redux Toolkit? (Why Redux?)

Redux Toolkit is the official, modern way to write Redux logic. It simplifies state management for large applications where Context API becomes complex.

Context API → Good for medium apps (theme, auth) Redux Toolkit → Great for large apps (complex state, frequent updates, powerful devtools)
💡 When to use Redux: Large scale apps with complex state that many components need (shopping cart, user permissions, global notifications).
2️⃣ Installation
npm install @reduxjs/toolkit react-redux
3️⃣ Redux Core Concepts

🎯 Store

Single source of truth — contains entire application state

📝 Slice

Piece of state + reducers + actions (all in one)

🔄 Reducer

Function that updates state based on action type

⚡ Action

Plain object describing what happened (type + payload)

📤 Dispatch

Function that sends actions to the store

👁️ Selector

Function that extracts specific data from store

Component → dispatch(action) → Reducer → Updates Store → Components re-render
4️⃣ Creating a Slice — Counter Example
// features/counter/counterSlice.js
import { createSlice } from '@reduxjs/toolkit';

const initialState = { value: 0 };

const counterSlice = createSlice({
name: 'counter',
initialState,
reducers: {
increment: (state) => { state.value += 1; },
decrement: (state) => { state.value -= 1; },
incrementByAmount: (state, action) => { state.value += action.payload; },
reset: (state) => { state.value = 0; }
}
});

export const { increment, decrement, incrementByAmount, reset } = counterSlice.actions;
export default counterSlice.reducer;
5️⃣ Configuring the Store
// store.js
import { configureStore } from '@reduxjs/toolkit';
import counterReducer from './features/counter/counterSlice';
import cartReducer from './features/cart/cartSlice';

export const store = configureStore({
reducer: {
counter: counterReducer,
cart: cartReducer
}
});
6️⃣ Provide Store to React App
// main.jsx
import React from 'react';
import ReactDOM from 'react-dom/client';
import { Provider } from 'react-redux';
import { store } from './store';
import App from './App';

ReactDOM.createRoot(document.getElementById('root')).render(
<Provider store={store}>
<App />
</Provider>
);
7️⃣ Using Redux — useSelector & useDispatch
🎯 Redux Counter Demo (Live Simulation):
0

✨ Redux state is global — any component can access/modify it!

// Counter.jsx
import { useSelector, useDispatch } from 'react-redux';
import { increment, decrement, reset } from './counterSlice';

function Counter() {
const count = useSelector((state) => state.counter.value);
const dispatch = useDispatch();

return (
<div>
<h2>Count: {count}</h2>
<button onClick={() => dispatch(increment())}>+</button>
<button onClick={() => dispatch(decrement())}>-</button>
<button onClick={() => dispatch(reset())}>Reset</button>
</div>
);
}
8️⃣ Async Actions — API Calls with createAsyncThunk
// features/products/productsSlice.js
import { createSlice, createAsyncThunk } from '@reduxjs/toolkit';
import axios from 'axios';

export const fetchProducts = createAsyncThunk(
'products/fetchProducts',
async () => {
const response = await axios.get('https://fakestoreapi.com/products');
return response.data;
}
);

const productsSlice = createSlice({
name: 'products',
initialState: { items: [], loading: false, error: null },
reducers: {},
extraReducers: (builder) => {
builder
.addCase(fetchProducts.pending, (state) => { state.loading = true; })
.addCase(fetchProducts.fulfilled, (state, action) => {
state.items = action.payload;
state.loading = false;
})
.addCase(fetchProducts.rejected, (state, action) => {
state.error = action.error.message;
state.loading = false;
});
}
});

export default productsSlice.reducer;
9️⃣ Complete Example — Shopping Cart Slice
// features/cart/cartSlice.js
const cartSlice = createSlice({
name: 'cart',
initialState: { items: [], totalQuantity: 0, totalPrice: 0 },
reducers: {
addItem: (state, action) => {
const existingItem = state.items.find(item => item.id === action.payload.id);
if (existingItem) {
existingItem.quantity++;
} else {
state.items.push({ ...action.payload, quantity: 1 });
}
state.totalQuantity++;
state.totalPrice += action.payload.price;
},
removeItem: (state, action) => {
const item = state.items.find(item => item.id === action.payload);
if (item) {
state.totalQuantity -= item.quantity;
state.totalPrice -= item.price * item.quantity;
state.items = state.items.filter(item => item.id !== action.payload);
}
}
}
});

⚠️ Common Redux Mistakes
  • ❌ Directly mutating state in reducers (Redux Toolkit's Immer allows "mutating", but not in plain Redux)
  • ❌ Forgetting to add reducer to store configuration
  • ❌ Not wrapping app with <Provider> → useSelector returns undefined
  • ❌ Putting non-serializable values (functions, promises) in state
💼 Common Interview Questions

❓ What is Redux Toolkit?

Official, modern way to write Redux — less boilerplate, includes Immer and Thunk.

❓ What is a slice?

Collection of reducer logic and actions for a feature.

❓ Redux vs Context?

Redux has devtools, middleware, better for complex state; Context is simpler for medium apps.

❓ What is createAsyncThunk?

Helper function for handling async API calls in Redux.
