1️⃣ What is Docker? (Containers Explained)

Docker is a platform for containerizing applications — packaging code with all its dependencies into a standardized unit that runs consistently anywhere.

App Code + Dependencies + Config → Docker Image → Docker Container (runs anywhere)
💡 Analogy: Docker is like a shipping container. Your app (like cargo) goes into the container, and the container can be shipped (deployed) to any port (server) without worrying about differences.
2️⃣ Why Docker? (Benefits for MERN Developers)

🚀 Consistent Environment

"Works on my machine" problem solved — same environment everywhere

🐳 Easy Deployment

Deploy to any cloud (AWS, GCP, Azure, DigitalOcean) with same commands

📦 Isolated Services

MongoDB, Node.js backend, React frontend in separate containers

⚡ Fast Scaling

Spin up multiple containers for high traffic

4️⃣ Dockerfile — Node.js Backend
# backend/Dockerfile
FROM node:20-alpine

WORKDIR /app

COPY package*.json ./
RUN npm ci --only=production

COPY . .

EXPOSE 5000

CMD ["node", "server.js"]
📝 Best practices: Use Alpine for smaller images, multi-stage builds, don't copy node_modules to cache properly.
5️⃣ Dockerfile — React Frontend (Multi-stage)
# frontend/Dockerfile (multi-stage build)
# Stage 1: Build
FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

# Stage 2: Serve with nginx
FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
6️⃣ Docker Compose — Complete MERN Stack
# docker-compose.yml
version: '3.8'
services:
mongodb:
image: mongo:6
container_name: mongodb
restart: always
ports:
- "27017:27017"
volumes:
- mongodb_data:/data/db
environment:
MONGO_INITDB_ROOT_USERNAME: admin
MONGO_INITDB_ROOT_PASSWORD: password

backend:
build: ./backend
container_name: backend
restart: always
ports:
- "5000:5000"
environment:
- NODE_ENV=production
- MONGO_URI=mongodb://admin:password@mongodb:27017/myapp?authSource=admin
- JWT_SECRET=${JWT_SECRET}
depends_on:
- mongodb

frontend:
build: ./frontend
container_name: frontend
restart: always
ports:
- "80:80"
depends_on:
- backend

8️⃣ Build & Run Docker Containers
# Build all images and start containers
docker-compose up -d --build

# View running containers
docker ps

# View logs
docker-compose logs -f backend

# Stop containers
docker-compose down

# Stop and remove volumes (clear data)
docker-compose down -v
9️⃣ Essential Docker Commands Reference
🔟 Development vs Production Docker

🛠️ Development

• Use nodemon for hot reload
• Mount local volumes for code changes
• Separate docker-compose.override.yml

🚀 Production

• Multi-stage builds (smaller images)
• Environment variables for secrets
• Use nginx to serve React build
• Health checks

1️⃣1️⃣ Health Checks — Monitor Container Status
# Add to backend service in docker-compose.yml
healthcheck:
test: ["CMD", "curl", "-f", "http://localhost:5000/health"]
interval: 30s
timeout: 10s
retries: 3

# Backend health endpoint
app.get('/health', (req, res) => {
res.status(200).json({ status: 'ok', timestamp: new Date() });
});

⚠️ Common Docker Mistakes
  • ❌ Forgetting .dockerignore → copying node_modules (huge image)
  • ❌ Using root user inside containers (security risk)
  • ❌ Hardcoding environment variables in Dockerfile (use .env)
  • ❌ Not handling container networking — frontend can't reach backend by localhost
💼 Common Interview Questions

❓ What is a Docker container?

Lightweight, standalone executable package with everything needed to run an application.

❓ Difference between Docker image and container?

Image = blueprint/template, Container = running instance of image.

❓ What is docker-compose?

Tool to define and run multi-container Docker applications (MERN stack).

❓ Why multi-stage builds?

Smaller final images — build tools in first stage, only runtime artifacts in final.
