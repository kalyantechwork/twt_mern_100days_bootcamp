🚀 Deploy Backend (Render / Railway)
Host Node.js API · MongoDB Atlas · Environment Variables · Production Ready

1️⃣ Why Deploy Backend? (Take Your App Live)

Deploying your backend makes your API accessible from anywhere on the internet. Users can interact with your app without running local servers.

Localhost:5000 → Only your computer Deployed API → Anyone on internet can access (render.com/railway.app)
💡 Analogy: Local development is like rehearsing at home. Deployment is like performing on stage for the whole world!

3️⃣ Prerequisites Before Deployment
  • ✅ Node.js app with Express server (port from env variable: process.env.PORT || 5000)
  • ✅ Package.json with start script: "start": "node server.js"
  • ✅ MongoDB Atlas account for cloud database
  • ✅ GitHub repository with your backend code
4️⃣ MongoDB Atlas — Cloud Database
# Step 1: Sign up at mongodb.com/atlas
# Step 2: Create a cluster (free tier)
# Step 3: Create database user
# Step 4: Network Access → Add IP address 0.0.0.0/0 (allow anywhere)
# Step 5: Get connection string
mongodb+srv://username:password@cluster.mongodb.net/dbname

# Step 6: Replace local MongoDB URI in .env
MONGO_URI=mongodb+srv://user:pass@cluster.mongodb.net/myapp
🔐 Security Tip: Never commit MongoDB credentials to GitHub! Always use environment variables.
5️⃣ Deploy on Render — Step by Step
# Step 1: Push your backend code to GitHub
git add .
git commit -m "Ready for deployment"
git push origin main

# Step 2: Go to render.com → Dashboard → New + → Web Service
# Step 3: Connect GitHub repository
# Step 4: Configure:
# Name: myapp-backend
# Environment: Node
# Build Command: npm install
# Start Command: npm start
# Step 5: Add Environment Variables:
# MONGO_URI = your_atlas_connection_string
# JWT_SECRET = your_secret_key
# PORT = 5000 (Render sets automatically)
# Step 6: Click "Create Web Service"
# Step 7: Wait for build → Your API is live!
6️⃣ Deploy on Railway — Simpler Alternative
# Step 1: Go to railway.app → Start New Project
# Step 2: Deploy from GitHub → Select your repository
# Step 3: Railway auto-detects Node.js
# Step 4: Add environment variables:
# MONGO_URI, JWT_SECRET, etc.
# Step 5: Wait for deployment → Get URL like:
# https://myapp-backend.up.railway.app

# Railway automatically: installs dependencies, runs build, starts server

8️⃣ CORS — Allow Frontend to Call Backend
// backend/server.js
const cors = require('cors');
const allowedOrigins = process.env.NODE_ENV === 'production'
? [process.env.CLIENT_URL, 'https://your-app.vercel.app']
: ['http://localhost:5173'];

app.use(cors({
origin: allowedOrigins,
credentials: true
}));

⚠️ Common Deployment Issues & Solutions
  • ❌ Port already in use? Use process.env.PORT — platforms assign dynamic ports
  • ❌ MongoDB connection error? Check IP whitelist (0.0.0.0/0) and credentials
  • ❌ CORS error? Set CLIENT_URL environment variable correctly
  • ❌ Build failing? Check package.json scripts (start, build)
  • ❌ Environment variables missing? Add them in platform dashboard
