Error Handling Middleware – Professional Error Management

Learn how to handle errors like a senior developer! Centralized error handling, custom error classes, and production-ready error responses.

Telugu WebTech · Professional Error Handling in Express
1. Why Professional Error Handling?
Purpose: Without proper error handling, your app crashes, exposes stack traces to users, and becomes impossible to debug. Professional error handling keeps your app running and protects sensitive info.
Without proper error handling: App crashes randomly, users see ugly stack traces, security vulnerabilities exposed.
With professional error handling: App never crashes unexpectedly, clean user-friendly error messages, centralized logging for debugging.
🏢 Real-world: Production app with 10k users. Without error handler → one database error crashes entire server. With error handler → returns 500 + logs error, server keeps running, users see "Something went wrong" instead of stack trace.
2. Custom Error Class (AppError)
What & why: Extends native Error to add statusCode and isOperational flag. Distinguishes between predictable errors (user input wrong) vs programming bugs (undefined variable).
// src/utils/appError.js\nclass AppError extends Error {\n constructor(message, statusCode) {\n super(message);\n this.statusCode = statusCode;\n this.isOperational = true;\n this.status = `${statusCode}`.startsWith('4') ? 'fail' : 'error';\n Error.captureStackTrace(this, this.constructor);\n }\n}\nmodule.exports = AppError;\n\n// Usage: throw new AppError('Product not found', 404);
📌 Example: `next(new AppError('User with id 999 not found', 404))` → Error middleware catches it, sends 404 response with message. `isOperational=true` tells middleware it's safe to show message to client.
3. catchAsync Wrapper (No More try/catch)
Problem: Every async controller needs try-catch. Solution: Wrapper that catches errors automatically and forwards to next().
// src/utils/catchAsync.js\nconst catchAsync = (fn) => (req, res, next) => {\n fn(req, res, next).catch(next);\n};\nmodule.exports = catchAsync;\n\n// Controller without catchAsync (messy)\nexports.getUser = async (req, res, next) => {\n try { const user = await User.findById(req.params.id); res.json(user); }\n catch(err) { next(err); }\n};\n\n// Controller WITH catchAsync (clean)\nexports.getUser = catchAsync(async (req, res, next) => {\n const user = await User.findById(req.params.id);\n if(!user) return next(new AppError('Not found', 404));\n res.json(user);\n});
✨ Without catchAsync: 5 lines of try-catch per controller. With catchAsync: 0 lines of try-catch. Any thrown error or rejected promise automatically goes to error middleware. Reduces boilerplate by 70%.
4. Complete Error Handling Middleware
// src/middleware/errorMiddleware.js\nconst AppError = require('../utils/appError');\n\nconst sendErrorDev = (err, res) => {\n res.status(err.statusCode).json({\n success: false,\n message: err.message,\n error: err,\n stack: err.stack\n });\n};\n\nconst sendErrorProd = (err, res) => {\n if(err.isOperational) {\n res.status(err.statusCode).json({ success: false, message: err.message });\n } else {\n console.error('ERROR 💥:', err);\n res.status(500).json({ success: false, message: 'Something went wrong!' });\n }\n};\n\nconst errorHandler = (err, req, res, next) => {\n err.statusCode = err.statusCode || 500;\n if(process.env.NODE_ENV === 'development') sendErrorDev(err, res);\n else sendErrorProd(err, res);\n};\n\nmodule.exports = errorHandler;
🖥️ Development: Full stack trace + error object → helps debugging.
🚀 Production: Only clean message for operational errors, generic "Something went wrong" for programming bugs → no sensitive info leaked.
5. Controller with Professional Error Handling
// src/controllers/userController.js\nconst catchAsync = require('../utils/catchAsync');\nconst AppError = require('../utils/appError');\n\nlet users = [{ id: 1, name: 'Ramesh', email: 'ramesh@example.com' }];\n\nexports.getAllUsers = catchAsync(async (req, res) => {\n res.json({ success: true, data: users });\n});\n\nexports.getUser = catchAsync(async (req, res, next) => {\n const user = users.find(u => u.id === parseInt(req.params.id));\n if(!user) return next(new AppError('User not found', 404));\n res.json({ success: true, data: user });\n});\n\nexports.createUser = catchAsync(async (req, res, next) => {\n const { name, email } = req.body;\n if(!name || !email) return next(new AppError('Name and email required', 400));\n const existing = users.find(u => u.email === email);\n if(existing) return next(new AppError('Email already exists', 400));\n const newUser = { id: users.length+1, name, email };\n users.push(newUser);\n res.status(201).json({ success: true, data: newUser });\n});\n\nexports.deleteUser = catchAsync(async (req, res, next) => {\n const index = users.findIndex(u => u.id === parseInt(req.params.id));\n if(index === -1) return next(new AppError('User not found', 404));\n users.splice(index, 1);\n res.json({ success: true, message: 'User deleted' });\n});
📡 GET /users/999 → 404 { success: false, message: "User not found" }
📡 POST /users missing email → 400 { success: false, message: "Name and email required" }
📡 POST duplicate email → 400 { success: false, message: "Email already exists" }
6. Complete Server with Error Handling
// server.js\nrequire('dotenv').config();\nconst express = require('express');\nconst morgan = require('morgan');\nconst cors = require('cors');\nconst helmet = require('helmet');\nconst userRoutes = require('./src/routes/userRoutes');\nconst errorHandler = require('./src/middleware/errorMiddleware');\nconst AppError = require('./src/utils/appError');\n\nconst app = express();\nconst PORT = process.env.PORT || 5000;\n\napp.use(helmet());\napp.use(cors());\napp.use(morgan('dev'));\napp.use(express.json());\napp.use('/api/v1/users', userRoutes);\n\n// 404 handler for unmatched routes\napp.all('*', (req, res, next) => {\n next(new AppError(`Can't find ${req.originalUrl} on this server`, 404));\n});\n\n// Global error handler (MUST be last middleware!)\napp.use(errorHandler);\n\napp.listen(PORT, () => console.log(`✅ Server running on port ${PORT}`));
🔄 Error Flow: Request → routes → controller throws error or calls next(err) → skips remaining middleware → goes to errorHandler → sends formatted response. 404 handler catches any unmatched route.

8. Handling Unhandled Promise Rejections
// Add to server.js for production robustness\nprocess.on('uncaughtException', (err) => {\n console.error('UNCAUGHT EXCEPTION! 💥 Shutting down...');\n console.error(err);\n process.exit(1);\n});\n\nprocess.on('unhandledRejection', (err) => {\n console.error('UNHANDLED REJECTION! 💥');\n console.error(err);\n server.close(() => process.exit(1));\n});
⚠️ Without these, an unhandled promise rejection crashes the entire Node process. With these handlers, you log the error and gracefully shutdown, preventing corrupted state. Essential for production deployments.
9. Common HTTP Error Status Codes
400 Bad Request
Invalid input, missing fields
401 Unauthorized
Authentication required
403 Forbidden
Authenticated but not authorized
404 Not Found
Resource doesn't exist
429 Too Many Requests
Rate limit exceeded
500 Server Error
Database down, programming bug
🎯 400: user sent invalid data. 401: missing login. 403: logged in but wrong role. 404: resource doesn't exist. 500: your server's fault (database down, bug). Always use correct status code — frontend depends on it!
