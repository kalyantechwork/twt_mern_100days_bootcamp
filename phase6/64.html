Controllers – Moving Logic Out of Routes

Learn the Controller pattern – the secret to clean, maintainable Express code! Separate your business logic from routes and become a professional backend developer.

Telugu WebTech · Clean Architecture with Controllers
1. The Problem: Fat Routes (Messy Code)
Why avoid fat routes? When you put all logic directly inside route handlers, code becomes unreadable, untestable, and hard to maintain. Controllers solve this by organizing logic into reusable functions.

❌ Without Controllers

app.get('/api/users', (req, res) => { /* validation, DB, filter, pagination, errors mixed */ });

Hard to read, test, reuse

✅ With Controllers

app.get('/api/users', userController.getAllUsers);

Clean, testable, reusable

❌ Fat route example: 50+ lines of validation, database calls, filtering all inside one callback. ✅ After refactoring: route is one line, controller handles logic separately. Easy to test with Jest/Vitest.
2. What is a Controller?
Purpose: Controllers are JavaScript functions that handle the "business logic" for a specific route. They receive request, process data, and send response. They act as the bridge between routes and models/database.
// Controller signature: (req, res, next) => { ... } const getUsers = (req, res) => { // 1. extract data from req.params, req.query, req.body // 2. validate input // 3. interact with model/database // 4. send response or call next(error) };
🌊 Flow: Request → Route → Controller → Business logic → Response. Example: POST /users → createUser controller runs → validates input → creates user → sends 201 with new user.
3. Basic Controller – User CRUD
What it does: This controller handles all user operations (get all, get by id, create, update, delete). It also processes query filters like ?age=25.
// controllers/userController.js let users = [{ id:1, name:'Ramesh', email:'ramesh@example.com', age:25 }]; const getAllUsers = (req, res, next) => { try { let result = [...users]; if(req.query.age) result = result.filter(u => u.age === parseInt(req.query.age)); res.status(200).json({ success:true, count:result.length, data:result }); } catch(error) { next(error); } }; const getUserById = (req, res, next) => { const user = users.find(u => u.id === parseInt(req.params.id)); if(!user) return res.status(404).json({ success:false, message:'User not found' }); res.json({ success:true, data:user }); }; const createUser = (req, res, next) => { const { name, email, age } = req.body; if(!name || !email) return res.status(400).json({ success:false, message:'Name/email required' }); const newUser = { id: users.length+1, name, email, age: age || 18 }; users.push(newUser); res.status(201).json({ success:true, data:newUser }); }; const updateUser = (req, res) => { /* update logic */ }; const deleteUser = (req, res) => { /* delete logic */ }; module.exports = { getAllUsers, getUserById, createUser, updateUser, deleteUser };
📌 GET /api/users → { "success":true, "count":1, "data":[{"id":1,"name":"Ramesh","email":"ramesh@example.com","age":25}] }
📌 POST /api/users (body: { "name":"Priya", "email":"priya@test.com" }) → 201 Created returns new user with id=2.
4. Clean Routes Using Controllers
Why clean routes? Routes file should only define endpoints and map them to controller functions — no business logic. This makes your API structure instantly understandable.
// routes/userRoutes.js const express = require('express'); const router = express.Router(); const { getAllUsers, getUserById, createUser, updateUser, deleteUser } = require('../controllers/userController'); router.route('/').get(getAllUsers).post(createUser); router.route('/:id').get(getUserById).put(updateUser).delete(deleteUser); module.exports = router; // server.js app.use('/api/users', userRoutes);
✨ Clean routes: each HTTP method points to a controller. No messy logic inside routes file. Easy to navigate: looking at userRoutes.js you instantly know all user endpoints and which controller handles them.
5. Async Controllers (Database Ready)
When & why: Real apps use databases (MongoDB, PostgreSQL). Controllers must be async to await database queries. Always use try/catch and pass errors to next() for proper error handling.
const Product = require('../models/Product'); const getAllProducts = async (req, res, next) => { try { const products = await Product.find(); res.json({ success: true, data: products }); } catch (error) { next(error); // passes to Express error handler } }; const createProduct = async (req, res, next) => { try { const { name, price } = req.body; if(!name || !price) return res.status(400).json({ error: 'Missing fields' }); const newProduct = await Product.create({ name, price }); res.status(201).json({ success: true, data: newProduct }); } catch (err) { next(err); } }; module.exports = { getAllProducts, createProduct };
⏳ Async controller: When request arrives, controller uses "await Product.find()" (non-blocking). Meanwhile Node.js handles other requests. If DB error occurs, next(error) sends it to global error handler. No server freeze!
6. Async Handler Wrapper (No More Try/Catch Repetition)
Problem: Writing try-catch in every controller is repetitive. Solution: Create a wrapper that automatically catches errors and forwards them to next(). This is a professional pattern used in thousands of projects.
// utils/asyncHandler.js const asyncHandler = (fn) => (req, res, next) => { Promise.resolve(fn(req, res, next)).catch(next); }; module.exports = asyncHandler; // controllers/userController.js (clean version!) const asyncHandler = require('../utils/asyncHandler'); const User = require('../models/User'); exports.getAllUsers = asyncHandler(async (req, res) => { const users = await User.find(); // no try/catch needed! res.json({ success: true, data: users }); }); exports.createUser = asyncHandler(async (req, res) => { const user = await User.create(req.body); res.status(201).json({ success: true, data: user }); });
⚡ Without wrapper: each controller needs try/catch. With asyncHandler: just write logic, any error automatically sent to Express error middleware. Reduces boilerplate by 60%! Used in production by major Express APIs.
7. Controller Best Practices
Single responsibility
One function does one thing (getAllUsers, getUserById)
Validate early
Check required fields at start of controller
Consistent response format
{ success, data } or { success, message }
Proper status codes
201 for created, 400 for bad request, 404 for not found
No business logic in routes
Routes only map URL → controller
Use next(error)
Never swallow errors – pass them to error middleware
🏆 Example: A controller that follows best practices -> validates req.body, uses correct status 400 for missing fields, returns { success: false, message: ... }, uses asyncHandler. The route file is clean: router.post('/users', createUser). This makes debugging and testing straightforward.
8. Complete Example – Todo API with Controllers
What you'll learn: Full CRUD with filtering (completed, priority), proper error handling, and modular organization.
// controllers/todoController.js (using asyncHandler) const asyncHandler = require('../utils/asyncHandler'); let todos = [{ id:1, title:'Learn Express', completed:false, priority:'high' }]; exports.getAllTodos = asyncHandler(async (req, res) => { let filtered = [...todos]; if(req.query.completed !== undefined) filtered = filtered.filter(t => t.completed === (req.query.completed === 'true')); if(req.query.priority) filtered = filtered.filter(t => t.priority === req.query.priority); res.json({ success:true, count:filtered.length, data:filtered }); }); exports.getTodoById = asyncHandler(async (req, res) => { const todo = todos.find(t => t.id === parseInt(req.params.id)); if(!todo) return res.status(404).json({ success:false, message:'Todo not found' }); res.json({ success:true, data:todo }); }); exports.createTodo = asyncHandler(async (req, res) => { const { title, priority='medium' } = req.body; if(!title) return res.status(400).json({ success:false, message:'Title required' }); const newTodo = { id: todos.length+1, title, completed: false, priority }; todos.push(newTodo); res.status(201).json({ success:true, data:newTodo }); }); exports.updateTodo = asyncHandler(async (req, res) => { /* similar logic */ }); exports.deleteTodo = asyncHandler(async (req, res) => { /* delete logic */ }); // routes/todoRoutes.js const router = require('express').Router(); const todoController = require('../controllers/todoController'); router.route('/').get(todoController.getAllTodos).post(todoController.createTodo); router.route('/:id').get(todoController.getTodoById).put(todoController.updateTodo).delete(todoController.deleteTodo); // server.js app.use('/api/todos', todoRoutes);
📝 GET /api/todos?completed=false → returns incomplete todos
📝 POST /api/todos { "title": "Study controllers" } → 201 Created: { id:3, title:"Study controllers", completed:false }
📝 GET /api/todos/1 → returns specific todo. Clean, controller-based, easy to extend.
