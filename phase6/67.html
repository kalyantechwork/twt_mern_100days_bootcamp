1. What is Postman?
Purpose: Postman is the most popular API testing tool. It lets you send HTTP requests, inspect responses, write automated tests, and share API documentation — all without writing a single line of code.
Why Postman?
✅ Test APIs without coding
✅ Organize requests in collections
✅ Environment variables for dev/prod
✅ Write automated tests
✅ Generate API docs
✅ Share with team
🏢 Real-world: Your team builds an API. Using Postman, you create a collection with all endpoints + tests. New developers can import it and test immediately. CI/CD runs Newman (Postman CLI) automatically on each deployment.
2. Postman Interface Overview
Request Builder
HTTP method, URL, Params, Headers, Body, Send button
Collections
Save and organize requests into folders
Response Viewer
Status code, time, body (Pretty/Raw/Preview), headers
Environments
Variables like {{baseUrl}} for different stages

8. Writing Automated Tests in Postman
Why automate? Instead of manually checking responses, write tests once. Postman runs them every time you send a request — green means working, red means broken.
📝 Test Script for GET /products
// Add to "Tests" tab pm.test("Status is 200", () => pm.response.to.have.status(200)); pm.test("Response has success, count, data", () => { const json = pm.response.json(); pm.expect(json).to.have.property('success'); pm.expect(json).to.have.property('count'); pm.expect(json).to.have.property('data'); }); pm.test("Success is true", () => pm.response.json().success.should.be.true); pm.test("Data is array", () => pm.expect(pm.response.json().data).to.be.an('array')); // Save first product ID const productId = pm.response.json().data[0]?.id; if(productId) pm.environment.set("productId", productId);
✅ Each test appears in Postman's "Test Results" tab. If status is 200, first test passes. If response structure matches expected, all green. Failing tests highlight exactly what's broken — prevents regression bugs.
📝 Test Script for POST /products
pm.test("Status 201", () => pm.response.to.have.status(201)); pm.test("Returns created product with ID", () => { const json = pm.response.json(); pm.expect(json.success).to.be.true; pm.expect(json.data).to.have.property('id'); }); // Save new ID for later tests pm.environment.set("newProductId", pm.response.json().data.id);
