Express Routes – Deep Dive into Routing

Master Express routing – the heart of any web application! Learn route parameters, query strings, route handlers, chaining, and advanced routing techniques.

Telugu WebTech · Become a Routing Expert
1. What is Routing?
Purpose: Routing determines how your server responds to client requests at specific URLs and HTTP methods. Think of it as a "traffic controller" – based on the URL and HTTP verb (GET, POST...), Express executes the matching function. Without routes, your API has no endpoints!

Routing refers to how an application responds to client requests for specific endpoints (URLs) and HTTP methods (GET, POST, PUT, DELETE).

Example:
📨 GET /users → "Show me all users"
📨 POST /users → "Create a new user"
📨 GET /users/5 → "Show user with ID 5"
📨 DELETE /users/5 → "Delete user with ID 5"
// Basic route structure app.METHOD(PATH, HANDLER); // METHOD: HTTP method (get, post, put, delete) // PATH: URL path (/, /users, /about) // HANDLER: Function that runs when route is matched
💡 What happens?
When a request arrives at GET /users, Express looks for a registered route. If found, it executes the handler and sends a response.
✅ Example output for GET /users → "List of users in JSON"
✅ Example output for POST /users → "New user created" status 201.
2. Route Methods – HTTP Verbs
Why different methods? They follow REST conventions: GET (read), POST (create), PUT/PATCH (update), DELETE (remove). This makes APIs predictable and clean.
app.get()

Fetch data without side effects.

📥 GET /users → responds with: { "users": ["Alice", "Bob"] }
app.post()

Create new resource.

📝 POST /users with body { "name":"John" } → 201 Created, returns new user with ID.
app.put()

Full update.

🔄 PUT /users/1 → replaces entire user data, returns updated user.
app.patch()

Partial update.

✏️ PATCH /users/1 → updates only specific fields (e.g., { "email": "new@mail.com" }).
app.delete()

Remove resource.

🗑️ DELETE /users/1 → removes user, responds with 200 and confirmation message.
app.all()

Matches any HTTP method.

🌐 GET, POST, PUT to /secret → same response: "This route works for any method!"
3. Route Parameters (Dynamic URLs)
What & why: Route parameters let you capture values from the URL (like user ID). Instead of creating separate routes for each user, you write /users/:userId and extract req.params.userId. Very useful for profile pages, product details, etc.
// Single parameter: GET /users/123 -> userId = 123 app.get('/users/:userId', (req, res) => { res.send(`User ID: ${req.params.userId}`); });
📍 Request: GET /users/42
📦 Server output: "User ID: 42"
📍 Multiple params: GET /products/electronics/phone → params = { category: "electronics", productId: "phone" }.
4. Query Parameters (URL ?key=value)
Purpose: Query strings are used for filtering, sorting, pagination (e.g., /products?category=books&page=2). They are optional and do not change the route definition — you access via req.query.
app.get('/search', (req, res) => { const { q, page = 1 } = req.query; res.json({ searchTerm: q, page }); });
🔍 Request: GET /search?q=express&page=3
🧾 Response: { "searchTerm": "express", "page": "3" }
🎯 Use case: /api/products?minPrice=100&maxPrice=500&sort=price → filters products.
5. Route Handlers – Multiple Callbacks
Why multiple handlers? Separate concerns: authentication, validation, logging, then final response. Each callback calls next() to pass control.
app.get('/protected-data', (req, res, next) => { console.log('Auth'); next(); }, (req, res) => { res.json({ data: 'secret' }); } );
▶️ When you request GET /protected-data with valid auth header:
1️⃣ First handler logs "Auth" and calls next()
2️⃣ Second handler sends JSON: { "data": "secret" }
❌ Without auth header → first handler returns 401 and stops.
6. Chaining Routes with app.route()
Benefit: Avoid repeating the path. Group all HTTP methods for the same endpoint in a clean chain.
app.route('/users') .get((req, res) => res.send('Get users')) .post((req, res) => res.send('Create user'));
🔗 GET /users → "Get users"
🔗 POST /users → "Create user"
📌 Clean, readable, easier maintenance.
7. Modular Routes with express.Router()
What & why: For large apps, keep routes in separate files. express.Router() creates mini-apps that you can mount at specific prefixes (e.g., /api/users). Perfect for team collaboration & clean architecture.
// routes/users.js const router = require('express').Router(); router.get('/', (req, res) => res.json({ users: [] })); module.exports = router; // server.js: app.use('/api/users', userRouter);
🧩 Request GET /api/users → calls userRouter's GET '/' handler.
🧩 All routes inside userRouter are relative: /api/users/profile if you define router.get('/profile').
✅ Keeps server.js clean; can be reused across projects.
8. Advanced Route Patterns
Regex, custom param validators, wildcards — gives you flexibility to match complex URLs.
app.get(/.*fly$/, (req, res) => res.send('Fly ending URL')); app.param('id', (req,res,next,id) => { if(isNaN(id)) return res.status(400).send('Invalid'); next(); });
🧪 Regex route: /butterfly, /dragonfly → all match "fly ending".
🛡️ app.param: For route /users/:id, it validates id before main handler, so only numeric ids proceed. Good for reusable validation.
9. Route Order Matters!
Express matches routes in the order they are defined. Put specific routes BEFORE generic ones!
⚠️ Wrong order:
app.get('/users/:id', ...)
app.get('/users/new', ...) → /users/new will be captured by :id and never reaches '/users/new' handler!
✅ Correct: Define '/users/new' before '/users/:id'. Also 404 handler should be last.

🚀 Daily Challenge – Build a Music Playlist API

Task: Create a complete REST API for music playlists with the following routes:

  • GET /api/songs → Get all songs (with query filters: genre, artist, limit, page)
  • GET /api/songs/:id → Get single song
  • POST /api/songs → Create new song (title, artist, genre, duration)
  • PUT /api/songs/:id → Update song
  • PATCH /api/songs/:id/play → Increment play count
  • DELETE /api/songs/:id → Delete song
  • GET /api/artists/:artistName/songs → Get all songs by an artist
  • GET /api/stats/top → Get top 5 most played songs
🎵 Hint: Use in-memory array. For filters: req.query.genre to filter. Pagination: .slice((page-1)*limit, page*limit). For top songs: sort by plays descending and slice(0,5). Example endpoint output for GET /api/stats/top → [ { title: "Blinding Lights", plays: 3000 } , ... ].
