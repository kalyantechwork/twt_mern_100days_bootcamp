Middleware – The Power of Express

Middleware is what makes Express so powerful! Learn how to intercept, modify, and control the request-response cycle like a pro.

Telugu WebTech · Master Express Middleware
1. What is Middleware?
Purpose: Middleware functions have access to `req`, `res`, and `next`. They can modify request/response, end the cycle, or pass control to the next middleware. Think of them as "layers" that every request must pass through.
📨 Request → [Logger] → [Auth] → [Parser] → [Route Handler] → 📬 Response
const myMiddleware = (req, res, next) => { console.log('Do something'); next(); // pass control }; app.use(myMiddleware);
✅ Example: A logger middleware intercepts every request, logs method + URL, then calls next(). The route handler later sends the response. Without middleware, you'd have to add logging to every route manually.
2. Understanding the Middleware Flow
How it works: Middleware functions execute sequentially. Each must call `next()` to proceed to the next one. If you don't call `next()`, the request gets stuck!
app.use((req, res, next) => { console.log('1️⃣ First'); next(); }); app.use((req, res, next) => { console.log('2️⃣ Second'); next(); }); app.get('/api/data', (req, res) => { res.json({ msg: 'Done' }); }); // Visiting /api/data prints: 1️⃣ → 2️⃣ → route handler runs
📌 When a request arrives: Middleware1 runs → next() → Middleware2 runs → next() → route handler sends response. If any middleware skips next(), the chain stops.
3. Five Types of Middleware
Why types? Each type serves different scope: app-level runs globally, router-level scoped to routes, error middleware catches exceptions, built-in provided by Express, third-party from npm.
1. Application-Level
app.use(logger) → runs on all requests
2. Router-Level
router.use(auth) → only for this router
3. Error-Handling
app.use((err,req,res,next) => {}) → 4 params
4. Built-in
express.json(), express.static()
5. Third-Party
morgan, cors, helmet, compression
🛠️ Example: app-level: logger for all requests. router-level: auth only for /admin routes. error: handles crashes gracefully. built-in: express.json() parses JSON. third-party: morgan logs HTTP requests beautifully.
4. Application-Level Middleware (app.use)
What it does: Runs for every incoming request regardless of URL. Perfect for logging, adding request IDs, global rate limiting, or setting common headers.
app.use((req, res, next) => { console.log(`📅 ${new Date().toISOString()} ${req.method} ${req.url}`); req.requestId = Math.random().toString(36).substring(7); next(); }); app.get('/', (req, res) => res.json({ id: req.requestId }));
🖨️ Console: `📅 2025-01-15T10:30:00.000Z GET /`
Response: `{ "id": "k3j9f2" }` → every request gets a unique ID automatically.
5. Router-Level Middleware
Purpose: Attach middleware to specific router instances — only affects routes in that router. Great for route-specific validation, authorization, or parameter validation.
const router = require('express').Router(); router.use((req, res, next) => { console.log('User router only'); next(); }); router.param('userId', (req, res, next, id) => { if(isNaN(id)) return res.status(400).json({ error: 'Invalid ID' }); req.userId = parseInt(id); next(); }); router.get('/users/:userId', (req, res) => res.json({ id: req.userId }));
🎯 GET /users/abc → 400 error "Invalid ID".
GET /users/42 → { "id": 42 }. The middleware validates before route handler runs.
6. Built-in Middleware (Express Defaults)
Why built-in? Express gives you these out-of-the-box: parse JSON, handle URL-encoded forms, serve static files, parse raw/text bodies. No extra installs needed!
app.use(express.json()); // parse JSON bodies → req.body app.use(express.urlencoded({ extended: true })); // parse form data app.use(express.static('public')); // serve static files app.post('/api/data', (req, res) => res.json(req.body));
📦 Client sends JSON `{"name":"Ramesh"}`. express.json() parses it, so req.body = { name: "Ramesh" }. Without it, req.body would be undefined.
7. Third-Party Middleware (Popular Packages)
Community power: Install from npm to add logging (morgan), security (helmet), CORS, compression, rate limiting — production-ready features in minutes.
const morgan = require('morgan'); // HTTP logger const cors = require('cors'); // Enable CORS const helmet = require('helmet'); // Security headers const compression = require('compression'); // Gzip app.use(morgan('dev')); app.use(cors()); app.use(helmet()); app.use(compression());
📊 morgan: logs `GET /api/users 200 5ms` in console. cors: allows your React frontend to call API. helmet: adds security headers (XSS protection). compression: reduces response size by ~70%.
8. Error Handling Middleware (Must be LAST!)
Critical: Error middleware has 4 parameters (err, req, res, next). Place it AFTER all routes. It catches errors from sync code or next(err).
app.get('/error', (req, res, next) => { next(new Error('Something broke!')); // pass to error handler }); app.use('*', (req, res) => res.status(404).json({ error: 'Not found' })); app.use((err, req, res, next) => { console.error(err.stack); res.status(500).json({ error: err.message }); });
⚠️ When route calls `next(err)`, Express skips all normal middleware and jumps to the error handler. Client receives `{ "error": "Something broke!" }`. Without it, the app would crash.
9. Creating Custom Middleware (Real Examples)
Build your own: Authentication, validation, rate limiting, request logging — all reusable across routes.
// Auth middleware const authMiddleware = (req, res, next) => { const token = req.headers.authorization; if (!token) return res.status(401).json({ error: 'No token' }); // verify token logic... req.user = { id: 1 }; next(); }; // Validation middleware const validateProduct = (req, res, next) => { const { name, price } = req.body; if (!name || price <= 0) return res.status(400).json({ error: 'Invalid data' }); next(); }; // Rate limiter middleware (simple) const rateLimiter = () => { const hits = new Map(); return (req, res, next) => { const ip = req.ip; const count = (hits.get(ip) || 0) + 1; if (count > 10) return res.status(429).json({ error: 'Too many requests' }); hits.set(ip, count); next(); }; }; app.post('/api/products', validateProduct, (req, res) => res.json({ ok: true }));
🛡️ POST /api/products with body `{"price": -5}` → validateProduct returns 400 error. With proper data `{"name":"Laptop","price":1000}` → passes to route handler.
10. Middleware Execution Order (CRITICAL!)
Order defines behavior: Middleware runs in the order you declare them. Put generic middleware first, then route-specific, then 404, and error handler LAST.
// ✅ Correct order app.use(express.json()); // 1. Parse body app.use(logger); // 2. Log request app.use('/api', authMiddleware); // 3. Auth for API routes app.get('/api/users', handler); // 4. Route handler app.use('*', notFoundHandler); // 5. 404 for unknown routes app.use(errorHandler); // 6. Error handler (LAST!)
🚨 If you put error handler before routes, it will never catch route errors. If you put 404 before route, all requests become 404. Always: built-in → custom → routes → 404 → error-handler.
11. Complete Example – API with Full Middleware Stack
Production-ready setup: Combines logging, security, parsing, rate limiting, auth, validation, and error handling.
const express = require('express'); const morgan = require('morgan'); const cors = require('cors'); const helmet = require('helmet'); const rateLimit = require('express-rate-limit'); const app = express(); // 1. Security & logging (global) app.use(helmet()); app.use(cors()); app.use(morgan('combined')); app.use(express.json()); // 2. Rate limiting const limiter = rateLimit({ windowMs: 15*60*1000, max: 100 }); app.use('/api', limiter); // 3. Custom auth middleware const auth = (req, res, next) => { if (req.headers['x-token'] !== 'secret123') { return res.status(401).json({ error: 'Unauthorized' }); } next(); }; // 4. Validation middleware const validateUser = (req, res, next) => { if (!req.body.name) return res.status(400).json({ error: 'Name required' }); next(); }; // 5. Routes app.get('/api/public', (req, res) => res.json({ message: 'Public' })); app.post('/api/users', auth, validateUser, (req, res) => { res.json({ user: req.body }); }); // 6. 404 handler app.use('*', (req, res) => res.status(404).json({ error: 'Not found' })); // 7. Error handler (LAST) app.use((err, req, res, next) => { console.error(err); res.status(500).json({ error: 'Server error' }); }); app.listen(3000);
✅ GET /api/public → works for anyone. POST /api/users with header x-token: secret123 and body {name:"Ramesh"} → 200 OK. Without token → 401. Invalid body → 400. Unknown route → 404.
