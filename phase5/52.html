1️⃣ What is NPM? (Node Package Manager)

📌 Purpose & Usage

NPM (Node Package Manager) is the default package manager for Node.js. It has over 2 million packages and is the largest software registry in the world! NPM allows you to install, manage, and share reusable JavaScript code packages.

Where it's used: Every Node.js project — install Express for servers, React for frontend, testing libraries, build tools, and thousands of utilities.

NPM Registry (cloud) → npm install → node_modules (local) → Use in your code
2️⃣ Why NPM? (Benefits)

🔧 Code Reuse

Millions of pre-built packages — don't reinvent the wheel!

🔒 Version Management

Specify exact versions, update safely with semver

📋 package.json

Single file that tracks all dependencies

⚡ Scripts Runner

Run custom scripts like `npm start`, `npm test`

6️⃣ Installing Packages — npm install
# Install a package (adds to dependencies)
npm install lodash

# Install as dev dependency (for testing/building)
npm install --save-dev nodemon

# Install multiple packages
npm install express mongoose dotenv
$ npm install chalk

package.json updated:
"dependencies": {
"chalk": "^5.3.0"
}
✅ npm install downloads packages to node_modules/ and updates package.json automatically!
7️⃣ Semantic Versioning (semver)
"express": "^4.18.2"

// ^4.18.2 means: >=4.18.2 but <5.0.0 (compatible updates)
// ~4.18.2 means: >=4.18.2 but <4.19.0 (patch updates only)
// 4.18.2 means: exact version only

🔴 MAJOR

Breaking changes

🟡 MINOR

New features

🟢 PATCH

Bug fixes

8️⃣ npm Scripts — Custom Commands
"scripts": {
"start": "node index.js",

"test": "jest"
}

// Run scripts
npm start # runs `node index.js`
npm run dev # runs `nodemon index.js`
$ npm run dev
> nodemon index.js

Server running on port 3000
✅ npm scripts simplify running common commands — no need to remember long command lines!
9️⃣ Node.js Modules — require() and module.exports
// math.js (module)
const add = (a, b) => a + b;
const subtract = (a, b) => a - b;
module.exports = { add, subtract };

// app.js (main file)
const math = require('./math.js');
console.log(math.add(5, 3)); // 8
// math.js exports functions
module.exports = { add, subtract };

// index.js imports and uses
const math = require('./math');
console.log(math.add(10, 5)); // 15
✅ require() imports modules, module.exports exports them — this is the CommonJS pattern used in Node.js!

⚠️ Common NPM Mistakes
  • ❌ Forgetting to add node_modules/ to .gitignore → repository becomes huge (hundreds of MB)
  • ❌ Installing packages globally when not needed (use `--save-dev` or no flag for local)
  • ❌ Running `npm install` without package.json — nothing to install!
  • ❌ Using outdated packages — regularly run `npm update` for security fixes
💼 Common Interview Questions

❓ What is NPM?

Node Package Manager — default package manager for Node.js with millions of packages.

❓ What is package.json?

Manifest file containing project metadata, dependencies, and scripts.

❓ Difference between dependencies and devDependencies?

dependencies are for production, devDependencies are for development only (testing, building).

❓ What is the difference between require and import?

require is CommonJS (Node.js default). import is ES6 (modern, needs type:module).
