HTTP Module – Create Your First Web Server

Learn how to turn your computer into a web server using Node.js! Handle requests, send responses, and understand how the web works.

Telugu WebTech · Build Your First Backend Server
1. What is the HTTP Module?

The http module is a built-in core module in Node.js that allows you to create web servers. It handles the HTTP protocol – the language browsers and servers use to talk to each other.

How it works:
Browser (Client) → Sends HTTP Request → Your Node.js HTTP Server → Sends HTTP Response → Browser displays page
// Import http module (no npm install needed!) const http = require('http');

Why this matters: Every website you visit uses HTTP. Learning this helps you understand how the entire web works!

2. Create Your First Web Server (Step by Step)
🎯 Step 1: Create server.js file
// server.js - Your first web server const http = require('http'); // Create a server const server = http.createServer((request, response) => { // Write response header (200 = success, content type = plain text) response.writeHead(200, { 'Content-Type': 'text/plain' }); // Send response body response.end('Hello from Telugu WebTech Server! 🚀'); }); // Start server on port 3000 server.listen(3000, () => { console.log('Server is running at http://localhost:3000'); });
🎯 Step 2: Run your server
# Open terminal in your project folder node server.js # Output: Server is running at http://localhost:3000
🎯 Step 3: Open browser and visit
Type: http://localhost:3000 → You'll see "Hello from Telugu WebTech Server! 🚀"
Congratulations! You just built your first web server! Press Ctrl+C in terminal to stop the server.
3. Understanding Request and Response
📨 Request (what browser sends)
  • URL (which page?)
  • Method (GET, POST, etc)
  • Headers (browser info)
  • Body (form data)
📨 Response (what server sends back)
  • Status Code (200 = OK, 404 = Not Found)
  • Headers (content type, etc)
  • Body (HTML, JSON, or text)
const http = require('http'); const server = http.createServer((req, res) => { // req = request object (from browser) console.log('Request URL:', req.url); console.log('Request Method:', req.method); // res = response object (what we send back) res.writeHead(200, { 'Content-Type': 'text/html' }); res.end(`

You requested: ${req.url}

`); }); server.listen(3000);
4. Handling Different Routes (URL Paths)

In real websites, different URLs show different content. Here's how to handle routes:

const http = require('http'); const server = http.createServer((req, res) => { const url = req.url; // Home page if (url === '/') { res.writeHead(200, { 'Content-Type': 'text/html' }); res.end('

🏠 Welcome Home!

This is the homepage.

'); } // About page else if (url === '/about') { res.writeHead(200, { 'Content-Type': 'text/html' }); res.end('

📖 About Us

Telugu WebTech teaches MERN stack for free!

'); } // Contact page else if (url === '/contact') { res.writeHead(200, { 'Content-Type': 'text/html' }); res.end('

📞 Contact

Email: hello@teluguwebtech.com

'); } // 404 - Page not found else { res.writeHead(404, { 'Content-Type': 'text/html' }); res.end('

❌ 404 - Page Not Found

The page you requested does not exist.

'); } }); server.listen(3000, () => { console.log('Server running on http://localhost:3000'); console.log('Try these routes: / , /about , /contact'); });
Test in browser:
localhost:3000/ → Home page
localhost:3000/about → About page
localhost:3000/anything → 404 page
5. HTTP Status Codes (What Every Number Means)
✅ 200 Series - Success

200 OK → Everything worked!
201 Created → New resource created

🔵 300 Series - Redirect

301 Moved Permanently
302 Found (temporary redirect)

⚠️ 400 Series - Client Error

400 Bad Request
401 Unauthorized
403 Forbidden
404 Not Found

🔥 500 Series - Server Error

500 Internal Server Error
503 Service Unavailable

// Examples of status codes in practice: res.writeHead(200) // Page found res.writeHead(301, { Location: '/new-page' }) // Redirect res.writeHead(404) // Page not found res.writeHead(500) // Server error
6. Serving HTML Files (Not Just Text)

Instead of writing HTML in JavaScript, you can serve actual HTML files. First, create an index.html file:

// index.html (create this file in same folder) /* Telugu WebTech

🎉 My First Node.js Server

This HTML is being served by my HTTP server!

*/ // server.js - Read and serve HTML file const http = require('http'); const fs = require('fs').promises; const path = require('path'); const server = http.createServer(async (req, res) => { if (req.url === '/') { try { const filePath = path.join(__dirname, 'index.html'); const htmlContent = await fs.readFile(filePath, 'utf8'); res.writeHead(200, { 'Content-Type': 'text/html' }); res.end(htmlContent); } catch (error) { res.writeHead(500); res.end('Server Error'); } } else { res.writeHead(404); res.end('Page not found'); } }); server.listen(3000);
7. Creating a Simple JSON API (Backend Basics)

In MERN apps, your backend sends JSON data to React frontend. Here's how:

const http = require('http'); const users = [ { id: 1, name: 'Ramesh', course: 'MERN' }, { id: 2, name: 'Priya', course: 'React' }, { id: 3, name: 'Suresh', course: 'Node.js' } ]; const server = http.createServer((req, res) => { // API endpoint: /api/users if (req.url === '/api/users') { res.writeHead(200, { 'Content-Type': 'application/json' }); res.end(JSON.stringify(users)); } // API endpoint: /api/users/1 (single user) else if (req.url.match(/\/api\/users\/(\d+)/)) { const id = parseInt(req.url.split('/')[3]); const user = users.find(u => u.id === id); if (user) { res.writeHead(200, { 'Content-Type': 'application/json' }); res.end(JSON.stringify(user)); } else { res.writeHead(404, { 'Content-Type': 'application/json' }); res.end(JSON.stringify({ error: 'User not found' })); } } else { res.writeHead(404); res.end('Not found'); } }); server.listen(3000, () => { console.log('JSON API running at http://localhost:3000/api/users'); });
This is exactly how real backends work! React will fetch data from these API endpoints.
8. Reading Query Parameters (Search ?name=value)
const http = require('http'); const url = require('url'); const server = http.createServer((req, res) => { // Parse the URL to get query parameters const parsedUrl = url.parse(req.url, true); const query = parsedUrl.query; // Example: /greet?name=John&city=Hyderabad if (parsedUrl.pathname === '/greet') { const name = query.name || 'Guest'; const city = query.city || 'your city'; res.writeHead(200, { 'Content-Type': 'text/html' }); res.end(`

Hello ${name} from ${city}!

Welcome to Telugu WebTech

`); } else { res.writeHead(404); res.end('Try: /greet?name=Raj&city=Delhi'); } }); server.listen(3000);
9. Common Beginner Errors & Fixes
❌ Error: EADDRINUSE :::3000

Port 3000 is already in use.

✅ Fix: Change port number or kill the process using that port.
❌ Server not responding

Forgot to call res.end()

✅ Fix: Always call res.end() to send response.
❌ Cannot GET /something

No route handler for that URL.

✅ Fix: Add an else block for 404 handling.
