Nodemon – Auto-restart Server on Changes

Stop manually restarting your server! Learn how Nodemon automatically restarts Node.js when you save code changes – saving hours of development time.

Telugu WebTech · Boost Your Development Workflow
1. The Problem: Manual Restart is Painful!

When you build Node.js apps, every time you change your code, you need to:

The Annoying Workflow:
1️⃣ Make a change in server.js
2️⃣ Press Ctrl+C to stop server
3️⃣ Type node server.js to restart
4️⃣ Wait for server to start
5️⃣ Test your change → Repeat hundreds of times a day!

❌ Without Nodemon

Stop → Restart → Wait → Test → Repeat

Wastes time & breaks focus

✅ With Nodemon

Save file → Auto-restarts instantly! → Test

Saves hours every week

2. What is Nodemon?

Nodemon (Node Monitor) is a tool that automatically restarts your Node.js server whenever it detects file changes in your project. It's a development dependency – used only while coding, not in production.

How Nodemon works:
Nodemon watches your files → You save a file → Nodemon detects change → Automatically restarts server → You see your changes instantly!
Fun Fact: "Nodemon" = Node + Daemon (a background process that monitors). Created by Remy Sharp in 2011, now used by millions of developers!
3. Installing Nodemon
📦 Global Installation (use anywhere on your computer)
npm install -g nodemon # or sudo npm install -g nodemon # Mac/Linux with permissions
📦 Local Installation (recommended for projects)
# Inside your project folder npm install --save-dev nodemon # or shorthand npm i -D nodemon
Tip: Local installation is better because each project can have its own version. Add --save-dev since nodemon is only needed for development, not production.
4. Using Nodemon (Super Simple!)
🚀 Instead of node server.js, use:
# If installed globally nodemon server.js # If installed locally (use npx) npx nodemon server.js # Or add to package.json scripts (best way!)
📝 The Best Way: Add to package.json
{ "name": "my-express-app", "version": "1.0.0", "scripts": { "start": "node server.js", "dev": "nodemon server.js" }, "devDependencies": { "nodemon": "^3.0.0" } } # Then run: npm run dev
Now every time you save server.js, Nodemon restarts automatically! Try changing a console.log message and watch it restart.

6. Configuring Nodemon (nodemon.json)

Create a nodemon.json file in your project root to customize behavior:

{ "watch": ["server.js", "src/"], "ext": "js,json,html", "ignore": ["node_modules/", "logs/", "*.test.js"], "delay": "500", "env": { "NODE_ENV": "development", "PORT": 4000 }, "exec": "node server.js", "verbose": true }
📁 watch
Which folders/files to monitor (default: current directory)
🔧 ext
File extensions to watch (default: js, mjs, json)
🚫 ignore
Files/folders to ignore (like node_modules)
⏱️ delay
Wait before restarting (prevents multiple restarts)
7. Useful Nodemon Commands & Options
# Watch specific files/folders nodemon --watch src --watch server.js server.js # Ignore specific files nodemon --ignore public/* --ignore test/* server.js # Set environment variables nodemon --env NODE_ENV=production server.js # Specify file extension to watch nodemon --ext js,json,ejs server.js # Delay restart (milliseconds) nodemon --delay 1.5 server.js # Show extra debug info nodemon --verbose server.js # Graceful shutdown (kill all child processes) nodemon --signal SIGTERM server.js

mern-project/ ├── backend/ │ ├── server.js │ ├── controllers/ │ ├── models/ │ ├── routes/ │ ├── package.json │ └── nodemon.json ├── frontend/ │ └── src/ └── .gitignore // package.json scripts { "scripts": { "dev": "nodemon server.js", "start": "node server.js" } }

9. Common Nodemon Issues & Fixes
❌ "nodemon: command not found"

Nodemon not installed or not in PATH.

✅ Fix: Use npx nodemon server.js or install globally.
❌ Windows: Script execution policy error

PowerShell restricts running scripts.

✅ Fix: Use npx nodemon or run PowerShell as admin: Set-ExecutionPolicy RemoteSigned
❌ Not restarting on changes

File might be in ignored folder or nodemon not watching.

✅ Fix: Check nodemon.json watch settings or run with --verbose to debug.
❌ Port already in use after crash

Previous node process still running.

✅ Fix: Kill process: npx kill-port 3000 or restart terminal.
10. Alternative Tools (When to Use What)
nodemon
✅ Most popular, simple, works great
✅ Watches files, restarts Node
✅ Best for beginners and most projects
node --watch (Node.js 18+)
✅ Built-in, no install needed
⚠️ Less features than nodemon
Usage: node --watch server.js
pm2
✅ Production process manager
✅ Keeps apps running forever
❌ Overkill for development
ts-node-dev
✅ For TypeScript projects
✅ Faster restart for TS
Usage: ts-node-dev server.ts
