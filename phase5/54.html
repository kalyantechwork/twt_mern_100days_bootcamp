File System (fs) Module – Read, Write & Manage Files

Learn how to interact with your computer's file system using Node.js. Create files, read data, update content, delete files — all with simple JavaScript!

Telugu WebTech · Master Backend with fs module
1. What is the File System (fs) Module?

The fs module is a built-in core module in Node.js that allows you to work with the file system on your computer. You can read, write, update, delete files and folders. It's like having a superpower to manage files using JavaScript code!

Your Node.js app ↔ fs module ↔ Hard Drive / SSD
You write code → fs reads/writes actual files → perfect for logs, user data, configuration files.

// Import the fs module (no npm install needed!) const fs = require('fs');
2. Synchronous vs Asynchronous methods

fs gives you two styles of doing file operations. For beginners, understand the difference:

🛑 Synchronous (blocking)

Code waits until file operation finishes. Simple to understand, but can freeze your app for large files.

const data = fs.readFileSync('./file.txt', 'utf8');
console.log(data); // waits until done

✅ Asynchronous (non-blocking) 👍 BEST

Doesn't block the program. Uses callbacks or promises. Recommended for real apps.

fs.readFile('./file.txt', 'utf8', (err, data) => {
if(err) throw err;
console.log(data);
});
console.log('This runs first!');
Beginner Tip: Start with async + callbacks, then learn fs.promises (modern async/await) on Day 55. For learning, both are fine!
3. Reading Files (readFile / readFileSync)
📖 Asynchronous read (callback style)
const fs = require('fs'); fs.readFile('./welcome.txt', 'utf8', (err, data) => { if (err) { console.error('Error reading file:', err); return; } console.log('File content:', data); });
📖 Synchronous read (blocking but simple)
try { const content = fs.readFileSync('./data.txt', 'utf8'); console.log(content); } catch (err) { console.error('Something went wrong:', err); }

What is 'utf8'? It tells Node to give you text instead of raw binary data. Without it you get buffer data.

4. Writing & Appending to Files
✍️ Write (overwrites file or creates new)
const fs = require('fs'); fs.writeFile('myLog.txt', 'Hello Telugu WebTech!', 'utf8', (err) => { if (err) console.error(err); else console.log('File written successfully ✅'); }); // Synchronous version: fs.writeFileSync('syncLog.txt', 'Important data');
➕ Append (add content to end of file)
// Keeps old content and adds new text at the end fs.appendFile('myLog.txt', '\nNew line added!', 'utf8', (err) => { if (err) throw err; console.log('Content appended'); }); // Also exists: appendFileSync
5. Delete & Rename Files
🗑️ Delete file: fs.unlink
fs.unlink('./tempFile.txt', (err) => { if (err) console.error('Delete failed', err); else console.log('File deleted!'); });
✏️ Rename file: fs.rename
fs.rename('oldName.txt', 'newName.txt', (err) => { if (err) throw err; console.log('File renamed successfully'); });
6. Working with Directories (mkdir, readdir, rmdir)
📁 Create folder: fs.mkdir
fs.mkdir('./myNewFolder', { recursive: true }, (err) => { if (err) console.log(err); else console.log('Folder created!'); });
📋 Read folder contents: fs.readdir
fs.readdir('./', (err, files) => { if (err) throw err; console.log('Files in current directory:', files); });
🗑️ Remove empty folder: fs.rmdir
fs.rmdir('./emptyFolder', (err) => { if (err) console.error(err); else console.log('Folder removed'); });
7. Check if File Exists (fs.stat / fs.existsSync)

Before reading or deleting, you might want to know if a file exists:

const fs = require('fs'); // Simple synchronous check (recommended for beginners) if (fs.existsSync('./config.json')) { console.log('Config file exists!'); } else { console.log('Config file not found'); } // Advanced: fs.stat gives more info (size, modified time) fs.stat('./data.txt', (err, stats) => { if (err) console.log('File does not exist'); else { console.log(`File size: ${stats.size} bytes`); console.log(`Last modified: ${stats.mtime}`); } });
8. Real Project Example – Logger with fs

Imagine building a server that logs every request to a file. This is how real MERN backends work!

// logger.js - custom logger module const fs = require('fs'); function logMessage(message) { const timestamp = new Date().toISOString(); const logEntry = `[${timestamp}] ${message}\n`; // Append to 'app.log' file fs.appendFile('app.log', logEntry, (err) => { if (err) console.error('Logging failed', err); }); } module.exports = { logMessage }; // In your main app.js: // const logger = require('./logger'); // logger.logMessage('Server started on port 5000'); // logger.logMessage('User John logged in');
Every time someone visits your website → log entry saved in app.log file → you can debug later.
9. Visual: Async vs Sync Performance

🔵 Asynchronous (non-blocking):
Task 1: Read file (starts) → while waiting, Task 2 runs → file ready → callback executes → faster for multiple operations.

🔴 Synchronous (blocking):
Task 1: Read file (waits until fully read) → then Task 2 runs → slower for many I/O operations.

✅ Best practice: Use async methods (callbacks or promises) in real web servers.

10. Common fs Errors Beginners Face
Error: ENOENT: no such file or directory

👉 You tried to read a file that doesn’t exist. Solution: Check file path or use fs.existsSync() first.

Error: EACCES: permission denied

👉 Node doesn't have rights to write. Run terminal as admin or change folder permissions.

Callback Hell / Pyramid of Doom

👉 Nested callbacks become messy. Learn fs.promises + async/await (Day 55).

11. 🚀 Daily Challenge – Build a Note Taker CLI

Task: Create a simple command-line notes app using fs module that can:

  • 📝 Write a new note (overwrite notes.txt)
  • ➕ Append a new line to notes.txt
  • 📖 Read and display the note content
const fs = require('fs'); const command = process.argv[2]; const newNote = process.argv[3]; if (command === 'write') { fs.writeFile('notes.txt', newNote + '\n', err => { if (err) throw err; console.log('Note saved!'); }); } else if (command === 'append') { fs.appendFile('notes.txt', newNote + '\n', err => { if (err) throw err; console.log('Note appended!'); }); } else if (command === 'read') { fs.readFile('notes.txt', 'utf8', (err, data) => { if (err) console.log('No notes yet'); else console.log('Your notes:\n', data); }); } else { console.log('Usage: node app.js write "My note" OR node app.js read'); }
