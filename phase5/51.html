1️⃣ What is Node.js?

📌 Purpose & Usage

Node.js is a JavaScript runtime built on Chrome's V8 engine that allows you to run JavaScript outside the browser — on the server, command line, or any computer. It enables full-stack JavaScript development, meaning you can use the same language for both frontend and backend.

Where it's used: Backend APIs, real-time applications (chat, gaming), CLI tools, web scraping, automation scripts, build tools (Webpack, Vite). Used by Netflix, LinkedIn, Uber, PayPal, NASA.

Before Node.js → JavaScript only in browsers ❌
After Node.js → JavaScript anywhere (servers, desktop apps, IoT) ✅
2️⃣ Why Node.js? (Benefits)

⚡ Same Language

Use JavaScript everywhere — frontend AND backend. No need to learn Python, PHP, or Java for backend.

🚀 Non-Blocking I/O

Handles thousands of concurrent connections efficiently — perfect for real-time apps.

📦 NPM Ecosystem

Largest package registry with over 2 million packages. Install anything with npm install.

🏢 Used by Giants

Netflix, LinkedIn, Uber, PayPal, NASA use Node.js in production.

7️⃣ Node.js Global Object
console.log(global); // Global object (like window)
console.log(__dirname); // Current folder path
console.log(__filename); // Current file path
setTimeout(() => console.log("After 1s"), 1000);
console.log(__dirname);
/Users/kalyan/projects/node-demo

console.log(__filename);
/Users/kalyan/projects/node-demo/app.js
✅ __dirname = directory path, __filename = file path — very useful for file operations!
8️⃣ Process Object — Environment & Arguments
// Command line arguments
console.log(process.argv);
// Run: node app.js hello world
// Output: ['node', 'app.js', 'hello', 'world']

// Environment variables
console.log(process.env.USER);
$ node app.js --name Kalyan --age 25
process.argv: ['node', 'app.js', '--name', 'Kalyan', '--age', '25']
process.env.USER: kalyan
✅ process.argv gives command line arguments, process.env gives environment variables — essential for CLI tools!
9️⃣ Creating and Running Node.js Scripts

🔟 Node.js Architecture — V8 Engine & Libuv
JavaScript Code → V8 Engine (compiles JS) → Libuv (handles async I/O) → Operating System
🏗️ Built on: Chrome's V8 JavaScript engine + libuv (C++ library for async I/O). This combination makes Node.js fast and efficient for I/O-heavy operations.

⚠️ Common Node.js Beginner Mistakes
  • ❌ Trying to use DOM APIs (document, window) in Node.js → doesn't exist! Node.js has no browser environment.
  • ❌ Forgetting to install Node.js before trying to run scripts → download from nodejs.org.
  • ❌ Using alert() in Node.js → use console.log() instead.
  • ❌ Not understanding that Node.js is single-threaded but handles concurrency via event loop.
💼 Common Interview Questions

❓ What is Node.js?

JavaScript runtime built on Chrome's V8 engine that runs JavaScript outside the browser.

❓ Is Node.js single-threaded?

Yes, but it uses an event loop and libuv to handle concurrent operations efficiently.

❓ What is NPM?

Node Package Manager — default package manager for Node.js with millions of packages.

❓ Difference between Node.js and browser JS?

Node.js has no DOM, has file system access, uses CommonJS modules, and runs on servers.
