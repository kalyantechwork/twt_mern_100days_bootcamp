1. What is a Module?

In Node.js, a module is like a small, independent toolbox. Each file (.js) is a module. You write code in one file, and then export parts of it to be used in another file using require() or import. This helps avoid messy code and makes maintenance easy.

📁 project/
├── math.js → module (exports functions)
├── app.js → imports math.js and uses it

↺ Each file = one module. Reuse anywhere!
2. Three Types of Modules

🔧 Core Modules

Built into Node.js (no install). Examples: fs, http, path, os.

const fs = require('fs');

📁 Local Modules

Your own files: create, export, and import functions or variables.

const myMath = require('./math.js');

📦 Third-party Modules

Install via npm (like express, mongoose, lodash).

npm install express
const express = require('express');
3. Module Systems: require() vs import

Node.js traditionally uses CommonJS (require/module.exports). But modern JavaScript uses ES Modules (import/export). For beginners, we focus on CommonJS — still widely used and simple.

CommonJS (default in Node)
// 📁 greet.js function sayHello(name) { return `Hello ${name}!`; } module.exports = { sayHello }; // 📁 app.js const greet = require('./greet.js'); console.log(greet.sayHello('Telugu WebTech'));
ES Modules (modern)
// 📁 greet.mjs (or "type": "module" in package.json) export function sayHello(name) { return `Hi ${name}`; } // app.mjs import { sayHello } from './greet.mjs'; console.log(sayHello('learner'));

Beginner tip: Stick with require and module.exports — no extra config needed! Perfect for learning.

4. Hands-on: Build your first local module

Let’s create a reusable calculator module and use it in another file. This is the heart of modular programming.

✅ Step 1: Create calculator.js (module)
// calculator.js : a simple math module function add(a, b) { return a + b; } function subtract(a, b) { return a - b; } function multiply(a, b) { return a * b; } const PI = 3.1416; // Export everything we want to be available outside module.exports = { add, subtract, multiply, PI };
✅ Step 2: Create app.js (main file)
// app.js - import our custom calculator module const mathOps = require('./calculator.js'); console.log(mathOps.add(15, 7)); // 22 console.log(mathOps.subtract(20, 5)); // 15 console.log(mathOps.multiply(4, 3)); // 12 console.log(mathOps.PI); // 3.1416
How it works: calculator.js exports functions → app.js uses require() → gets object with all methods.
5. Exploring Node.js Core Modules

These modules come for free with Node.js. No installation needed. Here are 3 vital ones:

File System (fs)
const fs = require('fs'); // Read a file fs.readFile('./hello.txt', 'utf8', (err, data) => { if (err) console.error(err); else console.log(data); });
HTTP module (create server)
const http = require('http'); const server = http.createServer((req, res) => { res.writeHead(200, { 'Content-Type': 'text/plain' }); res.end('Namaste from Node.js!'); }); server.listen(3000, () => console.log('Server running on port 3000'));
Path module (handle file paths safely)
const path = require('path'); const fullPath = path.join(__dirname, 'views', 'index.html'); console.log(fullPath); // shows absolute path
6. Behind the scenes: Module Wrapper

Node.js doesn't run your module code directly. It wraps it inside a function that gives you special variables like __dirname, __filename, exports, require, module. That's why you can use them without defining!

What Node.js does internally:
(function(exports, require, module, __filename, __dirname) {
// your module code here
});

✅ This keeps variables inside a module private unless you export them.

Caching: Node.js caches modules after the first require(). That means if you require same module multiple times, it runs only once — improves performance!
7. Real Project Structure using modules

Imagine building a backend for a blog. Using modules we separate concerns:

📁 my-blog-backend/ ├── 📁 utils/ │ └── logger.js (export log function) ├── 📁 routes/ │ └── postRoutes.js (export route handlers) ├── app.js (main server) ├── db.js (database connection)
Example: logger.js (utility module)
// utils/logger.js function info(message) { console.log(`[INFO] ${new Date().toISOString()} - ${message}`); } function error(message) { console.error(`[ERROR] ${message}`); } module.exports = { info, error };
Using it in app.js
const logger = require('./utils/logger'); logger.info("Server starting..."); // Output: [INFO] 2025-05-29T... - Server starting...
8. Visual: How require() loads a module

📥 require('./myModule') steps:

1️⃣ Check cache → if cached, return cached exports
2️⃣ Load file content
3️⃣ Wrap code inside wrapper function
4️⃣ Evaluate & execute module code
5️⃣ Return module.exports
6️⃣ Store in cache for future requires.

This modular pattern is why Node.js apps are scalable and maintainable.

9. Mistakes Beginners Make & Pro Tips
❌ Forgetting to export

If you don't use module.exports, the imported variable will be {} empty object.

✅ Always check export

module.exports = { myFunction } ;

❌ Circular dependencies

Module A requires Module B and vice versa → confusing.

✅ Keep modules small & single-purpose

One module = one responsibility (e.g., database, helper, routes).
