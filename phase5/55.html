Path Module + Modern fs.promises

Master file paths like a pro! Learn the Path module for cross-platform file handling and fs.promises for modern async/await file operations.

Telugu WebTech · Clean Code with Path & Promises
1. Why do we need the Path module?

Different operating systems use different path separators:

  • Windows uses backslash: C:\projects\file.txt
  • Mac/Linux uses forward slash: /home/user/file.txt

The Path module solves this problem! It automatically uses the correct separators for any OS. Your code works everywhere.

// Import path module (built-in, no install) const path = require('path');
Write once, run anywhere: path.join('folder', 'file.txt') → Works on Windows, Mac, Linux!
2. Essential Path Methods with Examples
📂 path.join() – Join path segments safely
const path = require('path'); const fullPath = path.join('users', 'john', 'documents', 'resume.pdf'); console.log(fullPath); // On Windows: users\john\documents\resume.pdf // On Mac/Linux: users/john/documents/resume.pdf
📍 path.resolve() – Get absolute path
// Resolves a sequence of paths into an absolute path const absolute = path.resolve('projects', 'app.js'); console.log(absolute); // Outputs full path from root: /Users/yourname/projects/app.js
📛 path.basename() – Get filename from path
const filePath = '/home/user/photo.jpg'; const fileName = path.basename(filePath); console.log(fileName); // photo.jpg // Remove extension too: const nameOnly = path.basename(filePath, '.jpg'); console.log(nameOnly); // photo
📁 path.dirname() – Get folder path
const folder = path.dirname('/home/user/photo.jpg'); console.log(folder); // /home/user
🔌 path.extname() – Get file extension
const ext = path.extname('script.js'); console.log(ext); // .js const ext2 = path.extname('style.css'); console.log(ext2); // .css
📋 path.parse() – Get all parts as object
const parsed = path.parse('/users/anna/project/index.html'); console.log(parsed); // { // root: '/', // dir: '/users/anna/project', // base: 'index.html', // ext: '.html', // name: 'index' // }
3. __dirname & __filename – Your Secret Weapons

Node.js gives you two special variables in every module:

  • __dirname → Current folder path (where your file lives)
  • __filename → Full path of current file
// In your app.js file console.log(__dirname); // /Users/you/project console.log(__filename); // /Users/you/project/app.js // Build paths relative to current file (ALWAYS works!) const configPath = path.join(__dirname, 'config', 'database.json'); console.log(configPath);
Pro Tip: Always use path.join(__dirname, 'someFolder') to make your file paths work no matter where your project is located!
4. Modern File Operations: fs.promises + async/await

Remember callbacks from Day 54? Now we use Promises – much cleaner code! fs.promises gives you Promise-based versions of all fs methods.

Callback Hell → Promises → async/await (Modern & Beautiful!)
✅ Read file with fs.promises
const fs = require('fs').promises; // Notice .promises! async function readMyFile() { try { const data = await fs.readFile('welcome.txt', 'utf8'); console.log('File content:', data); } catch (error) { console.error('Error reading file:', error.message); } } readMyFile();
✍️ Write file with fs.promises
const fs = require('fs').promises; async function writeExample() { try { await fs.writeFile('note.txt', 'Hello Telugu WebTech!'); console.log('File written!'); const data = await fs.readFile('note.txt', 'utf8'); console.log('Content:', data); } catch (err) { console.log('Oops:', err); } } writeExample();
📂 Read directory contents
const fs = require('fs').promises; async function listFiles() { const files = await fs.readdir(__dirname); console.log('Files in current directory:', files); } listFiles();
5. Evolution of File Operations (Visual Comparison)
🔴 Old Way (Callbacks) – Messy
fs.readFile('a.txt', 'utf8', (err, data) => { if(err) throw err; fs.writeFile('b.txt', data, (err) => { if(err) throw err; console.log('Done'); }); });
✅ Modern Way (async/await) – Clean!
const fs = require('fs').promises; async function copyFile() { const data = await fs.readFile('a.txt', 'utf8'); await fs.writeFile('b.txt', data); console.log('Done'); }

For MERN projects: Always use fs.promises + async/await. It's cleaner and easier to debug!

6. Real-World Example: Load Config File Safely

In real backends, you often read configuration files. Here's a robust example using Path + fs.promises:

const fs = require('fs').promises; const path = require('path'); async function loadConfig() { // Build absolute path to config file (works anywhere!) const configPath = path.join(__dirname, 'config', 'app-config.json'); try { const data = await fs.readFile(configPath, 'utf8'); const config = JSON.parse(data); console.log('Config loaded:', config); return config; } catch (error) { if (error.code === 'ENOENT') { console.error('Config file missing at:', configPath); } else { console.error('Error reading config:', error.message); } return null; } } loadConfig();
7. Path Module – Quick Reference
path.join()
Joins segments with OS-specific separator
path.resolve()
Returns absolute path
path.basename()
Gets filename from path
path.dirname()
Gets directory name
path.extname()
Gets file extension (.js, .html)
path.parse()
Returns object with all parts
8. fs.promises – Common Methods
const fs = require('fs').promises; fs.readFile() // Read file fs.writeFile() // Write/overwrite file fs.appendFile() // Add to end of file fs.unlink() // Delete file fs.readdir() // Read folder contents fs.mkdir() // Create folder fs.rmdir() // Remove folder fs.rename() // Rename file/folder fs.stat() // Get file info

All these return Promises, so you can use await inside an async function!

9. Mistakes Beginners Make (and How to Fix)
❌ Using + to join paths

'folder' + '/' + 'file.txt' → Breaks on Windows!

✅ Always use path.join()
❌ Forgetting .promises

const fs = require('fs'); await fs.readFile() → Doesn't work!

✅ const fs = require('fs').promises;
❌ Not using try/catch with async/await

Errors crash your app silently.

✅ Always wrap in try/catch
10. 🚀 Daily Challenge – File Explorer Tool

Task: Build a simple script that:

  1. Uses path.join() with __dirname to point to a 'data' folder
  2. Creates the 'data' folder if it doesn't exist (using fs.mkdir)
  3. Writes a file called 'info.txt' with your name and today's date
  4. Reads the file back and prints the content
  5. Uses ONLY fs.promises and async/await
const fs = require('fs').promises; const path = require('path'); async function fileExplorer() { const dataFolder = path.join(__dirname, 'data'); // Create folder if needed try { await fs.mkdir(dataFolder, { recursive: true }); console.log('✅ Folder ready'); } catch (err) { console.log('Folder exists or error:', err.message); } // Write file const filePath = path.join(dataFolder, 'info.txt'); const content = `Name: Telugu WebTech Learner\nDate: ${new Date().toDateString()}`; await fs.writeFile(filePath, content); console.log('✅ File written'); // Read file const readContent = await fs.readFile(filePath, 'utf8'); console.log('📄 File content:\n', readContent); } fileExplorer();
