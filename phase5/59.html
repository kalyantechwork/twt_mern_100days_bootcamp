Environment Variables – dotenv & Keeping Secrets Safe

Learn how to protect passwords, API keys, and sensitive data using environment variables. Essential for real-world MERN applications!

Telugu WebTech · Security Best Practices
1. What are Environment Variables?

Environment variables are special variables that live outside your code. They store configuration data like passwords, API keys, database URLs, and secret tokens.

Why use environment variables?
❌ Hardcoding secrets: const password = "mySecret123" → Anyone can see!
✅ Environment variables: process.env.DB_PASSWORD → Safe and hidden!

📁 Different environments have different variables:
Development (localhost) → Testing → Production (live website)
Never commit secrets to GitHub! Always use .gitignore to exclude .env files.
2. Accessing Environment Variables with process.env

Node.js gives you access to environment variables through the global object process.env.

// Access environment variables console.log(process.env.USER); // Your computer username console.log(process.env.PATH); // System paths console.log(process.env.NODE_ENV); // Usually 'development' or 'production' // Run with: USERNAME=Ramesh node app.js // You can set variables before running your script
📝 Setting variables temporarily (command line):
# Windows (Command Prompt) set PORT=5000 && node server.js # Windows (PowerShell) $env:PORT=5000; node server.js # Mac / Linux PORT=5000 node server.js
This method is temporary! Next terminal session, variables are gone. For permanent solutions, use dotenv package.
3. The .env File – Store All Secrets in One Place

The dotenv package loads variables from a .env file into process.env. This is the standard way to manage configuration.

📦 Step 1: Install dotenv
npm install dotenv
📁 Step 2: Create a .env file in your project root
# .env file (DO NOT commit to GitHub!) PORT=3000 DB_NAME=mern_tutorial DB_USER=admin DB_PASSWORD=SuperSecret123 JWT_SECRET=my_super_secret_key_change_this API_KEY=abc123xyz789 NODE_ENV=development
⚙️ Step 3: Load and use variables
// Load environment variables from .env file require('dotenv').config(); // Now you can access them anywhere! const port = process.env.PORT || 3000; const dbName = process.env.DB_NAME; const jwtSecret = process.env.JWT_SECRET; console.log(`Server will run on port: ${port}`); console.log(`Database: ${dbName}`); console.log(`JWT Secret is set: ${jwtSecret ? '✅ Yes' : '❌ No'}`); // Use in your server const http = require('http'); const server = http.createServer((req, res) => { res.end(`Environment: ${process.env.NODE_ENV}`); }); server.listen(port, () => { console.log(`🚀 Server running on http://localhost:${port}`); });
4. .gitignore – Never Leak Secrets to GitHub
DANGER: If you commit .env to GitHub, anyone can see your passwords! Hackers scan GitHub for exposed .env files.
📝 Create a .gitignore file:
# .gitignore .env .env.local .env.production node_modules/ .DS_Store
📋 Create a .env.example file (share structure, not secrets):
# .env.example – Share this on GitHub PORT= DB_NAME= DB_USER= DB_PASSWORD= JWT_SECRET= API_KEY= NODE_ENV=development
Best Practice: Commit .env.example to GitHub so other developers know what variables they need. Never commit the real .env!
5. Real MERN Project – Database Connection with .env

Here's how a real MERN backend uses environment variables for MongoDB connection:

// .env file PORT=5000 MONGODB_URI=mongodb://localhost:27017/mern_ecommerce JWT_SECRET=your_jwt_secret_key_here JWT_EXPIRES_IN=7d EMAIL_USER=your_email@gmail.com EMAIL_PASSWORD=app_specific_password CLOUDINARY_CLOUD_NAME=your_cloud_name CLOUDINARY_API_KEY=123456789 CLOUDINARY_API_SECRET=abcdefg // server.js require('dotenv').config(); const mongoose = require('mongoose'); const express = require('express'); const app = express(); const PORT = process.env.PORT || 5000; // Connect to MongoDB using environment variable mongoose.connect(process.env.MONGODB_URI) .then(() => console.log('✅ MongoDB Connected')) .catch(err => console.error('MongoDB Error:', err)); // JWT secret (used for authentication) const jwtSecret = process.env.JWT_SECRET; if (!jwtSecret) { console.error('❌ JWT_SECRET is not defined!'); process.exit(1); } app.listen(PORT, () => { console.log(`🚀 Server running on port ${PORT}`); console.log(`Environment: ${process.env.NODE_ENV}`); });
6. Different Environments – Development vs Production
Development (.env.development)
  • Local database
  • Debug logging ON
  • Mock APIs
  • Error details shown
Production (.env.production)
  • Cloud database (MongoDB Atlas)
  • Debug logging OFF
  • Real API keys
  • Error details hidden
// Load different .env files based on NODE_ENV const envFile = `.env.${process.env.NODE_ENV || 'development'}`; require('dotenv').config({ path: envFile }); console.log(`Loading config from: ${envFile}`); console.log(`Database: ${process.env.DB_NAME}`); // Run with: // NODE_ENV=production node server.js // NODE_ENV=development node server.js
7. Validating Environment Variables (Don't Let Your App Crash!)

Always check that required environment variables exist before starting your app:

require('dotenv').config(); const requiredEnvVars = [ 'PORT', 'MONGODB_URI', 'JWT_SECRET', 'API_KEY' ]; const missingVars = requiredEnvVars.filter(varName => !process.env[varName]); if (missingVars.length > 0) { console.error('❌ Missing required environment variables:'); missingVars.forEach(varName => console.error(` - ${varName}`)); console.error('\nPlease create a .env file with these variables'); process.exit(1); // Stop the app } console.log('✅ All environment variables are present!'); // Or use a package like 'envalid' for advanced validation: // npm install envalid
8. Environment Variables in React (Create React App / Vite)

React also uses environment variables, but with REACT_APP_ prefix (CRA) or VITE_ prefix (Vite).

📁 Create React App (.env file in root)
# .env (Create React App) REACT_APP_API_URL=http://localhost:5000/api REACT_APP_GOOGLE_MAPS_KEY=abc123
// In your React component const apiUrl = process.env.REACT_APP_API_URL; console.log(apiUrl); // http://localhost:5000/api
⚡ Vite (.env file)
# .env (Vite) VITE_API_URL=http://localhost:5000/api VITE_STRIPE_KEY=pk_test_123
// In your React component (Vite) const apiUrl = import.meta.env.VITE_API_URL; console.log(apiUrl);
Important: Environment variables in React are embedded in the browser! Never put secrets (passwords, API keys) in frontend .env files – use your backend for that.
9. Common Beginner Mistakes
❌ Not restarting server after .env changes

dotenv loads variables only once at startup.

✅ Fix: Restart Node.js after changing .env file.
❌ Forgetting to require('dotenv').config()

process.env.VAR returns undefined.

✅ Fix: Add dotenv config at the VERY TOP of your file.
❌ Committing .env to GitHub

Your secrets are now public! Hackers will find them.

✅ Fix: Add .env to .gitignore immediately.
❌ Spaces around = in .env file

API_KEY = abc (with spaces) – not correct!

✅ Fix: API_KEY=abc (no spaces)
10. 🚀 Daily Challenge – Secure Configuration Manager

Task: Create a secure configuration system for a MERN app:

  • Create a .env file with 5+ configuration variables (PORT, DB_NAME, ADMIN_EMAIL, APP_NAME, FEATURE_FLAG)
  • Create a .env.example file (for GitHub)
  • Write a Node.js script that loads all variables and displays them (mask passwords partially)
  • Add validation – if any required variable is missing, show error and exit
  • Create a simple HTTP server that uses PORT from .env and responds with APP_NAME
// config.js – Complete challenge solution starter require('dotenv').config(); const requiredVars = ['PORT', 'APP_NAME', 'ADMIN_EMAIL']; const missing = requiredVars.filter(v => !process.env[v]); if (missing.length > 0) { console.error('❌ Missing variables:', missing.join(', ')); console.error('Create .env file with these variables'); process.exit(1); } console.log('=' .repeat(50)); console.log('✅ Configuration loaded successfully'); console.log('=' .repeat(50)); console.log(`📱 App Name: ${process.env.APP_NAME}`); console.log(`🌐 Port: ${process.env.PORT}`); console.log(`📧 Admin Email: ${process.env.ADMIN_EMAIL}`); console.log(`🗄️ Database: ${process.env.DB_NAME || 'default_db'}`); console.log(`🚩 Feature Flag: ${process.env.FEATURE_FLAG || 'disabled'}`); console.log('=' .repeat(50)); // HTTP Server const http = require('http'); const server = http.createServer((req, res) => { res.writeHead(200, { 'Content-Type': 'text/html' }); res.end(`

${process.env.APP_NAME}

Environment: ${process.env.NODE_ENV || 'development'}

Server running on port ${process.env.PORT}

Admin: ${process.env.ADMIN_EMAIL}

`); }); server.listen(process.env.PORT, () => { console.log(`🚀 Server running on http://localhost:${process.env.PORT}`); }); // Your .env file should look like: /* PORT=4000 APP_NAME=TeluguWebTech Secure App ADMIN_EMAIL=admin@teluguwebtech.com DB_NAME=challenge_db FEATURE_FLAG=enabled NODE_ENV=development */
