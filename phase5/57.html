Events Module – Event-Driven Architecture

Learn how Node.js handles events like a pro! Understand EventEmitter, create custom events, and build reactive applications.

Telugu WebTech · Master Node.js Events
1. What is Event-Driven Architecture?

Node.js is event-driven – meaning it reacts to events as they happen. Instead of waiting for tasks to complete, Node.js listens for events and responds immediately.

Real World Example:
📱 You order food on Swiggy/Zomato → "Order Placed" event triggers →
→ Restaurant gets notification → Delivery partner assigned → You get SMS
Everything happens because of EVENTS!
Key Concept: In event-driven programming, we have:
• Event Emitters – Things that create events
• Event Listeners – Functions that wait for events and respond
2. The Events Module & EventEmitter Class

The events module is built into Node.js. It provides the EventEmitter class – the heart of event-driven programming.

// Import events module const EventEmitter = require('events'); // Create an instance of EventEmitter const myEmitter = new EventEmitter(); // Register a listener (wait for event) myEmitter.on('greet', (name) => { console.log(`Hello ${name}! Welcome to Telugu WebTech`); }); // Emit (trigger) the event myEmitter.emit('greet', 'Ramesh'); // Output: Hello Ramesh! Welcome to Telugu WebTech
How it works:
myEmitter.on('eventName', callback) → Listener (waits)
myEmitter.emit('eventName', data) → Trigger (shout out!)
When emit happens → all listeners run immediately
3. Creating Your Own Custom Events

You can create any event name you want! Here are practical examples:

const EventEmitter = require('events'); const orderEmitter = new EventEmitter(); // Listener for 'orderPlaced' event orderEmitter.on('orderPlaced', (orderDetails) => { console.log(`📦 New order received: ${orderDetails.item} - ₹${orderDetails.price}`); console.log(`📧 Sending confirmation email to ${orderDetails.email}`); }); // Listener for 'paymentSuccess' event orderEmitter.on('paymentSuccess', (amount) => { console.log(`✅ Payment of ₹${amount} successful!`); console.log(`📱 Sending payment receipt via SMS`); }); // Listener for 'orderDelivered' event orderEmitter.on('orderDelivered', (orderId) => { console.log(`🎉 Order ${orderId} delivered to customer!`); console.log(`⭐ Request customer for feedback`); }); // Trigger events in sequence orderEmitter.emit('orderPlaced', { item: 'MERN Course', price: 999, email: 'student@example.com' }); orderEmitter.emit('paymentSuccess', 999); orderEmitter.emit('orderDelivered', 'ORD-1234'); /* OUTPUT: 📦 New order received: MERN Course - ₹999 📧 Sending confirmation email to student@example.com ✅ Payment of ₹999 successful! 📱 Sending payment receipt via SMS 🎉 Order ORD-1234 delivered to customer! ⭐ Request customer for feedback */
4. Multiple Listeners for the Same Event

You can attach multiple functions to the same event. All of them will run when the event is emitted!

const EventEmitter = require('events'); const userEmitter = new EventEmitter(); // Listener 1: Save to database userEmitter.on('userRegistered', (user) => { console.log(`💾 Saving user ${user.name} to MongoDB...`); }); // Listener 2: Send welcome email userEmitter.on('userRegistered', (user) => { console.log(`📧 Sending welcome email to ${user.email}`); }); // Listener 3: Create default playlist for user userEmitter.on('userRegistered', (user) => { console.log(`🎵 Creating default playlist for ${user.name}`); }); // Trigger the event userEmitter.emit('userRegistered', { name: 'Priya', email: 'priya@example.com' }); /* OUTPUT: 💾 Saving user Priya to MongoDB... 📧 Sending welcome email to priya@example.com 🎵 Creating default playlist for Priya */
Useful methods:
emitter.listenerCount('event') → How many listeners?
emitter.removeListener('event', listener) → Remove specific listener
5. One-Time Events with .once()

Sometimes you want an event to trigger only once, even if emit is called multiple times. Use .once()!

const EventEmitter = require('events'); const appEmitter = new EventEmitter(); // This will run only ONCE appEmitter.once('appStarted', () => { console.log('🚀 Application started! (This message appears only once)'); }); appEmitter.emit('appStarted'); appEmitter.emit('appStarted'); appEmitter.emit('appStarted'); // Output: "🚀 Application started! (This message appears only once)" // The message appears only ONE time, subsequent emits do nothing
Real use case: Database connection – you only want to log "Connected" once, not every time you query!
6. Passing Data with Events (Multiple Arguments)
const EventEmitter = require('events'); const logger = new EventEmitter(); // Can pass multiple arguments logger.on('error', (statusCode, message, timestamp) => { console.log(`❌ ERROR [${timestamp}]: ${message} (Status: ${statusCode})`); }); logger.on('success', (...args) => { // Spread operator for any number of args console.log(`✅ SUCCESS:`, ...args); }); // Emit with data logger.emit('error', 404, 'Page not found', new Date().toISOString()); logger.emit('success', 'User logged in', 'user@example.com', 'Admin role'); /* OUTPUT: ❌ ERROR [2025-01-15T10:30:00.000Z]: Page not found (Status: 404) ✅ SUCCESS: User logged in user@example.com Admin role */
7. Creating Your Own EventEmitter Classes

In real projects, you create classes that extend EventEmitter to add event capabilities to your own objects.

const EventEmitter = require('events'); // Create a custom class that can emit events class BankAccount extends EventEmitter { constructor(owner, balance = 0) { super(); // Call parent constructor this.owner = owner; this.balance = balance; } deposit(amount) { this.balance += amount; // Emit an event when money is deposited this.emit('deposit', { owner: this.owner, amount, newBalance: this.balance }); return this.balance; } withdraw(amount) { if (amount > this.balance) { this.emit('insufficientFunds', { owner: this.owner, attempted: amount, balance: this.balance }); return false; } this.balance -= amount; this.emit('withdraw', { owner: this.owner, amount, newBalance: this.balance }); return true; } } // Create an account const rameshAccount = new BankAccount('Ramesh', 5000); // Listen to events rameshAccount.on('deposit', (data) => { console.log(`💰 ${data.owner} deposited ₹${data.amount}. New balance: ₹${data.newBalance}`); }); rameshAccount.on('withdraw', (data) => { console.log(`💸 ${data.owner} withdrew ₹${data.amount}. Remaining: ₹${data.newBalance}`); }); rameshAccount.on('insufficientFunds', (data) => { console.log(`⚠️ ${data.owner} tried to withdraw ₹${data.attempted} but only has ₹${data.balance}`); }); // Test the account rameshAccount.deposit(2000); rameshAccount.withdraw(1000); rameshAccount.withdraw(10000); // This will trigger insufficientFunds /* OUTPUT: 💰 Ramesh deposited ₹2000. New balance: ₹7000 💸 Ramesh withdrew ₹1000. Remaining: ₹6000 ⚠️ Ramesh tried to withdraw ₹10000 but only has ₹6000 */
8. Built-in Events in Node.js (HTTP, Streams, etc.)

Many Node.js modules use events internally. For example, the HTTP server emits events!

const http = require('http'); const server = http.createServer((req, res) => { res.writeHead(200); res.end('Hello Events!'); }); // Server emits 'request' event (same as the callback above) server.on('request', (req, res) => { console.log(`📨 Request received: ${req.method} ${req.url}`); }); // Server emits 'listening' event server.on('listening', () => { console.log('🎧 Server is listening on port 3000'); }); // Server emits 'close' event server.on('close', () => { console.log('🛑 Server has been closed'); }); server.listen(3000); // To close server after 5 seconds: setTimeout(() => { server.close(); }, 5000); /* OUTPUT: 🎧 Server is listening on port 3000 📨 Request received: GET / 📨 Request received: GET /about 🛑 Server has been closed */
Did you know? Streams (readable/writable), Child Processes, and many core modules inherit from EventEmitter!
9. Error Events – Always Listen to 'error'!

If an error occurs in an EventEmitter and no listener handles it, Node.js throws an exception and crashes. Always listen for 'error' events!

const EventEmitter = require('events'); const safeEmitter = new EventEmitter(); // GOOD: Listen to error events safeEmitter.on('error', (err) => { console.error('❌ Caught error:', err.message); console.log('Application continues running...'); }); // Emit an error (won't crash) safeEmitter.emit('error', new Error('Something went wrong')); // Without error listener, Node.js would crash! console.log('✅ Program keeps running normally'); /* OUTPUT: ❌ Caught error: Something went wrong Application continues running... ✅ Program keeps running normally */
10. 🚀 Daily Challenge – Build a Smart Home Event System

Task: Create an event-driven smart home system with these features:

  • Events: doorOpen, doorClose, motionDetected, alarmTriggered
  • When door opens → turn on light & log "Door opened"
  • When motion detected after 10pm → trigger alarm
  • Count how many times door opened (use a variable)
  • Use .once() for first motion of the day
const EventEmitter = require('events'); class SmartHome extends EventEmitter { constructor() { super(); this.doorOpenCount = 0; this.isAlarmOn = false; } openDoor() { this.doorOpenCount++; this.emit('doorOpen', { count: this.doorOpenCount, time: new Date().toLocaleTimeString() }); } closeDoor() { this.emit('doorClose'); } detectMotion(hour) { this.emit('motionDetected', { hour }); } triggerAlarm(reason) { this.isAlarmOn = true; this.emit('alarmTriggered', { reason, time: new Date().toLocaleTimeString() }); } } const myHome = new SmartHome(); // Event listeners myHome.on('doorOpen', (data) => { console.log(`🚪 DOOR OPENED! (Count: ${data.count}) at ${data.time}`); console.log(`💡 Turning ON the lights...`); }); myHome.on('doorClose', () => { console.log(`🚪 Door closed. Lights staying on for 30 seconds.`); }); myHome.on('motionDetected', (data) => { console.log(`🏃 Motion detected at ${data.hour}:00`); if (data.hour >= 22 || data.hour < 6) { myHome.triggerAlarm('Motion detected after 10 PM'); } }); myHome.once('motionDetected', () => { console.log(`📱 [FIRST MOTION OF THE DAY] Sending notification to owner`); }); myHome.on('alarmTriggered', (data) => { console.log(`🔔🚨 ALARM! ${data.reason} at ${data.time}`); console.log(`📞 Calling security...`); }); // Simulate activities myHome.openDoor(); myHome.closeDoor(); myHome.detectMotion(14); // 2 PM - no alarm myHome.detectMotion(23); // 11 PM - triggers alarm! myHome.openDoor(); /* Expected output includes door events, first motion special message, and alarm */
